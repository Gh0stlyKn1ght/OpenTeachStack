# OTS-000 Artifact Matrix

Status: draft support document.

Use this matrix to review whether each OTS-000 chapter produces a useful artifact for the final Teacher Builder Starter Kit.

## Chapter artifact map

| Chapter | Artifact | What it should prove | Review question |
|---|---|---|---|
| 00. Welcome to the Teacher Builder Stack | Teacher builder vocabulary map | The teacher can explain basic tool and project terms in plain language. | Does the map reduce intimidation without oversimplifying risk? |
| 01. Your Computer as a Builder Workstation | Builder workstation checklist | The teacher understands what their device can and cannot safely do. | Does the checklist avoid hardware shaming and name realistic setup limits? |
| 02. Files, Folders, and Project Structure | Labeled project-folder map | The teacher can identify source, output, docs, config, and assets. | Does the map prevent editing stale or unsafe files? |
| 03. What Is an IDE? | IDE navigation diagram | The teacher can identify Explorer, Search, Terminal, Source Control, and Extensions. | Does the diagram make an IDE feel navigable rather than developer-only? |
| 04. VS Code for Teacher Builders | VS Code survival sheet | The teacher has a repeatable safe path for opening, finding, editing, previewing, and reviewing. | Does it support one safe first edit? |
| 05. The Terminal Without Fear | Terminal command cheat sheet | The teacher can explain basic commands, current folder, and command risk. | Does every command include location and effect? |
| 06. Git and GitHub Basics | GitHub project-reading checklist | The teacher can inspect a repo before editing or reusing material. | Does it distinguish finding a repo from trusting a repo? |
| 07. Documentation Literacy | Documentation evaluation checklist | The teacher can identify official docs, page type, version, and risk. | Does it prevent blind trust in tutorials or AI summaries? |
| 08. Markdown, MDX, and Content Files | Starter Markdown lesson | The teacher can create a structured student-facing content file. | Does the sample lesson teach, or only demonstrate syntax? |
| 09. Websites, Hosting, and Publishing | Website publishing flow diagram | The teacher can separate source, preview, build, deployment, published URL, and student view. | Does it separate technical success from classroom readiness? |
| 10. The Modern Project Stack | Project stack card | The teacher can name project layers and edit boundaries. | Does it identify what needs technical review? |
| 11. AI Assistants, Coding Agents, and MCP | AI tool comparison and MCP access map | The teacher can compare AI tools by access, risk, and review needs. | Does it make read/edit/run/connect boundaries visible? |
| 12. Safety, Backups, Secrets, and Permissions | Safe AI builder checklist | The teacher can stop unsafe AI or automation work before it damages the project. | Does it include real stop conditions? |
| 13. Capstone: Teacher Builder Starter Kit | Teacher Builder Starter Kit | The teacher can assemble a practical reference for future project work. | Does the final kit support OTS-101, OTS-240, OTS-301, or OTS-320 readiness? |

## Artifact quality bar

Each artifact should be:

- useful to a tired teacher
- short enough to revisit
- specific enough to guide a decision
- safe to share after private details are removed
- connected to a future pathway course
- honest about what the teacher still needs help with

## Review flags

Mark an artifact for revision if:

- it only repeats lesson notes
- it has no safety boundary
- it cannot guide a real decision
- it assumes a developer background
- it uses tool names without explaining the job
- it asks teachers to install or connect tools without a permission check
- it contains real student data, secrets, private URLs, or account details

## Final packet check

The final Teacher Builder Starter Kit should include at least:

- setup summary
- vocabulary map
- workstation checklist
- project-folder map
- VS Code survival sheet
- terminal cheat sheet
- GitHub checklist
- documentation links
- starter Markdown lesson
- publishing flow diagram
- project stack card
- AI/MCP access map
- Safe AI Builder Checklist

If an artifact does not help the teacher start safer project work, revise or remove it before release.
