# OTS-000 Human Review Checklist

Status: draft review required.

Use this checklist before changing OTS-000 from Coming Soon to public-facing draft or release status.

## Course Promise

The course should help teachers answer:

> What are these tools, why do they matter, what do I install, and how do I not break everything?

## Review Scope

- Chapters `00-13`
- Templates in `content/courses/ots-000/templates`
- Completed sample packet in `content/courses/ots-000/examples`
- Course-owned metadata in `course.json` and `status.json`

## Voice Review

Check that lessons sound:

- teacher-to-teacher
- beginner-friendly
- practical under classroom pressure
- skeptical of fake polish
- not coding-heavy
- not vendor hype
- not corporate professional development

## Content Review

For each chapter, confirm:

- The teacher problem is concrete.
- The explanation is plain-language.
- Weak and better versions are specific.
- The build step produces a usable artifact.
- The quality check helps a teacher decide what is ready.
- Safety, privacy, accessibility, source, or permission boundaries are named where relevant.
- The capstone connection is clear.

## Technical Accuracy Review

Check especially:

- VS Code descriptions
- terminal command descriptions
- Git vs GitHub distinctions
- Markdown and MDX explanations
- hosting, deployment, build, domain, and DNS vocabulary
- Node/npm/project stack descriptions
- AI assistant vs coding-agent distinctions
- MCP access and trust-boundary explanations
- secrets, `.env`, API keys, and privacy guidance

## Artifact Review

Confirm the templates are useful:

- Teacher Builder Starter Kit
- Builder Workstation Checklist
- Documentation Evaluation Checklist
- AI Tool Comparison and MCP Access Map
- Safe AI Builder Checklist

Confirm the completed sample packet:

- uses fake-safe details
- avoids private student data
- gives a realistic beginner model
- does not imply one required tool setup
