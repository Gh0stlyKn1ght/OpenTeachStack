> OTS-000 frame: you are not trying to become a software engineer. You are learning enough tool literacy to protect your work, understand the project, and ask better questions.

## The Teacher Problem

Modern curriculum work now touches words many teachers were never taught: workspace, repo, terminal, IDE, Markdown, build, deploy, agent, MCP, source of truth.

That vocabulary can make a capable teacher feel behind before the real work even starts.

## The Idea

This chapter builds the map first.

A teacher builder is an educator who can use software tools to organize, publish, verify, and revise course materials without pretending the tool is the curriculum.

You do not need to master every tool. You need to know what each tool is for, what it can touch, and what safety boundary belongs around it.

## Classroom Scenario

A teacher opens a curriculum project and sees folders named `content`, `src`, `app`, `public`, `scripts`, and `docs`.

A teammate says, "Just open it in VS Code, run npm install, and preview the route."

That sentence is not impossible. It is just stacked with unexplained words.

## Weak Version

```text
I am not technical, so I should stay out of this project.
```

This gives the tools too much power.

## Better Version

```text
I do not know every tool yet, but I can identify the project folder, open the README, find the content files, and ask what command safely previews the site.
```

That is enough to begin.

## Build Step

Start a teacher builder vocabulary map with three columns:

| Word | Plain meaning | Why a teacher cares |
|---|---|---|
| Project folder | The folder that holds the whole project | This is what you open in VS Code |
| Source of truth | The file or folder that owns the real content | This prevents editing the wrong copy |
| Preview | A local view before publishing | This lets you check before students see it |

Add five words you already hear often.

## Quality Check

A useful vocabulary map should help a tired teacher say, "I know what this tool is doing in the workflow."

## Safety / Accessibility / Source Note

Tool literacy is a safety issue. If a teacher cannot tell whether a tool is editing local files, uploading data, or publishing online, they cannot set a safe boundary around student-facing content.
