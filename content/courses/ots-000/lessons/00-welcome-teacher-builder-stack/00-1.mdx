## The Teacher Problem

Teachers are being asked to manage course sites, links, files, rubrics, slides, videos, AI drafts, source notes, and revisions.

When those pieces live in separate places, the teacher ends up holding the whole system in memory.

That breaks under classroom pressure.

## The Idea

The teacher builder stack is the set of tools and habits a teacher uses to build, organize, verify, publish, and maintain course materials.

It can include a computer, folders, VS Code, GitHub, Markdown, a course platform, AI assistants, documentation links, and safety routines.

The stack is not the goal. The goal is a course content system that students and teachers can actually use.

## Classroom Scenario

A teacher wants a small robotics safety unit. The work includes a lesson page, checklist, rubric, short video link, source note, and place to publish it.

That is not just "making a worksheet." It is building a small system.

## Weak Version

```text
I need whatever tools developers use so I can build this correctly.
```

This starts too wide.

## Better Version

```text
I need the smallest tool stack that lets me write the lesson, track the source, preview the page, publish safely, and revise later.
```

This keeps the stack tied to the teaching job.

## Build Step

Add a "tool job" section to your vocabulary map.

| Job | Tool example | Teacher use |
|---|---|---|
| Write | Markdown, Docs, MDX | Draft student-facing content |
| Organize | folders, GitHub, Drive | Keep versions findable |
| Preview | local site, LMS preview | Check before publishing |
| Verify | source bank, checklists, tests | Catch errors before students inherit them |
| Publish | LMS, website, Vercel, GitHub Pages | Put content where students can reach it |

## Quality Check

If a tool does not have a clear job, do not add it yet. Tools without jobs become clutter.

## Safety / Accessibility / Source Note

A stack should reduce risk, not create mystery. Avoid workflows where only one person knows which file is current, where links came from, or what was published.

## Reflection

Which part of your current teaching workflow is most fragile: writing, organizing, previewing, verifying, or publishing?
