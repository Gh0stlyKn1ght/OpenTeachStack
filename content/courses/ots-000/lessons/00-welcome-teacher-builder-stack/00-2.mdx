## The Teacher Problem

A teacher can be excellent at instruction and still get boxed out by tool language.

Someone says "clone the repo," "check the route," "open the terminal," or "review the diff," and the teacher has to decide whether to ask, guess, or step away.

Guessing is where projects get damaged.

## The Idea

Tool literacy is not coding expertise.

Tool literacy means you can answer basic workflow questions:

- What am I opening?
- What am I editing?
- What could this command change?
- Is this local or published?
- Where is the source of truth?
- What do I need to verify before students see it?

That is teacher judgment used inside software work.

## Classroom Scenario

An AI assistant offers to "update all lessons." That sounds helpful. But if the teacher does not know which folder contains authored lessons, the assistant might overwrite real content while cleaning up fake content.

Tool literacy is what lets the teacher say, "Only touch this folder. Show the diff. Do not overwrite authored files."

## Weak Version

```text
The AI probably knows the project better than I do. Let it clean everything.
```

That is not support. That is surrendering the course.

## Better Version

```text
The AI can help, but I will define the folder, the task, the files not to touch, and the checks that prove the change worked.
```

This keeps the teacher in charge.

## Build Step

Add three teacher control questions to your vocabulary map:

1. What is the tool allowed to access?
2. What is the tool allowed to change?
3. What evidence will I inspect before accepting the work?

Use those questions for any unfamiliar software tool.

## Quality Check

You do not need to explain every internal detail. You do need to explain the tool's job, access, risk, and verification step in plain language.

## Safety / Accessibility / Source Note

Tool literacy protects student privacy. If a tool can read files, upload data, connect accounts, or publish pages, it deserves a clear boundary before student information appears anywhere near it.

## Reflection

Where have you been treating a tool like magic because the vocabulary felt too technical?
