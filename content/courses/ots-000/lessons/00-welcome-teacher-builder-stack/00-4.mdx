## The Teacher Problem

Tool words disappear from memory when they stay abstract.

A teacher hears "repo" during a meeting, understands it for a minute, then forgets how it connects to files, GitHub, or the course site.

You need a small artifact you can return to.

## The Idea

A vocabulary map is not a glossary for show.

It is a working map of the tools, containers, and safety questions that affect your curriculum projects.

## Classroom Scenario

A teacher is preparing to ask for help with a broken course page. Instead of saying, "The website thing is broken," the teacher checks the map and can say, "The local preview works, but the published site is not updating after deployment."

That is a better help request.

## Weak Version

| Word | Definition |
|---|---|
| GitHub | A coding website |
| VS Code | A coding app |
| Terminal | A scary command place |

This is honest, but not useful enough yet.

## Better Version

| Word | Plain meaning | Teacher use | Safety question |
|---|---|---|---|
| GitHub | A place that stores project files and history | Keep curriculum project versions inspectable | Am I changing the right branch or file? |
| VS Code | A project editor | Open folders, search files, edit content | Am I editing the source of truth? |
| Terminal | A place to run typed project commands | Install, preview, test, or build a project | What folder am I in before I run this? |

## Build Step

Create a table with these columns:

```text
Word | Plain meaning | Teacher use | Safety question | Related course
```

Add at least ten entries.

## Quality Check

Each row should pass the tired-teacher test:

- Can I explain the word without sounding like a vendor page?
- Can I say why it matters to course work?
- Can I name one risk or safety question?

## Safety / Accessibility / Source Note

Do not paste passwords, account details, private URLs, student names, or API keys into this map. It is a learning artifact, not a secret storage file.

## Reflection

Which word did you understand better after writing the safety question?

## Capstone Connection

This vocabulary map becomes the first section of your Teacher Builder Starter Kit.

## Artifact Build Scan

### Artifact

Build or revise **Build Task: Teacher Builder Vocabulary Map** as a small working artifact, not a decorative note.

### Build Steps

1. Name the classroom job this artifact is supposed to do.
2. Fill in the artifact with the smallest useful version a teacher could reuse.
3. Add one piece of evidence that shows the artifact is usable.
4. Mark anything unfinished as revise or blocked instead of hiding it.

### Example Output

A finished example should be specific enough that another teacher can tell what changed, where it lives, and how it would be used in a real course build.

### Quality Check

- The artifact has a clear title.
- The teacher decision or classroom use is visible.
- The evidence is saved in a form someone else can inspect.
- The next revision trigger is named if the artifact is not ready.

### Capstone Connection

Save this artifact in the Teacher Builder Starter Kit. It becomes part of the evidence trail for deciding whether the starter kit is ready, needs revision, or is blocked.
