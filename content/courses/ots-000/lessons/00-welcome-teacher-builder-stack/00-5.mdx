## The Teacher Problem

A vocabulary map can look finished while still being useless.

If it only copies definitions, it will not help when a teacher is tired, a link is broken, or an AI agent asks for permission to edit files.

## The Idea

A good orientation artifact helps you make a decision.

This checkpoint asks whether your vocabulary map gives you enough language to ask better questions and set safer boundaries.

## Classroom Scenario

A teammate says, "Can you check whether the content source is different from the platform export?"

If your map helps you explain that difference, it is working.

## Weak Version

```text
I listed the words, so I am done.
```

Listing is not the same as understanding.

## Better Version

```text
I can use the map to explain what each tool does, why it matters, and what I should check before using it.
```

That is the goal.

## Build Step

Review your map and mark each row:

- Keep: clear and useful
- Revise: too vague
- Add example: understandable but not concrete
- Ask for help: still confusing

Then revise at least three rows.

## Quality Check

Use this checklist:

- At least ten terms are included.
- Each term has a plain meaning.
- Each term has a teacher use.
- Each term has a safety question.
- At least one term connects to OTS-101, OTS-240, OTS-301, or OTS-320.

## Safety / Accessibility / Source Note

If a term involves account access, file editing, publishing, or AI permissions, the safety question is required. Do not leave that cell blank.

## Reflection

Which term still needs a better example before you would trust yourself to use it in a project conversation?

## Capstone Connection

Save the revised map. It is now part of your Teacher Builder Starter Kit and will expand as the course introduces more tools.

## Checkpoint Decision Language

Use this checkpoint to make a plain decision about **Chapter Checkpoint**:

- **Ready** means the artifact can be used by another teacher without a side explanation from you.
- **Revise** means the idea is right, but the artifact still needs clearer steps, labels, evidence, or safer wording.
- **Blocked** means you cannot honestly finish the artifact yet because a tool, source, permission, example, or decision is missing.

## Evidence to Save

Save evidence that another teacher could inspect: a screenshot, copied command output, file path, short reflection note, or before-and-after excerpt. If the evidence is only in your memory, the checkpoint is not ready yet.

## Next Revision Trigger

Name the owner and trigger before you move on. Write who will revise it, when they will revisit it, and what would cause the next revision: a broken link, confusing student direction, missing source, unsafe permission, or a teammate review note.
