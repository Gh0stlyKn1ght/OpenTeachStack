> OTS-000 frame: the goal is not to buy the fanciest machine. The goal is to know what your current machine can handle, what belongs in the browser, and when local tools are required.

## The Teacher Problem

Teachers often discover computer requirements only after the tool fails.

A tutorial assumes a local terminal. A coding agent needs repo access. A website project expects Node. A video tool eats storage. A Chromebook works for planning but not for the local build.

The requirements were hidden.

## The Idea

A builder workstation is the computer setup you use to create, inspect, preview, and safely revise curriculum projects.

It includes the device, operating system, browser, storage, accounts, local tools, update habits, backup plan, and permission boundaries.

## Classroom Scenario

A teacher wants to preview a course website locally. The directions say to install Node, run `npm install`, then run `npm run dev`.

On a locked-down school Chromebook, that workflow may not be possible. The teacher needs a browser-based alternative or a different machine for local development.

## Weak Version

```text
This tool is broken because it will not run on my computer.
```

Maybe. But maybe the computer is fine for browser work and not meant for local development.

## Better Version

```text
My device can handle browser editing and GitHub review, but local builds need a Windows, macOS, or Linux machine with install permissions and enough storage.
```

This separates device limits from teacher ability.

## Build Step

Start a Builder Workstation Checklist with these categories:

- device and operating system
- browser
- storage and RAM
- install permissions
- accounts
- local tools
- backup method
- blocked or school-managed restrictions

## Quality Check

A useful checklist helps you answer: what can I do on this machine, what do I need another tool for, and what should I not attempt here?

## Safety / Accessibility / Source Note

Do not install tools on a school-managed device without permission. A safe setup respects school policy, student data boundaries, and account ownership.

## Reflection

What is one thing your current computer can do well, and one thing you are not sure it can handle yet?
