## The Teacher Problem

Computer requirement pages often read like parts catalogs.

CPU. RAM. Storage. GPU. WSL. Architecture. Terminal. Admin rights.

Teachers do not need to become hardware specialists, but they do need enough vocabulary to understand why a tool is slow, blocked, or unavailable.

## The Idea

Use plain meanings first.

| Term | Plain meaning | Why a teacher cares |
|---|---|---|
| CPU | the main processor doing the work | affects general speed |
| RAM | short-term working memory | affects how many apps and tools can run at once |
| Storage | space for files, apps, and project dependencies | project folders and packages can get large |
| GPU | graphics processor | usually not required for basic course projects, but can matter for media or local AI |
| WSL | Windows Subsystem for Linux | lets Windows run many Linux-style developer tools |

## Classroom Scenario

A teacher has a laptop with 4 GB RAM and little storage. Browser tools work, but running a local website preview while VS Code and a browser are open feels painful.

The teacher is not failing. The machine is under pressure.

## Weak Version

```text
My computer is slow, so I cannot do this work.
```

Maybe some tasks are too heavy. But the teacher can still do browser planning, documentation, GitHub review, and lightweight editing.

## Better Version

```text
My current machine can handle browser workflows. Local builds may need more RAM, more storage, or a different machine.
```

This turns frustration into a setup decision.

## Build Step

Add a hardware row to your checklist:

```text
RAM:
Available storage:
Can install apps:
Can run VS Code:
Can run local project commands:
WSL needed? yes / no / not sure
```

If you do not know an answer, write `not sure` instead of guessing.

## Quality Check

The checklist should help you decide which workflows belong on this machine: browser-only, light editing, local preview, media work, or coding-agent project work.

## Safety / Accessibility / Source Note

A more powerful computer is not automatically safer. Safety comes from account boundaries, backups, permissions, and verification habits.

## Reflection

Which limit would affect you most right now: install permissions, storage, RAM, or uncertainty about the setup?

## Capstone Connection

Your Starter Kit should include an honest workstation note so future instructions do not assume tools your computer cannot run.
