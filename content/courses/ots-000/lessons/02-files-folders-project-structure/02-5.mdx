## The Teacher Problem

A folder map can become decorative.

It may show the structure but fail to tell the teacher what to do with that information.

The point is not to admire the tree. The point is to make safer project decisions.

## The Idea

This checkpoint asks whether your map shows source, output, configuration, assets, and edit boundaries clearly enough to prevent mistakes.

## Classroom Scenario

A teacher asks an AI agent to "clean up course files." The folder map lets the teacher add a boundary:

```text
Only inspect content/courses/ots-000/lessons. Do not edit generated reports, exports, config files, or unrelated courses.
```

That is a safer prompt because the teacher knows the map.

## Weak Version

```text
I drew the folder tree, so I understand the project.
```

Not yet.

## Better Version

```text
I can identify source content, generated output, config files, assets, and folders that require review before editing.
```

That is usable.

## Build Step

Review each row and add one of these labels:

- Teacher-editable
- Review before editing
- Generated or output
- Do not touch yet
- Unknown

For every unknown item, write one question.

## Quality Check

Your map should include at least one "do not touch yet" or "review before editing" item. If everything looks safe, the map is probably too optimistic.

## Safety / Accessibility / Source Note

Before giving a folder to an AI tool, check whether it contains secrets, private notes, student data, exports, or work from another course.

## Reflection

What is one folder you will now avoid editing until you know more?

## Capstone Connection

Save the revised map. Later, you will use it to choose safe boundaries for VS Code, GitHub, and AI-agent work.

## Checkpoint Decision Language

Use this checkpoint to make a plain decision about **Chapter Checkpoint**:

- **Ready** means the artifact can be used by another teacher without a side explanation from you.
- **Revise** means the idea is right, but the artifact still needs clearer steps, labels, evidence, or safer wording.
- **Blocked** means you cannot honestly finish the artifact yet because a tool, source, permission, example, or decision is missing.

## Evidence to Save

Save evidence that another teacher could inspect: a screenshot, copied command output, file path, short reflection note, or before-and-after excerpt. If the evidence is only in your memory, the checkpoint is not ready yet.

## Next Revision Trigger

Name the owner and trigger before you move on. Write who will revise it, when they will revisit it, and what would cause the next revision: a broken link, confusing student direction, missing source, unsafe permission, or a teammate review note.
