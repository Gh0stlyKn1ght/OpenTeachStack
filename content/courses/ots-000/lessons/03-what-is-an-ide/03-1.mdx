## The Teacher Problem

Teachers may edit course material in Google Docs, an LMS page, GitHub's browser editor, Notepad, VS Code, or an AI workspace.

Those tools are not interchangeable.

The wrong editor can make a simple task harder or riskier.

## The Idea

Use the editor that matches the job.

| Tool type | Plain meaning | Good for | Watch out for |
|---|---|---|---|
| Browser editor | editing inside a website | quick docs, LMS pages, GitHub edits | easy to lose project context |
| Text editor | simple app for plain files | small edits, notes, quick fixes | fewer project tools |
| IDE | project editor with built-in tools | navigating, searching, editing, terminal, source control | can feel overwhelming |

VS Code is often called a code editor, but for this course you can think of it as a project workbench.

## Classroom Scenario

A teacher needs to fix one typo in a README. GitHub's browser editor may be enough.

The same teacher needs to update six lesson files and search the whole project. VS Code is a better fit.

## Weak Version

```text
I should use the most advanced tool for every edit.
```

That creates unnecessary friction.

## Better Version

```text
I choose the simplest safe editor that gives me enough project context.
```

## Quality Check

If the task affects more than one file, needs search, or needs a terminal command, the browser editor may not be enough.

## Safety / Accessibility / Source Note

Avoid editing project files in tools that change formatting without warning. Some rich-text editors can damage Markdown, code blocks, or frontmatter.

## Reflection

What is one task you currently do in the browser that might be easier in a project editor?

## Capstone Connection

Your Starter Kit should explain which editor you will use for quick edits, project edits, and review-only work.
