## The Teacher Problem

An IDE can feel like too much until the teacher has a map.

Without a map, every panel looks equally important and every icon looks risky.

## The Idea

An IDE navigation diagram labels the small set of areas a teacher-builder needs first.

It should answer:

- Where do I find files?
- Where do I edit?
- Where do I search?
- Where do I run commands?
- Where do I review changes?
- Where do extensions live?

## Classroom Scenario

A teacher is helping another teacher open a course project. Instead of saying, "Click around in VS Code," they use the diagram to point to Explorer, Search, Terminal, and Source Control.

That turns the IDE into a teachable workspace.

## Weak Version

```text
VS Code = where code happens.
```

This is too vague and too intimidating.

## Better Version

| IDE area | Teacher use | Be careful when |
|---|---|---|
| Explorer | find files and folders | deleting, moving, or renaming |
| Search | locate text across the project | replacing every match |
| Terminal | run project commands | in the wrong folder |
| Source Control | review changed files | unrelated files changed |

## Build Step

Create a diagram or table with these areas:

1. Explorer
2. Editor
3. Tabs
4. Search
5. Terminal
6. Source Control
7. Extensions

For each area, add:

```text
What it does:
Teacher-builder use:
Safety reminder:
```

## Quality Check

Your diagram should be useful before someone opens VS Code. If it only makes sense after they already know the tool, revise it.

## Safety / Accessibility / Source Note

If you use a screenshot, do not include private file paths, student names, account emails, API keys, or private repo names.

## Reflection

Which IDE area needs the strongest safety reminder?

## Capstone Connection

This diagram becomes the IDE section of your Teacher Builder Starter Kit and prepares you for the VS Code chapter.
