## The Teacher Problem

An IDE diagram can become a labeled screenshot that does not actually help anyone work.

The checkpoint is simple: can the diagram guide a safe first task?

## The Idea

A useful IDE diagram connects area, action, and boundary.

It should not just say "Terminal." It should say when the teacher uses the terminal and what to check before running a command.

## Classroom Scenario

A teacher wants to search for a broken link across a project. The diagram should send them to Search, not Terminal, not Explorer folder-by-folder, and not an AI agent first.

## Weak Version

```text
I labeled all the panels.
```

Labels help, but they are not enough.

## Better Version

```text
I can use the diagram to decide where to search, where to edit, where to run a command, and where to review changes.
```

## Safety / Accessibility / Source Note

Do not run terminal commands or install extensions just to prove you can. The first goal is navigation and judgment.
