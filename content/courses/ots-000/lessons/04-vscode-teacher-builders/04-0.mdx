## The Teacher Problem

VS Code can look like a developer-only tool, so teachers either avoid it or click around without a plan.

Neither move helps.

## The Idea

In this course, VS Code is a project workbench. You use it to open the project folder, read files, search across the project, edit content, preview Markdown, run safe commands, and review what changed.

You are not trying to learn every feature. You are learning the few moves that protect your curriculum work.

## Classroom Scenario

A teacher needs to update a lesson page and verify that only one file changed. VS Code can help them open the project, find the lesson, edit it, and review the changed file before committing.

## Weak Version

```text
I opened VS Code and started clicking until I found something that looked right.
```

That is how people edit the wrong file.

## Better Version

```text
I opened the project root, used Explorer to find the lesson, used Search to confirm the title, edited the source file, and checked Source Control.
```

## Build Step

Start a VS Code survival sheet with these headings:

- open the right folder
- find a file
- search text
- edit safely
- preview Markdown
- use terminal carefully
- review changed files

## Quality Check

Your survival sheet should help you repeat a safe path, not memorize icons.

## Safety / Accessibility / Source Note

If VS Code asks to trust a workspace or install an extension, pause before accepting. Workspace trust and extensions affect what the editor can run or access.

## Reflection

Which VS Code action do you most need first: open, search, edit, preview, run, or review?

## Capstone Connection

The VS Code survival sheet becomes the editor-use section of your Teacher Builder Starter Kit.
