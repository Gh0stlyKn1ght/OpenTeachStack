## The Teacher Problem

Opening one file is not the same as opening the project.

If the teacher opens only a lesson file, search, terminal commands, and source control may not have the full project context.

## The Idea

VS Code works best when you open the project root: the top-level folder that contains the README, content folders, package file, and project settings.

## Classroom Scenario

A teacher opens `02-4.mdx` directly and edits it. Then they cannot find the other lesson files or run the preview command because VS Code is not pointed at the whole project.

## Weak Version

```text
I opened the file I care about, so I opened the project.
```

Not always.

## Better Version

```text
I opened the project folder first, then opened the file from Explorer.
```

## Build Step

Add this to your survival sheet:

```text
Project folder I should open:
How I know it is the project root:
Where the README is:
Where course content lives:
```

If you are using Windows and the `code .` command is available, write what folder you must be in before running it.

## Quality Check

You should be able to point to the project root without guessing.

## Safety / Accessibility / Source Note

Only install VS Code from the official Visual Studio Code site or an approved software portal. On school-managed devices, follow your organization’s install rules.
