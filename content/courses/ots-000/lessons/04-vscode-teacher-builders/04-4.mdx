## The Teacher Problem

Teachers do not need a full VS Code manual. They need a short survival sheet they can use when they are tired and trying not to break the project.

## The Idea

The survival sheet is a practical checklist for your first safe VS Code moves.

It should cover:

- opening the project root
- finding files
- searching text
- editing Markdown or MDX
- previewing
- using terminal carefully
- reviewing changed files
- choosing extensions

## Classroom Scenario

A teacher is asked to update a course lesson. The survival sheet keeps the workflow small:

1. Open the project root.
2. Find the lesson folder.
3. Search the title.
4. Edit the lesson body.
5. Preview.
6. Review changed files.

## Weak Version

```text
VS Code notes:
- Explorer
- Search
- Terminal
```

This names features but does not guide action.

## Better Version

```text
Before editing:
- Open the project root, not just one file.
- Confirm the file is source content.
- Search before changing repeated text.
- Review changed files before saving the work as done.
```

## Build Step

Create a one-page sheet with these sections:

```text
Open:
Find:
Search:
Edit:
Preview:
Run:
Review:
Do not:
```

Write one practical instruction under each heading.

## Quality Check

The sheet should help another teacher complete one small edit without you explaining every step live.

## Safety / Accessibility / Source Note

Add a clear warning: do not paste secrets, student data, private account details, or API keys into VS Code, extensions, terminal commands, or AI prompts.

## Reflection

Which heading on your survival sheet will you probably use most often?

## Capstone Connection

This survival sheet becomes the VS Code section of your Teacher Builder Starter Kit.
