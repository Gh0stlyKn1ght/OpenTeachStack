## The Teacher Problem

A survival sheet can look helpful until the first real task.

If it does not tell the teacher what to check before editing, it is not ready yet.

## The Idea

This checkpoint asks whether your VS Code sheet supports one safe first edit.

The point is not confidence theater. The point is a repeatable path.

## Classroom Scenario

A teacher wants to change one sentence in a Markdown lesson. The survival sheet should guide them to open the project, find the file, edit the body, preview, and review changes.

## Weak Version

```text
I know where the buttons are.
```

Buttons are not a workflow.

## Better Version

```text
I can use the sheet to make one small content edit and check what changed.
```

## Build Step

Run a paper rehearsal.

Pick one task:

- find a lesson
- search for a phrase
- edit a Markdown paragraph
- preview a Markdown file
- review changed files

Write the exact steps you would take.

## Quality Check

Your steps should include one pause before editing and one review after editing.

## Safety / Accessibility / Source Note

Do not practice on private student material. Use sample files or teacher-created practice content.

## Reflection

What would you still want beside you the first time you use VS Code on a real project?

## Capstone Connection

Save the revised survival sheet. It will support the terminal, GitHub, and AI-agent chapters later.

## Checkpoint Decision Language

Use this checkpoint to make a plain decision about **Chapter Checkpoint**:

- **Ready** means the artifact can be used by another teacher without a side explanation from you.
- **Revise** means the idea is right, but the artifact still needs clearer steps, labels, evidence, or safer wording.
- **Blocked** means you cannot honestly finish the artifact yet because a tool, source, permission, example, or decision is missing.

## Evidence to Save

Save evidence that another teacher could inspect: a screenshot, copied command output, file path, short reflection note, or before-and-after excerpt. If the evidence is only in your memory, the checkpoint is not ready yet.

## Next Revision Trigger

Name the owner and trigger before you move on. Write who will revise it, when they will revisit it, and what would cause the next revision: a broken link, confusing student direction, missing source, unsafe permission, or a teammate review note.
