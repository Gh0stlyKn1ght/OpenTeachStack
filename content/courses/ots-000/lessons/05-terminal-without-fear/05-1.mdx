## The Teacher Problem

The terminal has a reputation problem.

People treat it like a secret professional space instead of explaining what it does.

## The Idea

The terminal is a text-based way to tell the computer or project tools what to do.

Useful words:

| Word | Plain meaning |
|---|---|
| Terminal | the window where commands are typed |
| Shell | the program interpreting your commands |
| Prompt | the line waiting for your command |
| Command | the instruction you type |
| Current directory | the folder the terminal is in right now |

## Classroom Scenario

A teacher uses the terminal to start a local preview server. The browser shows the course site, but the terminal is what started the process.

## Weak Version

```text
The terminal is for coding.
```

Too vague.

## Better Version

```text
The terminal runs project commands, and I need to know the current folder before I use it.
```

## Build Step

Add these terms to your command cheat sheet:

- terminal
- shell
- prompt
- command
- current directory

Write each in your own plain language.

## Quality Check

Your definitions should help you explain what is happening to another teacher, not impress a developer.

## Safety / Accessibility / Source Note

Some terminal commands only read information. Some change files. Some install packages. Some start servers. Some delete things. Treat the effect as the main question.

## Reflection

Which terminal word should you learn before any command: prompt, command, or current directory?

## Capstone Connection

Your Starter Kit should include enough terminal vocabulary to help you ask safer questions.
