## The Teacher Problem

Commands often depend on the folder where they are run.

Beginners copy the command but miss the location.

## The Idea

Before running a command, ask:

1. What folder am I in?
2. What command am I running?
3. What should it do?
4. What could it change?

The prompt often shows part of the current folder, but you can also use a command to check.

## Classroom Scenario

A teacher tries to run `npm run dev` from the desktop. The project is in a different folder, so the command fails.

The error does not mean the teacher is bad at this. It means the terminal is in the wrong place.

## Weak Version

```text
The command failed, so the project is broken.
```

Maybe not.

## Better Version

```text
The command failed. First I will check whether I am in the project folder.
```

## Build Step

Add this pre-command checklist:

```text
Before I press Enter:
- I know the current folder.
- I know what the command is supposed to do.
- I know whether it reads, installs, changes, starts, or deletes.
- I know how to stop if it hangs or looks wrong.
```

## Quality Check

If you cannot fill in one bullet, pause and ask before running the command.

## Safety / Accessibility / Source Note

Be extra careful with commands that include words like `delete`, `remove`, `rm`, `clean`, `force`, `reset`, or `overwrite`.
