## The Teacher Problem

Terminal tutorials jump from navigation to project commands without explaining the difference.

Teachers need to know which commands help them move around and which commands make the project do something.

## The Idea

Start small.

| Command | Plain meaning | Risk level |
|---|---|---|
| `pwd` | show current folder | low |
| `ls` | list files and folders | low |
| `cd folder-name` | move into a folder | low |
| `cd ..` | move up one folder | low |
| `mkdir name` | make a folder | changes files |
| `npm run dev` | run the project's dev script | starts a process |
| `npm run build` | run the project's build script | creates output and checks build |

Navigation commands help you get oriented. Project commands ask the project to do work.

## Classroom Scenario

A teacher uses `pwd` and `ls` to confirm they are in the project root before running `npm run dev`.

That small habit prevents a lot of confusion.

## Weak Version

```text
All terminal commands are equally risky.
```

No. Reading the current folder is different from deleting a directory.

## Better Version

```text
I can start with low-risk navigation commands and pause before commands that change files or run project scripts.
```

## Build Step

Add a command table:

```text
Command | Meaning | Reads or changes? | When I would use it
```

Include `pwd`, `ls`, `cd`, `mkdir`, and one project command from a README.

## Quality Check

Each command needs a purpose. Do not collect commands like trading cards.

## Safety / Accessibility / Source Note

Windows PowerShell, macOS Terminal, Linux shells, and Git Bash may use slightly different commands or output. Do not panic if screenshots differ from your machine.
