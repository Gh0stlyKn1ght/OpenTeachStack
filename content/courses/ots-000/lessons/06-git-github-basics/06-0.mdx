## The Teacher Problem

GitHub looks like a coding site, so teachers may assume it has nothing to do with curriculum.

But modern curriculum projects often use GitHub to store files, track history, publish sites, review changes, and coordinate work.

## The Idea

Git and GitHub protect projects by keeping history inspectable.

You do not need to become a Git expert in OTS-000. You need enough literacy to read a repo, understand commits and branches, and avoid letting AI tools change work without review.

## Classroom Scenario

A teacher asks an AI coding agent to revise a lesson page. Git history shows exactly which files changed. Without that history, the teacher has to trust memory.

## Weak Version

```text
GitHub is for programmers, so I do not need to understand it.
```

That leaves teacher-owned content dependent on other people.

## Better Version

```text
GitHub can hold curriculum projects, and version history helps me review changes before students see them.
```

## Build Step

Start a GitHub project-reading checklist with these headings:

- repo purpose
- README
- folders
- latest commit
- branches
- issues or pull requests
- license
- safety notes

## Quality Check

The checklist should help you enter a repo without immediately editing anything.

## Safety / Accessibility / Source Note

Public repos are public. Do not put student data, private school documents, secrets, or unpublished sensitive materials into a public repository.

## Reflection

What would version history help you recover from: accidental edits, stale files, unclear authorship, or AI changes?

## Capstone Connection

The GitHub checklist becomes the version-control section of your Teacher Builder Starter Kit.
