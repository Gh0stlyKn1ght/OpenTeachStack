## The Teacher Problem

GitHub repos can look like file dumps.

Without a reading checklist, teachers may skip the project context and jump straight into editing.

## The Idea

A GitHub project-reading checklist helps you understand a repo before you touch it.

It should identify purpose, source folders, setup notes, history, review process, and safety boundaries.

## Classroom Scenario

A teacher finds a public curriculum repo. The checklist helps them ask:

- What is this project?
- Can I reuse it?
- Is it current?
- Where is the source content?
- What should I not copy or publish?

## Weak Version

```text
I found a repo with useful files.
```

Useful for what? Safe to reuse how? Current according to whom?

## Better Version

```text
I read the README, checked the license, found the source folder, looked at recent commits, and wrote down what I can safely use.
```

## Build Step

Create your checklist:

```text
Repository name:
Purpose:
README summary:
Main source folders:
Docs or setup notes:
Recent activity:
License or reuse note:
Open issues or warnings:
Safe next action:
Do not touch or copy:
```

## Quality Check

The checklist should end with a decision, not just notes.

Use one of these:

- safe to read only
- safe to copy with attribution
- safe to fork or clone
- needs permission
- do not use yet

## Safety / Accessibility / Source Note

Check license and source notes before reusing project material. Public does not always mean free to republish however you want.

## Reflection

What question would stop you from reusing a repo until you had a better answer?

## Artifact Build Scan

### Artifact

Build or revise **Build Task: GitHub Project-Reading Checklist** as a small working artifact, not a decorative note.

### Build Steps

1. Name the classroom job this artifact is supposed to do.
2. Fill in the artifact with the smallest useful version a teacher could reuse.
3. Add one piece of evidence that shows the artifact is usable.
4. Mark anything unfinished as revise or blocked instead of hiding it.

### Example Output

A finished example should be specific enough that another teacher can tell what changed, where it lives, and how it would be used in a real course build.

### Quality Check

- The artifact has a clear title.
- The teacher decision or classroom use is visible.
- The evidence is saved in a form someone else can inspect.
- The next revision trigger is named if the artifact is not ready.

### Capstone Connection

Save this artifact in the Teacher Builder Starter Kit. It becomes part of the evidence trail for deciding whether the starter kit is ready, needs revision, or is blocked.
