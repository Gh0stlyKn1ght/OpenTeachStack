## The Teacher Problem

A GitHub checklist can become another form to fill out instead of a decision tool.

This checkpoint makes it practical.

## The Idea

The checklist should help you decide what to do next with a repo: read, clone, fork, copy with attribution, ask permission, or leave it alone.

## Classroom Scenario

A teacher finds a repo with lesson files and no license. The checklist should prevent copying the material directly into a course until reuse rights are clearer.

## Weak Version

```text
It is on GitHub, so I can use it.
```

No.

## Better Version

```text
It is on GitHub, so I can inspect it. Reuse depends on license, source notes, and context.
```

## Build Step

Review your checklist with one repo or sample project.

Write one of these decisions:

- read only
- use as inspiration
- reuse with attribution
- fork or clone for practice
- ask permission first
- do not use yet

## Quality Check

Your decision should cite evidence from the README, license, source notes, commits, or repo structure.

## Safety / Accessibility / Source Note

Do not treat GitHub as automatically safe, current, accurate, accessible, or classroom-ready. It is a place where work lives, not a guarantee of quality.
