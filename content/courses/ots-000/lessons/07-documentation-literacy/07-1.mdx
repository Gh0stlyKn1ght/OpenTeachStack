## The Teacher Problem

Not all help sources have the same job.

A tutorial may be clear but outdated. A blog post may solve one case but skip warnings. An AI summary may be readable but unverified.

## The Idea

Use the source type on purpose.

| Source type | Good for | Watch out for |
|---|---|---|
| Official docs | install, setup, current behavior, warnings | can be dense |
| Tutorial | step-by-step learning | may be old or opinionated |
| Blog post | one person's solution | may not match your setup |
| Forum answer | troubleshooting clues | may be incomplete |
| AI summary | plain-language orientation | must be verified |

## Classroom Scenario

A teacher wants to install VS Code. The official install page is the right source. A random video might help visually, but the install link should come from the official site or approved software portal.

## Weak Version

```text
The clearest explanation is automatically the best source.
```

Clear can still be wrong.

## Better Version

```text
I can use a clear explanation to understand, but I verify setup steps and commands against official docs.
```

## Build Step

Add a source type row to your checklist:

```text
Source type:
Why I am using it:
Risk:
Official source checked? yes / no / not needed
```

Use it on one tool you expect to install or learn.

## Quality Check

If the source tells you to install, connect accounts, paste commands, or grant permissions, official verification is required.

## Safety / Accessibility / Source Note

Do not use AI-generated links, commands, citations, or policy claims without checking them. AI can be helpful and still be wrong.
