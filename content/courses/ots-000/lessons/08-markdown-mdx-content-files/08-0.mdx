## The Teacher Problem

Teachers may open a course file and see plain text mixed with symbols, metadata, links, and components.

It looks like code, but much of it is structured writing.

## The Idea

Markdown is a plain-text writing format. MDX is Markdown that can also use components.

Curriculum projects use these formats because they are readable, versionable, and easier to publish into websites than random copy-paste content.

## Classroom Scenario

A teacher wants to write a student-facing lesson that can live in a website, be reviewed in GitHub, and stay easy to revise.

Markdown gives the teacher headings, lists, links, tables, and code blocks without a heavy design tool.

## Weak Version

```text
This file has symbols, so it must be code I should not edit.
```

Some parts require care, but the lesson body may be teacher-editable.

## Better Version

```text
This is structured content. I can edit the lesson body and pause before changing metadata or components.
```

## Build Step

Start a starter Markdown lesson with these headings:

```md
# Lesson Title

## Learning Target

## Why This Matters

## Directions

## Example

## Before You Submit
```

## Quality Check

The file should still be readable if someone opens it as plain text.

## Safety / Accessibility / Source Note

Markdown does not automatically make content accessible. Headings, link text, image descriptions, and table simplicity still matter.

## Reflection
