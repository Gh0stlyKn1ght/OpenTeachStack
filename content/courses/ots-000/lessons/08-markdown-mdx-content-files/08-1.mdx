## The Teacher Problem

Teachers are used to rich editors where formatting is applied by buttons.

Markdown uses visible text symbols instead. That can feel strange at first.

## The Idea

Markdown is plain text with simple marks for structure.

Examples:

```md
# Big heading

## Smaller heading

- bullet item

[link text](https://example.com)
```

The symbols are not decoration. They tell the publishing system what the content is.

## Classroom Scenario

A teacher writes lesson directions in Markdown. Another teacher can review the exact words in GitHub, suggest changes, and keep the history.

## Weak Version

```text
Markdown is a worse version of Google Docs.
```

That misses the point.

## Better Version

```text
Markdown is useful when the content needs to stay portable, reviewable, and publishable.
```

## Build Step

In your starter lesson, write:

- one `#` title
- two `##` headings
- one bullet list
- one link with clear link text

## Quality Check

The Markdown should be understandable before it is rendered. If it only works after styling, it is probably too fragile.

## Safety / Accessibility / Source Note

Use descriptive link text. `Click here` does not tell students or screen readers where the link goes.

## Reflection

What is one classroom document that would benefit from being easier to review as plain text?

## Capstone Connection

Your Starter Kit should include one short Markdown file you can safely edit and explain.
