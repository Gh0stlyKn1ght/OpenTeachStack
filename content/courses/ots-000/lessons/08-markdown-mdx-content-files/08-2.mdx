## The Teacher Problem

Teachers may use formatting to make a page look better, but formatting also teaches students how to move through the page.

Bad structure makes students work harder before they can learn.

## The Idea

Use Markdown structure for a teaching reason.

| Structure | Use it for | Watch out for |
|---|---|---|
| Headings | sections and signposts | skipping levels randomly |
| Links | sources, tools, pages | vague link text |
| Images | visuals that support learning | missing alt text or context |
| Tables | small comparisons or checklists | huge tables on phones |
| Code blocks | commands, examples, exact text | unclosed fences |

## Classroom Scenario

A teacher writes directions as one long paragraph. Students miss the submission step.

Breaking the page into headings and bullets makes the task easier to follow.

## Weak Version

```md
Read the article then answer questions then submit your response then check the rubric.
```

This hides the steps.

## Better Version

```md
## Directions

1. Read the article.
2. Answer the three questions.
3. Check the rubric.
4. Submit your response.
```

## Build Step

Add these to your starter lesson:

- a directions list
- one descriptive link
- one small table or checklist
- one code block or exact-text example

## Quality Check

Ask: does the structure make the student action easier to see?

## Safety / Accessibility / Source Note

If you include an image, add enough context for students who cannot see it or who need the purpose explained.

## Reflection

Where does your starter lesson need structure, not more words?

## Capstone Connection

Your Starter Kit should include a Markdown sample that demonstrates clear student-facing structure.

## Framework Purpose

This framework exists to keep **Headings, Links, Images, Tables, and Code Blocks** from turning into a pile of tips. A teacher should be able to use it when the work feels fuzzy: name the parts, make the next move smaller, and check whether the result is actually useful in a classroom.

## How to Use This Framework

Apply the framework in three passes:

1. Name the classroom job this tool or idea is supposed to do.
2. Match each part of the framework to one visible choice in your project.
3. Review the artifact for the obvious failure mode: polished wording with no evidence, unclear steps, missing source, or a tool choice that creates extra work for students or teachers.

Save the result in your Starter Kit as evidence of the decision, not as decoration.
