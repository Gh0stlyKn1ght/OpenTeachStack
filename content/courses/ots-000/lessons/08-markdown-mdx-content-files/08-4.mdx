## The Teacher Problem

It is easy to learn Markdown as disconnected syntax.

Teachers need to use it for a real teaching page.

## The Idea

Build a small lesson that a student could actually follow.

Keep it short. The goal is structure and clarity, not a huge unit.

## Classroom Scenario

A teacher creates a five-minute mini lesson on evaluating a source. The Markdown file includes a learning target, directions, example, checklist, and submission note.

## Weak Version

```md
# Source Lesson

Read this and answer questions.
```

This is too thin.

## Better Version

```md
# Evaluating a Source

## Learning Target
I can identify who made a source and why it might be useful.

## Directions
1. Open the source.
2. Find the creator.
3. Write one reason the source may or may not be trustworthy.

## Before You Submit
- I named the creator.
- I explained my reason.
- I used my own words.
```

## Build Step

Create your starter Markdown lesson with:

- title
- learning target
- why this matters
- materials or links
- directions
- example
- before-you-submit checklist
- safety or source note

## Quality Check

Ask: could a student begin without you explaining the page live?

## Safety / Accessibility / Source Note

Use fake or public-safe examples. Do not use real student work, private screenshots, or copyrighted material without a clear reuse reason.

## Reflection

Which heading most improved the lesson's usability?

### Artifact

### Build Steps

1. Name the classroom job this artifact is supposed to do.
2. Fill in the artifact with the smallest useful version a teacher could reuse.
3. Add one piece of evidence that shows the artifact is usable.
4. Mark anything unfinished as revise or blocked instead of hiding it.

### Example Output

A finished example should be specific enough that another teacher can tell what changed, where it lives, and how it would be used in a real course build.

### Quality Check

- The artifact has a clear title.
- The teacher decision or classroom use is visible.
- The evidence is saved in a form someone else can inspect.
- The next revision trigger is named if the artifact is not ready.
