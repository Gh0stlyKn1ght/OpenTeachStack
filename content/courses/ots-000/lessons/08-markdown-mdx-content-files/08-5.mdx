## The Teacher Problem

A Markdown lesson can render nicely and still be weak instruction.

Structure helps, but the content still needs a real learning target, directions, example, and check.

## The Idea

This checkpoint asks whether the starter lesson is usable, not just valid Markdown.

## Classroom Scenario

A teacher previews a Markdown lesson and sees headings and bullets. The page looks clean, but the student task does not say what to submit.

The formatting passed. The lesson did not.

## Weak Version

```text
The Markdown preview looks good, so the lesson is ready.
```

Preview is not the same as instructional review.

## Better Version

```text
The page renders, and I checked that the student knows what to learn, do, and submit.
```

## Build Step

Review your starter lesson:

- target is clear
- directions are numbered or chunked
- example is included
- links are descriptive
- submission expectation is visible
- source or safety note is present

Revise one weak area.

## Quality Check

Ask the tired-teacher question: could this page stand on its own for a student who missed the verbal explanation?

## Safety / Accessibility / Source Note

Check headings, link text, image context, and table size. Markdown structure should help students, not just satisfy a renderer.

## Reflection

What did the Markdown format make easier to see?

## Capstone Connection

Save the revised starter lesson. Later it can become the first file you safely edit in VS Code or GitHub.

## Checkpoint Decision Language

Use this checkpoint to make a plain decision about **Chapter Checkpoint**:

- **Ready** means the artifact can be used by another teacher without a side explanation from you.
- **Revise** means the idea is right, but the artifact still needs clearer steps, labels, evidence, or safer wording.
- **Blocked** means you cannot honestly finish the artifact yet because a tool, source, permission, example, or decision is missing.

## Evidence to Save

Save evidence that another teacher could inspect: a screenshot, copied command output, file path, short reflection note, or before-and-after excerpt. If the evidence is only in your memory, the checkpoint is not ready yet.

## Next Revision Trigger

Name the owner and trigger before you move on. Write who will revise it, when they will revisit it, and what would cause the next revision: a broken link, confusing student direction, missing source, unsafe permission, or a teammate review note.
