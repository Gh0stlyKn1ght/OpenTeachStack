## The Teacher Problem

A site can work on one computer and still fail online.

That feels mysterious until the teacher separates local preview, build, hosting, domain, and deployment.

## The Idea

Publishing is a path.

Content starts as files. A project turns those files into pages. A host serves the pages. A domain may point people to the site.

Each step can work or fail separately.

## Classroom Scenario

A teacher updates a course page locally and sees it in the browser. Students still see the old page online because the site has not deployed yet.

The edit worked locally. Publishing did not happen yet.

## Weak Version

```text
It works on my computer, so it is published.
```

Local is not public.

## Better Version

```text
It works locally. Now I need to check build, deployment, and the published URL.
```

## Build Step

Start a website publishing flow diagram:

```text
Source files -> local preview -> build -> host -> domain or URL -> student view
```

Add one note under each step.

## Quality Check

Your diagram should show where a teacher checks the page before students receive it.

## Safety / Accessibility / Source Note

Publishing multiplies mistakes. Check privacy, links, accessibility, and source rights before a page becomes public.

## Reflection

Which step do you currently understand least: local preview, build, hosting, domain, deployment, or student view?
