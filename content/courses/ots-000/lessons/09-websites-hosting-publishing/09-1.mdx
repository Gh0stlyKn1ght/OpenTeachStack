## The Teacher Problem

Teachers may think of a website as one thing: "the site."

But when something breaks, that single word is not specific enough.

## The Idea

A website is a set of pages, routes, assets, and code or generated files served by a host.

Plain pieces:

- page: what someone reads or uses
- route: the URL path to the page
- asset: image, file, video, or download
- source file: where the page content is maintained
- host: the service that serves the site
- domain: the human-friendly address

## Classroom Scenario

A teacher says, "The website is broken." The actual issue is one image path on one route.

Better vocabulary makes the fix smaller.

## Weak Version

```text
The whole site is broken.
```

Maybe. But maybe one link, route, image, or page is broken.

## Better Version

```text
The page loads, but the image asset is missing on the student resources route.
```

## Quality Check

Your explanation should help someone narrow a problem instead of saying "the site is broken."

## Safety / Accessibility / Source Note

Images, downloads, and embedded media can create privacy or copyright problems even when the page text is fine.

## Reflection

When you say "website," which piece do you usually mean: the page, the URL, the content, or the host?
