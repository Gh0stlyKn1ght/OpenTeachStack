## The Teacher Problem

Publishing failures sound scarier when the vocabulary is missing.

"The domain is broken" might actually mean DNS is not pointing correctly, the deployment failed, or the page route does not exist.

## The Idea

Use plain terms.

| Term | Plain meaning |
|---|---|
| Host | service that serves the site |
| Domain | human-friendly name like `example.org` |
| DNS | directions that point the domain to the right service |
| Build | process that turns project files into a site or app output |
| Deployment | sending the built site/app to the host |
| Build failure | project could not produce a valid output |

## Classroom Scenario

A teacher updates content, but the deployment fails. The old site remains online. The issue is not the domain. The build failed before the host received the new version.

## Weak Version

```text
The website did not update, so the domain is broken.
```

Maybe not.

## Better Version

```text
The local edit exists, but the deployment failed during build, so the published site stayed on the old version.
```

## Quality Check

Your notes should help you describe where the failure happened before asking for help.

## Safety / Accessibility / Source Note

A successful deployment is not automatically a safe release. Run student-view, privacy, link, and accessibility checks before announcing content.
