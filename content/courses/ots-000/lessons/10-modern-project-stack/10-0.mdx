## The Teacher Problem

A teacher opens a project and sees names like Node, npm, React, Next.js, TypeScript, Tailwind, MDX, Vite, Vercel, and GitHub Actions.

The project can feel like ten courses at once.

## The Idea

A tech stack is the set of tools and frameworks a project uses.

In OTS-000, you do not need to master the stack. You need to recognize what each piece roughly does so you can ask better questions and avoid touching the wrong layer.

## Classroom Scenario

A course site has MDX lesson files, React components, TypeScript route files, Tailwind styling, and Vercel deployment. The teacher mostly edits content, but they still need to know which pieces affect the page.

## Weak Version

```text
I do not know React, so I cannot understand this project at all.
```

That is too absolute.

## Better Version

```text
I do not need to write React yet, but I can identify which files are content, which are interface code, and which tools build the site.
```

## Build Step

Start a project stack card:

```text
Project:
Content format:
Frontend framework:
Styling system:
Package manager:
Host:
Checks or automation:
What I am safe to edit:
What needs review:
```

## Quality Check

The card should help you explain the project to another teacher in under two minutes.

## Safety / Accessibility / Source Note

Stack vocabulary is not permission to edit everything. Mark boundaries clearly before using AI help or changing project files.
