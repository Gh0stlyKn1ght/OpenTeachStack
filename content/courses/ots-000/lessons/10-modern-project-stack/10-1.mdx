## The Teacher Problem

People say "stack" as if the meaning is obvious.

For teachers, it can sound like a pile of software names with no classroom connection.

## The Idea

A tech stack is the collection of tools that make a project work.

Think in layers:

| Layer | Job |
|---|---|
| Content | lessons, docs, Markdown, MDX, data |
| Interface | pages, components, navigation, visuals |
| Runtime/tools | Node, package manager, scripts |
| Styling | CSS, Tailwind, design system |
| Hosting | Vercel, Netlify, GitHub Pages, server |
| Automation | tests, checks, GitHub Actions |

## Classroom Scenario

A teacher wants to revise lesson wording. That is the content layer. The teacher does not need to redesign the navigation component unless the page structure itself is wrong.

## Weak Version

```text
The stack is all technical, so it belongs to developers.
```

The teacher still owns the course content and review decisions.

## Better Version

```text
The stack has layers. I can work in the content layer and ask for help with framework or deployment layers.
```

## Build Step


## Quality Check

Each layer should have a job. If a tool name does not have a job, leave it off the card until you know more.

## Safety / Accessibility / Source Note

The content layer can still carry privacy, copyright, and accessibility risks. Do not treat it as safe just because it is not code.
