## The Teacher Problem

A project stack can stay invisible until something breaks.

Then the teacher has to learn the stack under pressure.

## The Idea

A stack card is a short explanation of what a project uses and what each piece does.

It is not a full technical manual. It is an orientation card.

## Classroom Scenario

A teacher opens a course-site repo and writes:

| Layer | Tool | Job | Edit boundary |
|---|---|---|---|
| Content | MDX | lesson files | teacher-editable with review |
| Framework | Next.js | routes and rendering | technical review |
| Styling | Tailwind | page styling | review for accessibility |
| Hosting | Vercel | published site | check deployments |

Now the project is less mysterious.

## Weak Version

```text
This project uses modern web stuff.
```

That does not guide action.

## Better Version

```text
This project uses MDX for content, Next.js for the site, Tailwind for styling, npm scripts for checks, and Vercel for hosting.
```

## Build Step

Create your stack card:

```text
Project name:
What it is for:
Content format:
Main folders:
Framework:
Styling:
Commands:
Host:
Checks:
Teacher-editable:
Review before editing:
Do not touch yet:
```

## Quality Check

The card should help someone decide what they can edit safely and what needs review.

## Safety / Accessibility / Source Note

Include a warning for secrets, `.env` files, student data, private screenshots, and deployment settings.

## Artifact Build Scan

### Artifact

Build or revise **Build Task: What This Project Uses Stack Card** as a small working artifact, not a decorative note.

### Build Steps

1. Name the classroom job this artifact is supposed to do.
2. Fill in the artifact with the smallest useful version a teacher could reuse.
3. Add one piece of evidence that shows the artifact is usable.
4. Mark anything unfinished as revise or blocked instead of hiding it.

### Example Output

A finished example should be specific enough that another teacher can tell what changed, where it lives, and how it would be used in a real course build.

### Quality Check

- The artifact has a clear title.
- The teacher decision or classroom use is visible.
- The evidence is saved in a form someone else can inspect.
- The next revision trigger is named if the artifact is not ready.

### Capstone Connection

Save this artifact in the Teacher Builder Starter Kit. It becomes part of the evidence trail for deciding whether the starter kit is ready, needs revision, or is blocked.
