## The Teacher Problem

AI tools get lumped together.

ChatGPT, Claude, Gemini, Copilot, Cursor, Codex, Claude Code, Gemini CLI, and MCP-connected tools do not all have the same access or risk.

## The Idea

Start by asking what the tool can see and change.

Some tools only respond in chat. Some suggest code. Some read files. Some edit files. Some run commands. Some connect to external systems through tools like MCP.

## Classroom Scenario

A teacher asks an AI chat to explain Markdown. Low risk.

The same teacher asks a coding agent to edit a repo and run commands. Higher risk. The workflow needs Git, diffs, scope, and verification.

## Weak Version

```text
AI is AI. The tool name does not matter.
```

Tool access matters a lot.

## Better Version

```text
Before I use an AI tool, I will name what it can read, edit, run, and connect to.
```

## Build Step

Start an AI tool comparison and MCP access map:

```text
Tool:
Can read files?
Can edit files?
Can run commands?
Can access external services?
Needs review before:
```

## Quality Check

The map should make tool access visible before you ask for help.

## Safety / Accessibility / Source Note

Never paste secrets, student data, private class details, or unpublished sensitive content into AI tools unless the privacy and access boundaries are approved.

## Reflection

Which AI tool do you currently use without knowing exactly what it can access?
