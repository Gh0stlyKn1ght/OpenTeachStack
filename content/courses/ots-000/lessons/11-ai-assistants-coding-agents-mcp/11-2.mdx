## The Teacher Problem

Tool comparison can turn into fanboy nonsense fast.

Teachers need practical differences, not brand loyalty.

## The Idea

Compare tools by workflow, access, review, and fit.

| Tool type | Example | Teacher question |
|---|---|---|
| Repo agent | Codex | What files can it edit and what checks will prove the work? |
| Terminal agent | Claude Code, Gemini CLI | What commands can it run and where? |
| IDE assistant | Cursor, Copilot, Gemini Code Assist | What suggestions can enter my files? |
| Chat assistant | ChatGPT, Claude, Gemini web | What context did I provide and what must I verify? |

## Classroom Scenario

A teacher asks Codex to audit stale course titles. That is a repo-aware task. It needs exact scope and a check script, not just a nice prompt.

## Weak Version

```text
This AI tool is smart, so it can decide the workflow.
```

No. The teacher defines the instructional and safety boundaries.

## Better Version

```text
I choose the AI tool based on task, access, review path, and risk.
```

## Quality Check

Each row should include a "do not use for" boundary. Tools without boundaries become trouble.

## Safety / Accessibility / Source Note

Do not connect AI tools to private accounts, student data, or production systems just to experiment.

## Reflection

Which tool would you use for explanation only, and which would you reserve for repo work?
