## The Teacher Problem

Teachers are often handed AI tool names without a decision framework.

That makes it easy to over-trust a tool or avoid it completely.

## The Idea

The map should answer two questions:

1. What is this AI tool good for?
2. What can it access?

## Classroom Scenario

A teacher writes:

| Tool | Best use | Can edit files? | Can run commands? | Review needed |
|---|---|---|---|---|
| ChatGPT web | explanation and drafting | no | no | fact/source check |
| Codex | repo edits and audits | yes | sometimes | diff and tests |
| Claude Code | terminal agent workflow | yes | yes | scope and approvals |

Now the tool choice is less mysterious.

## Weak Version

```text
Use the strongest AI for everything.
```

That adds risk without purpose.

## Better Version

```text
Use the lowest-risk AI tool that can do the job, then verify the output.
```

## Build Step

Create two tables.

Table 1:

```text
Tool | Best use | Access | Review needed | Do not use for
```

Table 2:

```text
Connection | Reads | Changes | Approval needed | Private boundary
```

## Quality Check

Every row should include a review step and a boundary.

## Safety / Accessibility / Source Note

If a tool can edit files or run commands, it belongs behind Git, diffs, backups, and explicit scope.

## Reflection

Which AI tool should stay in explanation mode until you know more?
