## The Teacher Problem

An AI tool map can become a comparison chart with no safety value.

The checkpoint is whether it changes what you would allow the tool to do.

## The Idea

A good map makes access visible and review required.

## Classroom Scenario

A teacher marks "chat explanation" as low access and "repo edit with commands" as high access. That teacher is ready to choose different safety steps for different AI tasks.

## Weak Version

```text
I listed the tools I know.
```

That does not show risk.

## Better Version

```text
I can name what each tool can read, edit, run, or connect to before I use it.
```

## Build Step

Review your map and add a risk label:

- Low: explanation or drafting only
- Medium: suggestions inside a file
- High: repo edits, commands, external connections
- Not approved: unknown access or private data risk

## Quality Check

Every high-risk row must include a review step.

## Safety / Accessibility / Source Note

If student data, private accounts, secrets, or production publishing are involved, the default answer is pause.

## Reflection

What AI task should you refuse until you have a clearer access boundary?

## Capstone Connection

Save the revised map. It will become a safety reference before OTS-320.

## Checkpoint Decision Language

Use this checkpoint to make a plain decision about **Chapter Checkpoint**:

- **Ready** means the artifact can be used by another teacher without a side explanation from you.
- **Revise** means the idea is right, but the artifact still needs clearer steps, labels, evidence, or safer wording.
- **Blocked** means you cannot honestly finish the artifact yet because a tool, source, permission, example, or decision is missing.

## Evidence to Save

Save evidence that another teacher could inspect: a screenshot, copied command output, file path, short reflection note, or before-and-after excerpt. If the evidence is only in your memory, the checkpoint is not ready yet.

## Next Revision Trigger

Name the owner and trigger before you move on. Write who will revise it, when they will revisit it, and what would cause the next revision: a broken link, confusing student direction, missing source, unsafe permission, or a teammate review note.
