## The Teacher Problem

Tool literacy without safety is not enough.

Teachers need habits for backups, secrets, private data, permissions, no-overwrite rules, and AI-generated work before tools get powerful.

## The Idea

Safety is not fear. Safety is the workflow that lets you keep building without losing control of the course.

## Classroom Scenario

A teacher asks an AI agent to reorganize files. The agent might help, but only if the teacher has a backup, a Git checkpoint, clear scope, and a way to review changes.

## Weak Version

```text
The tool seems reliable, so I can trust it.
```

Trust is not a workflow.

## Better Version

```text
Before the tool changes anything, I will protect the source, define scope, and review the result.
```

## Build Step

Start a Safe AI Builder Checklist:

```text
Backup exists:
Git state checked:
Secrets protected:
Student data removed:
Scope defined:
Diff review required:
Verification step:
```

## Quality Check

The checklist should stop unsafe work before it begins, not only explain mistakes afterward.

## Safety / Accessibility / Source Note

If a tool asks for broad file access, account access, or publishing permission, treat that as a safety decision.
