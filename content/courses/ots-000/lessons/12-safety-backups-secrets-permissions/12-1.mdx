## The Teacher Problem

Automation can make the same mistake very quickly.

That is why backups come before bulk edits, scripts, generated files, and agent work.

## The Idea

A backup is a recovery path.

For project work, recovery may come from:

- Git history
- a branch
- a copied folder
- cloud file history
- exported package
- backup zip
- deployment rollback

The right backup depends on the project.

## Classroom Scenario

A teacher asks an agent to rename files. The agent renames too much. Git history lets the teacher inspect or undo changes. Without history, the teacher is stuck reconstructing from memory.

## Weak Version

```text
I will make a backup if something goes wrong.
```

Too late.

## Better Version

```text
I will confirm a recovery path before the tool changes files.
```

## Build Step

Add a backup section to your checklist:

```text
Project:
Backup method:
Last safe checkpoint:
How I would restore:
Who can help:
```

## Quality Check

If you cannot explain how to restore, you do not have a practical backup yet.

## Safety / Accessibility / Source Note

Backups can also contain private data. Store them in approved locations and do not share backup archives casually.

## Reflection

What kind of project change would make you want a backup first?

## Capstone Connection

Your Starter Kit should name the backup habit you use before large edits or AI-agent work.
