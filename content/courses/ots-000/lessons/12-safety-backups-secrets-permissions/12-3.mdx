## The Teacher Problem

AI drafts can overwrite real authored work if the workflow does not protect source files.

This is one of the easiest ways to damage a course.

## The Idea

A diff shows what changed.

A no-overwrite rule tells tools and people what must not be replaced.

A draft folder gives generated work a safe place to land before review.

## Classroom Scenario

A teacher asks AI to improve lesson clarity. Without boundaries, the tool rewrites the whole lesson and removes a classroom-specific example. With boundaries, the tool suggests revisions in a draft note or only touches a named section.

## Weak Version

```text
Rewrite this course and make it better.
```

Too broad.

## Better Version

```text
Suggest revisions for the directions section only. Do not overwrite the authored lesson file. Put suggestions in a draft note.
```

## Build Step

Add no-overwrite rules:

```text
Do not overwrite:
Drafts go here:
Review before replacing:
Diff must show:
Stop if unrelated files change:
```

## Quality Check

Your rules should name specific protected folders or files, not just say "be careful."

## Safety / Accessibility / Source Note

Generated content should stay separate from student-facing published content until a teacher reviews accuracy, privacy, accessibility, source quality, and classroom fit.

## Reflection

What file or folder in your workflow should be protected from overwrite?

## Capstone Connection

Your Starter Kit should include no-overwrite rules before you use coding agents or bulk scripts.
