## The Teacher Problem

AI project help can be useful and still unsafe.

The checklist protects the teacher's judgment before speed takes over.

## The Idea

The Safe AI Builder Checklist should be used before any AI tool reads files, edits files, runs commands, connects to services, or prepares student-facing material.

## Classroom Scenario

A teacher wants Codex to update a course outline. The checklist forces the teacher to name the target files, protected files, backup state, verification commands, and review steps before work starts.

## Weak Version

```text
Ask AI to fix the course.
```

That is too broad to review.

## Better Version

```text
Ask AI to update one registry, protect authored lessons, show the diff, run the title check, and stop if unrelated files change.
```

## Build Step

Create the checklist:

```text
Task:
Allowed files:
Protected files:
Backup/Git checkpoint:
Private data removed:
Secrets protected:
Draft location:
Verification step:
Diff review:
Human approval before publish:
```

## Quality Check

The checklist should be specific enough that another teacher could tell whether the AI stayed in bounds.

## Safety / Accessibility / Source Note

If the task affects student-facing content, add checks for accuracy, source quality, privacy, accessibility, copyright, and classroom fit.

Which checklist item would stop the riskiest mistake in your current workflow?
