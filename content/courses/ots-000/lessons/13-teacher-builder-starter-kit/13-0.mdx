## The Teacher Problem

Tool literacy can scatter into separate notes.

The capstone pulls the useful pieces into one starter kit a teacher can actually use.

## The Idea

The Teacher Builder Starter Kit is a small packet that explains your setup, tools, docs, safety boundaries, and first project workflow.

It is not a portfolio of everything you learned. It is a ready reference for future work.

## Classroom Scenario

A teacher is about to start OTS-101 or OTS-240. The Starter Kit gives them their vocabulary map, workstation notes, folder map, VS Code sheet, terminal commands, GitHub checklist, docs links, Markdown sample, stack card, AI access map, and safety checklist.

## Weak Version

```text
I finished the lessons, so I am ready.
```

Completion is not the same as having a usable system.

## Better Version

```text
I assembled the artifacts I will use when I start real project work.
```

## Build Step

Create a Starter Kit index:

```text
1. Vocabulary map
2. Workstation checklist
3. Project-folder map
4. IDE and VS Code notes
5. Terminal cheat sheet
6. GitHub checklist
7. Documentation links
8. Starter Markdown lesson
9. Publishing flow
10. Stack card
11. AI/MCP access map
12. Safe AI Builder Checklist
```

## Quality Check

Each item should help you make a decision, not just prove you completed a lesson.

## Safety / Accessibility / Source Note

Remove secrets, private account details, and student information before sharing the kit.

## Reflection

Which artifact will you use first in real work?
