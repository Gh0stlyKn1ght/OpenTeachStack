## The Teacher Problem

Teachers collect tools faster than they collect reasons.

That leads to tool clutter.

## The Idea

A tool inventory names what you use, why you use it, and where the official docs live.

## Classroom Scenario

A teacher lists VS Code, GitHub, Node, Markdown, Vercel, and Codex. For each one, the teacher adds one job and one official documentation link.

Now the list supports work instead of becoming a pile of icons.

## Weak Version

```text
Tools I use: VS Code, GitHub, Node, AI.
```

Too vague.

## Better Version

```text
Tool: VS Code
Job: open and edit project folders
Docs: official VS Code documentation
Safety note: install extensions carefully
```

## Build Step

Create this table:

```text
Tool | Job | Official docs | My use | Safety note
```

Add at least six tools.

## Quality Check

Every tool needs a job. If you cannot name the job, remove it or mark it for later.

## Safety / Accessibility / Source Note

Use official docs for install and setup links. Use tutorials only as support.

## Reflection

Which tool in your inventory is most useful, and which one might be clutter?

## Capstone Connection

This inventory becomes the docs and tools section of the Starter Kit.
