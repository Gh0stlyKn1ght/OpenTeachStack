## The Teacher Problem

Teachers collect tools faster than they collect reasons.

That leads to tool clutter.

## The Idea

A tool inventory names what you use, why you use it, and where the official docs live.

## Classroom Scenario

A teacher lists VS Code, GitHub, Node, Markdown, Vercel, and Codex. For each one, the teacher adds one job and one official documentation link.

Now the list supports work instead of becoming a pile of icons.

## Weak Version

```text
Tools I use: VS Code, GitHub, Node, AI.
```

Too vague.

## Better Version

```text
Tool: VS Code
Job: open and edit project folders
Docs: official VS Code documentation
Safety note: install extensions carefully
```

## Build Step

Create this table:

```text
Tool | Job | Official docs | My use | Safety note
```

Add at least six tools.

## Quality Check

Every tool needs a job. If you cannot name the job, remove it or mark it for later.

## Safety / Accessibility / Source Note

Use official docs for install and setup links. Use tutorials only as support.

## Reflection

Which tool in your inventory is most useful, and which one might be clutter?

## Capstone Connection

This inventory becomes the docs and tools section of the Starter Kit.

## Artifact Build Scan

### Artifact

Build or revise **Create Your Tool Inventory and Docs Bookmark Pack** as a small working artifact, not a decorative note.

### Build Steps

1. Name the classroom job this artifact is supposed to do.
2. Fill in the artifact with the smallest useful version a teacher could reuse.
3. Add one piece of evidence that shows the artifact is usable.
4. Mark anything unfinished as revise or blocked instead of hiding it.

### Example Output

A finished example should be specific enough that another teacher can tell what changed, where it lives, and how it would be used in a real course build.

### Quality Check

- The artifact has a clear title.
- The teacher decision or classroom use is visible.
- The evidence is saved in a form someone else can inspect.
- The next revision trigger is named if the artifact is not ready.

### Capstone Connection

Save this artifact in the Teacher Builder Starter Kit. It becomes part of the evidence trail for deciding whether the starter kit is ready, needs revision, or is blocked.
