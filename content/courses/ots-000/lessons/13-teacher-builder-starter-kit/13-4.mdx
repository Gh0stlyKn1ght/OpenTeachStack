## The Teacher Problem

If the artifacts stay scattered, the course does not become a usable on-ramp.

The final build turns the work into a kit.

## The Idea

The kit should be small enough to use and complete enough to guide a first project.

## Classroom Scenario

A teacher starts OTS-101 after this course. Instead of guessing what VS Code, GitHub, Markdown, AI agents, or docs mean, they open the Starter Kit and use their own notes.

## Weak Version

```text
Folder of course notes.
```

That is not assembled.

## Better Version

```text
Teacher Builder Starter Kit:
- setup summary
- tool inventory
- folder map
- VS Code sheet
- terminal commands
- GitHub checklist
- docs links
- Markdown lesson
- publishing flow
- stack card
- AI/MCP access map
- safety checklist
```

## Build Step

Assemble the kit in this order:

1. Setup summary
2. Vocabulary map
3. Workstation checklist
4. Project-folder map
5. VS Code survival sheet
6. Terminal cheat sheet
7. GitHub checklist
8. Documentation links
9. Starter Markdown lesson
10. Publishing flow diagram
11. Project stack card
12. AI/MCP access map
13. Safe AI Builder Checklist

## Quality Check

For each artifact, ask:

- Does this help me make a decision?
- Does this protect student-facing work?
- Does this name a safe next action?
- Would another teacher understand it?

## Safety / Accessibility / Source Note

Remove passwords, keys, student data, private URLs, screenshots with account details, and school-sensitive notes before sharing.

## Reflection

Which part of the kit will prevent the most confusion later?

## Capstone Connection

This is the final OTS-000 artifact.

## Capstone Assembly Scan

### What You Are Assembling

Assemble **Final Build: Teacher Builder Starter Kit** as the proof that OTS-000 produced a usable teacher-builder starting point.

### Required Pieces

- Project type decision
- Builder workstation checklist
- Folder map
- Tool and documentation inventory
- Starter markdown lesson
- Safety and backup notes
- Final ready, revise, or blocked decision

### Assembly Steps

1. Put the artifacts in one place.
2. Remove duplicates and vague notes.
3. Add missing evidence links, screenshots, paths, or review notes.
4. Write the final revision note before calling the kit done.

### Release Check

Do not treat the kit as ready unless another teacher could open it, understand the setup, and see what still needs attention.

### Final Revision Note

Name the owner, the next review trigger, and the first part of the kit that should be revised when the project changes.
