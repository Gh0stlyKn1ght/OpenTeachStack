# OTS-000 to OTS-101 Alignment

Date: 2026-06-28

Purpose: keep OTS-101 aligned now that OTS-000 exists as the teacher tech-stack orientation.

## Decision

OTS-000 is the on-ramp. OTS-101 is the next course.

OTS-101 should not act like the first time a teacher has heard of files, folders, Markdown, GitHub, VS Code, hosting, AI assistants, coding agents, MCP, backups, secrets, or basic project vocabulary. Those belong to OTS-000.

OTS-101 should use that orientation as background and focus on AI-assisted course content foundations: learning targets, student-facing lessons, prompts, verification, assignments, rubrics, publishing decisions, and the final Mini Course Content Packet.

## What OTS-000 Owns

OTS-000 teaches plain-language orientation to the teacher builder stack:

| Area | OTS-000 ownership |
| --- | --- |
| Tool vocabulary | Browser tools, local tools, IDEs, VS Code, terminal, Git, GitHub, hosting, domains, deployment, MCP, and AI agents. |
| File literacy | Files, folders, project structure, paths, Markdown, MDX, frontmatter, links, tables, images, and code blocks. |
| Safety basics | Backups, secrets, permissions, diffs, no-overwrite rules, student-data boundaries, and safe AI builder habits. |
| Documentation habits | Reading official docs, tutorials, changelogs, quickstarts, version notes, and troubleshooting pages. |
| Capstone artifact | Teacher Builder Starter Kit. |

OTS-000 gives teachers enough vocabulary and safety awareness to enter later courses without being thrown into technical language cold.

## What OTS-101 Should Own

OTS-101 should stay focused on teacher-facing course content work:

| Area | OTS-101 ownership |
| --- | --- |
| Curriculum distinction | Separate curriculum, course content, instructional materials, course platform, and curriculum system. |
| Learning design | Turn standards, district goals, or messy teaching ideas into learning targets and lesson sequences. |
| AI drafting workflow | Prompt AI with context, constraints, examples, revision notes, and teacher judgment. |
| Verification | Check AI-assisted drafts before students see them. |
| Student-facing artifacts | Build lessons, assignments, labs, rubrics, feedback notes, and publishing checklists. |
| Course content system | Organize the packet so another educator can inspect purpose, evidence, safety boundary, and next revision step. |
| Capstone artifact | Mini Course Content Packet. |

## Repetition To Remove Or Reframe

OTS-101 can briefly reference OTS-000 vocabulary, but it should not reteach these as primary lessons:

| Topic | OTS-101 treatment |
| --- | --- |
| VS Code, local files, and Markdown mechanics | Reference OTS-000 as prerequisite support; only explain what matters for course-content artifacts. |
| GitHub, repositories, branches, commits, and pull requests | Keep out of OTS-101 unless the teacher is choosing where to store a packet; route deeper work to OTS-240 or later. |
| Hosting, domains, DNS, deployment, and build failures | Keep OTS-101 publishing platform decisions non-technical; route site-building to OTS-301. |
| Coding agents, MCP, and advanced AI tooling | Keep OTS-101 focused on AI-assisted instructional drafting and verification; route agent workflows to OTS-320. |
| General AI tool comparison | Use only enough to support responsible drafting and verification. |

## Rewrite Direction For OTS-101

This is not a blank rebuild. It is a sequel alignment pass.

Priority changes:

1. Add an OTS-000 prerequisite note to OTS-101 entry materials.
2. Replace any "first-course" language with "after orientation" language.
3. Tighten lessons that explain tools so they connect directly to course-content artifacts.
4. Keep the Mini Course Content Packet as the capstone.
5. Add a short bridge in Chapter 01: "Use your Teacher Builder Starter Kit from OTS-000 as your reference while building this packet."
6. Preserve OTS-101's core sequence: define the course-content system, plan learning, prompt, verify, build, assess, publish, and revise.

## Alignment Work Applied

2026-06-28 pass:

- Chapter overview bridges added across Chapters 01-10.
- Chapter 01 tightened so the course content inventory can carry OTS-000 context without confusing tool/platform location with curriculum decisions.
- Chapter 02 tightened so learning target maps include revision triggers and local/tool/source/access boundaries before AI drafts course content.
- Chapter 03 tightened so prompting stays focused on bounded course-content drafts, not general AI tool comparison.
- Chapter 04 tightened so verification records allowed tools, access boundaries, and blocked checks instead of letting AI certify its own safety.
- Chapter 05 tightened so student-facing lessons include stuck-point support and do not depend on live teacher narration.
- Chapter 06 tightened so assignments distinguish completion from evidence and name what would not count as enough evidence.
- Chapter 07 tightened so organization work stays a course-content system map, not a full drive, repository, or LMS migration.
- Chapter 08 tightened so safety/accessibility/source review names the relevant OTS-000 or local boundary and avoids generic safety language.
- Chapter 09 tightened so publishing stays a student-view and source/export verification routine, not hosting, DNS, deployment, or course-site building.
- Chapter 10 tightened so the capstone can reference OTS-000 context without depending on the full Starter Kit or pretending to be a complete course.
- Core OTS-101 templates and the robotics sample packet updated to match the sequel boundary and revised artifact fields.
- Package-layer docs, examples, and metadata updated so stale release claims, flagship-beginner framing, and old workflow-audit wording do not override the OTS-000 sequence boundary.

## Release Boundary

OTS-101 should remain draft until its lessons are reviewed against this sequence:

- Does the lesson assume OTS-000 where appropriate?
- Does it avoid re-teaching tool orientation?
- Does it produce course-content evidence?
- Does it keep teacher judgment, verification, safety, accessibility, and source quality visible?
- Does it point advanced technical work to the later pathway instead of expanding OTS-101?

Because OTS-000 is now an active draft, public pathway language should make the sequence honest: OTS-000 is the on-ramp, OTS-101 is the foundations course, and neither is live until human review approves release.
