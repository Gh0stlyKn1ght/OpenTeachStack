# OTS-101 Examples

These examples support the flagship beginner pathway.

Use them to make each lesson artifact specific, practical, and safe.

- completed-artifact.md: starter mini-unit packet samples and quality criteria.
- weak-version.md: anti-patterns to diagnose shallow artifact quality.
- strong-version.md: revised artifacts with explicit evidence requirements.
- reviewer-notes.md: completion checks used before release.

The flagship examples include:
- teacher workflow audit sample,
- reusable prompt-library sample,
- AI verification reviewer notes,
- standards unpacking row,
- 3-5 lesson mini-unit map,
- resource evaluation comparison,
- observable rubric rows.

Privacy, safety, and licensing assumptions:
- All students and classes are fictional placeholders.
- Keep source and copyright notes visible on each sample.
- Do not use real student records.
