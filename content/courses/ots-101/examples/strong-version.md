| Prompt name | Grade 8 robotics sensor lesson outline |
| Prompt text | Act as a curriculum planning assistant for grade 8 robotics. The learning target is "I can use sensor data to justify a design decision." Students have 45 minutes and will use ultrasonic sensors and a shared data table. Draft a lesson outline with launch, work time, check for understanding, and close. The student product is a data table and a two-sentence threshold justification. Do not invent standards, sources, or facts. End with a teacher verification checklist. |
| Learning target | I can use sensor data to justify a design decision. |
| Student evidence | Five readings, chosen stop threshold, and two-sentence justification |
| Materials | Ultrasonic sensor, robot base, shared data table, teacher sample row |
| Must avoid | Invented standards, real student names, unsupported facts, unavailable tools |
| Quality check | Teacher confirms timing, materials, source alignment, safety, accessibility, and student evidence before use |
| Review trigger | Revise after first use if students skip units, misunderstand threshold, or need more setup support |

## Why This Is Strong

- The teacher supplies the learning target and evidence before AI drafts anything.
- The prompt keeps source verification with the teacher.
- The output request ends with a checklist instead of pretending the first draft is finished.
- The revision trigger makes the reusable course-content prompt improve after classroom use.
