## The teacher problem

Before AI can help improve a course, the teacher needs to know what already exists and what job each piece does.

Without an inventory, AI may rewrite the wrong thing.

It may polish an export file when the source lesson is missing. It may revise teacher notes as if they were student directions. It may make a quiz sound better without fixing the target.

## The artifact

Build a course content inventory for one small unit, packet, or course-content sequence.

The inventory does not need to be pretty.

It needs to tell the truth.

Record these fields for each item (one item per row in your own sheet):

| Inventory field | What you record |
| --- | --- |
| Item | The file, page, lesson, quiz, or note you are logging. |
| Category | curriculum, student-facing content, instructional material, platform/export, source, draft, archive, or unclear. |
| Audience | students, teacher, public, classroom-only, or private. |
| Location | Where the item actually lives right now. |
| OTS-000 context | Tool, platform, source, access, or safety note that helps explain the item. |
| Status | keep, revise, verify, archive, missing, or blocked. |
| AI help needed | The one narrow task AI could help with, if any. |
| Verification needed | What the teacher must check before students see it. |
| Next action | The single next step for this item. |

## Example output

For a robotics safety packet, one item might look like this:

| Inventory field | Example |
| --- | --- |
| Item | Startup checklist |
| Category | Student-facing course content |
| Audience | Students |
| Location | Drive/source folder |
| OTS-000 context | Source file lives in the shared course folder; safety routine must be checked locally. |
| Status | usable but wordy |
| AI help needed | simplify language |
| Verification needed | verify safety wording |
| Next action | revise checklist |

A weak inventory says:

> Robotics safety files: notes, quiz, slides, checklist.

A stronger inventory names audience, status, and next action.

## Build step

1. Choose one course, unit, packet, or course-content sequence.
2. If you completed OTS-000, open your Teacher Builder Starter Kit and use it to locate the real folders, links, drafts, platform exports, and safety notes connected to this unit.
3. List at least eight items connected to it.
4. Assign each item a category: curriculum, student-facing content, instructional material, platform/export, source, draft, archive, or unclear.
5. Assign each item an audience: students, teacher, public, classroom-only, or private.
6. Mark the status: keep, revise, verify, archive, missing, or blocked.
7. Add one next action for the most important student-facing item.

## Quality check

- [ ] Every item has a category.
- [ ] Every item has an audience.
- [ ] OTS-000 context is used only when it clarifies tools, access, source, or safety.
- [ ] Student-facing pieces are visible.
- [ ] Teacher-only pieces are not mixed into student content.
- [ ] Export/platform copies are labeled as exports.
- [ ] At least one missing or weak piece is named honestly.

## Boundary note

Use fake student names and fictional links in any example inventory.

If your real inventory includes private documents, summarize the type of item instead of pasting sensitive content into AI.

## Capstone connection

This inventory becomes the first evidence page in your final Mini Course Content Packet.

It should prove that your course has a source of truth before you publish to Google Classroom, Canvas, Teachable, a website, or any other platform.

If OTS-000 gave you a map of the tools and containers, this artifact shows what those tools are holding and whether the material is ready for students.
