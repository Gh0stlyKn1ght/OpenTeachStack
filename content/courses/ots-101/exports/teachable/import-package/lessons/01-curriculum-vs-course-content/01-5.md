## What you are checking

You are not checking whether the folder is neat.

You are checking whether the inventory points to content a teacher can actually teach from and students can actually use.

A course can have slides, quizzes, exports, and folders while still missing the student-facing lesson, example, task, evidence, or verification note.

## Review criteria

Use these questions:

| Question | What to look for |
| --- | --- |
| What will students do? | A visible task or learning action. |
| What will students use? | A lesson page, handout, lab, source, or example. |
| What will students produce? | Evidence the teacher can review. |
| What is teacher-only? | Notes, answer keys, setup details, or private reminders. |
| What OTS-000 context matters? | Tool, platform, file, source, or safety context needed to understand the item. |
| What needs verification? | Safety, source, privacy, accessibility, or policy claims. |
| What is missing? | Any piece needed before students can start. |

## Self-audit

Mark each inventory item with one decision:

- `ready for students`
- `teacher-only`
- `needs AI draft`
- `needs teacher verification`
- `archive`
- `missing`
- `blocked`

Add an OTS-000 context note only when it helps explain access, source, platform, safety, or tool boundaries. Do not add it as decoration.

Then choose one next action.

Keep it small enough to complete before the next class or planning block.

## Revise / keep / remove decision

Use this decision rule:

| If the item is... | Then... |
| --- | --- |
| student-facing and clear | keep it |
| student-facing but vague | revise it |
| teacher-only | separate it |
| unsupported by a source | verify it |
| only an export copy | trace it back to source |
| outdated or duplicated | archive it |
| missing | name it honestly |

## Evidence to save

Save three things from this checkpoint:

1. your completed inventory
2. one missing or weak student-facing item
3. one OTS-000 context note that matters, if any
4. one next action you will complete before moving on

That is enough evidence for Chapter 1.

## Boundary note

If the inventory includes content around equipment, health, identity, public accounts, student data, or safety procedures, mark it for teacher verification before publication.

Do not let AI be the final reviewer for those decisions.

## Next step

Write the most honest status of your course content right now:

```txt
ready / needs revision / needs verification / missing key pieces / archive first
```

Then write one sentence explaining why.

## How this protects the packet

This checkpoint becomes the first status note in your Mini Course Content Packet.

If your packet only has platform exports, outlines, or templates, it is not ready.

If it has lessons, examples, tasks, evidence, safety checks, and revision notes, it is becoming teachable.
