## What this chapter is really about

Most teachers do not start with a clean curriculum system.

They start with a folder, a district pacing guide, a few slides, an old quiz, some links, a rubric that almost works, and a deadline.

## What to Look For

Look for the teacher decision that turns this from a draft into usable course content.

Then AI enters the room and offers to make everything look complete.

## Why the Structure Matters

The structure is there so another teacher can inspect the work without guessing the process.

That is useful and dangerous at the same time.

## What Teachers Should Notice

The chapter is trying to make one teacher decision visible before any AI draft gets treated as finished.

This chapter teaches one habit before anything else: name what you are building before you ask AI to help build it.

## The core confusion

Teachers often use one word for five different things.

They say "curriculum" when they may mean:

- a learning plan
- a lesson page
- an assignment
- a slide deck
- an LMS module
- a downloadable packet
- a folder full of files

AI will not stop and clarify unless you force the issue.

## A real classroom version

Imagine a robotics teacher building a short lesson on safe robot startup and shutdown.

The teacher might have:

```txt
startup-slides.pdf
safety-quiz-old.docx
robot-startup-checklist-v3.docx
classroom-demo-notes.md
google-classroom-post.txt
student-lab-handout-draft.md
```

That pile is not bad. It is normal.

The problem is that the pile does not tell the teacher what each file is for.

## Before AI Helps

Name the target before asking AI to improve the artifact.

Which file is the source? Which one do students see? Which one is teacher-only? Which one needs a safety check? Which one is just a platform copy?

## The chapter promise

By the end of Chapter 1, you will not have a finished course.

You will have something more important: a course content inventory that tells the truth.

## What the Inventory Will Show

It will show:

1. what you already have
2. what students actually see
3. what is teacher-only
4. what AI may help draft or revise
5. what the teacher must verify
6. what is missing
7. what belongs in the source course instead of the export platform

## What to watch for

The trap in this chapter is trying to make the inventory pretty too soon.

Do not do that.

## Keep the First Inventory Honest

This first inventory is allowed to be messy. It only needs to be honest.

| If you find this | Mark it honestly as |
| --- | --- |
| A lesson outline with no lesson body | missing lesson |
| A quiz that does not match the target | needs alignment |
| A slide deck with teacher notes inside | mixed audience |
| A Teachable upload packet | export support |
| A prompt that produced useful text | draft support |
| A student page that can be taught tomorrow | ready candidate |

## Chapter Purpose

This chapter gives teachers one usable way to handle curriculum vs course content without asking AI to invent the whole course. The point is to name the teaching decision first, then use AI only where it can support that decision.

## Lessons in This Chapter

- Start by naming what the teacher is actually building.
- Separate the learning plan from the student-facing material.
- Use AI for draft support, examples, and cleanup only after the teacher has named the target.
- Save one visible artifact so the chapter ends with evidence, not just reading.

## Exit Criteria

By the end of this chapter, the teacher should be able to point to Course content inventory and explain what changed because of the chapter, what still needs review, and what is safe to reuse with students.

## What You Will Build

You will build or revise Course content inventory so the chapter ends with a visible classroom artifact, not just a set of notes.
