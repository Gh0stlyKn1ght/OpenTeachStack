# The Scattered Files Problem

## Why This Matters

A teacher building a first reusable mini-unit needs a system another educator can inspect. Scattered files are one of the fastest ways for a good teaching idea to become unusable. The plan may exist, but the current slide deck, source notes, rubric, student handout, and revision comments live in different places.

The problem is not that teachers are careless. The problem is that teaching produces many kinds of material quickly: drafts, copies, student versions, private notes, answer keys, screenshots, links, and old examples. If the workflow does not say what belongs where, the folder becomes a junk drawer.

## What Scattered Looks Like

For one mini-unit, scattered files might include:

- a slide deck in last year's Drive folder,
- a handout copied into the learning management system,
- an answer key stored in the teacher's downloads folder,
- a source link pasted in a private note,
- a rubric attached to an old assignment,
- student examples mixed with current materials,
- a revised prompt that never replaced the first version.

When files scatter, teachers lose more than time. They lose the evidence trail that shows why a lesson is trustworthy.

## Sort By Role

Do not start by making fancy folders. Start by naming file roles:

| File role | Question |
| --- | --- |
| Source | What information or standard supports the lesson? |
| Teacher prep | What does the teacher need before class? |
| Student-facing | What will students see or use? |
| Evidence | What will students submit or produce? |
| Private or sensitive | What must not be shared publicly? |
| Archive | What version and revision note should remain after teaching? |

Once roles are clear, folder names become easier. For a small mini-unit, a plain structure works:

- `01-source-notes`
- `02-teacher-prep`
- `03-student-facing`
- `04-assessment-evidence`
- `05-archive-revisions`

## Classroom Example

For the ultrasonic sensor lesson, the teacher finds three old data tables. One has no units. One uses inches. One has a broken link to a video. The fix is not just deleting files. The teacher records a decision:

> Current student-facing data table uses centimeters, includes one sample row, and lives in the Unit 2 student-facing folder. Old inches table moved to archive with note: "Do not use; conflicts with current sensor directions."

That note prevents the same confusion from returning next year.

## Practice

Choose one lesson, lab, project, or unit folder. List five files or links connected to it. For each one, assign a role:

- source,
- teacher prep,
- student-facing,
- evidence,
- private,
- archive.

Then mark one item as current and one item as retired.

## Add To The Workflow Audit

Add a scattered-file repair row:

| Field | Entry |
| --- | --- |
| Workflow | Name the lesson or routine |
| Scattered item | Name the confusing file, duplicate, or missing link |
| Role | Source, prep, student-facing, evidence, private, or archive |
| Decision | Keep, revise, move, retire, or replace |
| Reason | Why this prevents confusion or risk |

## Quality Check

Your repair is strong if another teacher can tell which file is current, which file is retired, and why the decision matters for students.

## Reflection

Which file in your current teaching workflow looks harmless but could send another teacher or student to the wrong version?
