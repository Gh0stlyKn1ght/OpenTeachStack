# Source -> Prompt -> Build -> Verify -> Teach -> Archive -> Improve

## Why This Matters

A teacher building a first reusable mini-unit needs a system another educator can inspect. The chapter method gives that system a repeatable rhythm: **Source -> Prompt -> Build -> Verify -> Teach -> Archive -> Improve**.

This method is especially useful when AI or templates are involved. It keeps the teacher from accepting a polished draft before checking where the content came from, what students will do, and how the lesson will be revised after use.

## The Method

| Step | Teacher question |
| --- | --- |
| Source | What standard, source, prior lesson, or official documentation anchors this work? |
| Prompt | What request, outline, or task description will guide drafting? |
| Build | What student-facing material, example, assessment, or routine is created? |
| Verify | What must be checked before students use it? |
| Teach | How will students interact with the material in class? |
| Archive | What final version and evidence should be stored? |
| Improve | What revision should happen after the first use? |

The method is not meant to make every lesson bureaucratic. It is meant to keep the important decisions from disappearing.

## Robotics Example

Source:

> Students need to collect sensor data and justify a design decision using evidence.

Prompt:

> Draft grade 8 student directions for a 45-minute ultrasonic sensor activity. Students should collect five distance readings, mark one reading that may be unreliable, choose a robot stop threshold, and write a two-sentence justification. Use plain language and include a data table with centimeters.

Build:

> One-page student direction sheet, sample data row, and teacher setup note.

Verify:

> Check that the sensor works, the units are consistent, the directions do not expose private student data, and the task matches the learning target.

Teach:

> Teams collect readings, discuss reliability, and submit a threshold decision.

Archive:

> Store the final direction sheet, blank response table, one anonymous sample, and the teacher setup note.

Improve:

> Add a clearer warning about units because two teams mixed centimeters and inches.

## Practice

Choose one teaching material you recently created or need to create. Fill in each method step with one sentence. If a step is blank, that is useful information. It shows where the workflow is not yet inspectable.

Avoid vague entries:

- Source: "internet"
- Prompt: "make lesson"
- Verify: "look over it"
- Improve: "fix later"

Use inspectable entries:

- Source: "district pacing guide objective 2.3 and teacher-created sample data"
- Prompt: "draft student directions for 45-minute data collection"
- Verify: "check units, reading level, setup safety, and evidence alignment"
- Improve: "revise table header if students omit units"
