# Teacher Workflow Audit

## Why This Matters

A teacher building a first reusable mini-unit needs a system another educator can inspect. The **Teacher workflow audit** is the first proof that the system exists. It does not need to be beautiful. It needs to make one real workflow clear enough that someone else can understand the purpose, files, student task, evidence, checks, and next revision.

This build task turns the chapter ideas into a working artifact.

## What You Are Building

Create a small audit table for one mini-unit, lesson sequence, lab routine, assessment routine, or recurring classroom workflow.

Your audit should answer:

- What workflow are you auditing?
- What problem does the workflow solve?
- Which source or standard anchors the work?
- What will students do?
- What evidence will students produce?
- Which files, links, tools, or materials are involved?
- What must be verified before class?
- What should be archived after class?
- What revision is already visible?

## Minimum Useful Version

A complete first version can be small:

| Field | Example |
| --- | --- |
| Workflow | Sensor-threshold mini-unit |
| Problem | Students collect data but do not always connect readings to a design decision |
| Source | Course target: collect data and justify a design choice |
| Student task | Record five sensor readings and choose a stop threshold |
| Evidence | Data table and two-sentence design justification |
| Current materials | Slide deck, response sheet, setup note |
| Verification | Test sensor, confirm units, check table clarity |
| Archive | Final directions, blank response sheet, anonymized sample |
| Improvement | Add unit reminder and one sample row |

This is better than a long narrative because it is easy to inspect.

## Build Steps

1. Choose one workflow you can explain in less than one minute.
2. Name the student-facing task before naming the tool.
3. List only the files and materials that are current.
4. Mark at least one item that should be retired, revised, or archived.
5. Add one verification check that happens before students use the material.
6. Add one improvement note that would help the next teacher or future you.

## What To Avoid

Avoid an audit that only lists apps:

> Google Docs, Google Forms, slides, AI.

That list does not explain the teaching work.

Avoid an audit that only lists intentions:

> Make students understand sensors better.

That goal may be true, but it does not show what students will do or how the teacher will verify the material.

Use action and evidence:

> Students collect five readings, identify one possible bad reading, choose a stop threshold, and justify the choice. Teacher verifies units and setup before class. Revision note tracks whether the sample row was enough.

## Quality Check

Your workflow audit is ready for checkpoint review when it includes:

- one real workflow,
- one source or standard,
- one student-facing task,
- one piece of student evidence,
- one current material location,
- one verification check,
- one archive or revision decision.

## Reflection

Show your audit to a colleague or imagine doing so. What would they still have to ask you before they could reuse the workflow?
