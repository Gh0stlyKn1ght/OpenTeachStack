# Checkpoint Review

## Why This Matters

A teacher building a first reusable mini-unit needs a system another educator can inspect. The checkpoint asks whether your first workflow audit is actually inspectable or only familiar to you because you already know the lesson.

Do not grade the artifact by how polished it looks. Grade it by whether another teacher could understand the workflow, find the current materials, see what students produce, and know what to check before teaching.

## Review Questions

Use these questions to review your **Teacher workflow audit**:

1. Does the workflow name a specific teaching routine, mini-unit, lesson, lab, or assessment?
2. Is the student-facing task written plainly?
3. Is the student evidence visible?
4. Is the source or standard named well enough to check?
5. Are the current materials separated from old drafts or retired files?
6. Is there at least one verification step before students use the material?
7. Is there a revision note that will help after the first use?

If the answer to a question is no, the artifact is not a failure. The question has found the next useful revision.

## Ready, Revise, Blocked

Mark each part of the audit with one status:

| Status | Meaning |
| --- | --- |
| Ready | Another teacher could inspect or use this part now |
| Revise | The idea is right, but the wording, file location, evidence, or check is unclear |
| Blocked | A source, material, approval, privacy check, or decision is missing |

Most first drafts should include some revise items. That is normal. The point is to make the next action visible.

## Example Review

Audit entry:

> Students collect five sensor readings and choose a threshold.

Review:

| Question | Status | Revision |
| --- | --- | --- |
| Student task clear? | Ready | Task names readings and threshold |
| Evidence visible? | Revise | Add "data table plus two-sentence justification" |
| Source named? | Blocked | Add course standard or unit target |
| Materials current? | Revise | Link current response sheet and retire old inches table |
| Verification step? | Revise | Add "test one sensor and check centimeter units" |

The checkpoint creates a practical revision list instead of a vague feeling that the lesson needs work.

## Final Check

Before you leave this chapter, complete this sentence:

> This workflow is ready enough to test because another teacher can see ___, ___, and ___.

Then complete this sentence:

> The next revision is ___ because ___.
