# Prompting for Rubrics and Quizzes

## Why This Matters

A teacher building a first reusable mini-unit needs a system another educator can inspect. Rubrics and quizzes are high-risk AI tasks because they can look official while measuring the wrong thing.

Assessment prompts must begin with the evidence of learning. If students are supposed to justify a threshold with sensor data, the assessment should not only ask vocabulary questions.

## Rubric Prompt

Use this when students create a product:

> Act as an assessment assistant. Create a four-level rubric for [student product]. The learning target is [target]. Criteria must be observable in student work. Include three criteria: accuracy, evidence use, and clarity. Use student-friendly language. Do not add criteria that are not visible in the product. End with a teacher review note.

Robotics example:

> Create a four-level rubric for a two-sentence threshold justification. The learning target is "I can use sensor data to justify a design decision." Criteria must be observable: sensor data used, threshold decision explained, and units clear.

## Quiz Prompt

Use this when checking understanding:

> Create a five-question check for understanding on [concept]. Include the correct answer, the misconception each question checks, and one reteach note. Do not include trick questions or facts not provided by the teacher.

## Teacher Review

Check every generated assessment for:

- alignment to the learning target,
- observable evidence,
- student-friendly language,
- answer-key accuracy,
- bias or confusing wording,
- whether private student data is absent.
