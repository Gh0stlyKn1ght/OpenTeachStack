# Prompt Library

## Why This Matters

A teacher building a first reusable mini-unit needs a system another educator can inspect. A **Reusable prompt library** gives AI-assisted work a memory. It records what the prompt is for, what context it needs, what it must not include, and how the teacher checks the result.

## What You Are Building

Create a small library with at least three reusable prompts:

1. Planning or student directions
2. Assessment or rubric drafting
3. Revision or verification

Each prompt should have a note that explains when to use it.

## Example Prompt Card

| Field | Entry |
| --- | --- |
| Prompt name | Student lab directions |
| Use case | Turn teacher notes into student-facing steps |
| Prompt text | Rewrite these notes for grade 8 students using numbered steps. Keep the task the same. Include what students submit. Mark unclear teacher decisions as questions. |
| Teacher context needed | Grade, time, materials, student product |
| Privacy rule | No student names, grades, behavior, or private accommodations |
| Source/alignment rule | Use only the learning target supplied by the teacher |
| Teacher verification step | Check task accuracy, reading level, materials, timing, and evidence |
| Example classroom use | Ultrasonic sensor data-table directions |

## Build Task

Create three prompt cards. Then choose one card and test it with a classroom-safe example. Revise the prompt based on what the AI output gets wrong or leaves unclear.

## Reflection

Which prompt in your library would save another teacher the most time without hiding the teacher review step?
