# Checkpoint Review

## Why This Matters

A teacher building a first reusable mini-unit needs a system another educator can inspect. The checkpoint asks whether your prompt library is safe and usable, not whether the prompts sound impressive.

## Review Questions

Check each prompt card:

1. Does the prompt name a clear teaching task?
2. Does it include the context the teacher must provide?
3. Does it forbid private student or school information?
4. Does it stop the AI from inventing standards, sources, or facts?
5. Does it name the output format?
6. Does it include a teacher verification step?
7. Does the example use classroom-safe information?

## Ready, Revise, Blocked

Use one of three labels:

| Status | Meaning |
| --- | --- |
| Ready | The prompt can be tested with classroom-safe information |
| Revise | The prompt needs clearer context, constraints, or review criteria |
| Blocked | The prompt depends on missing source material, private data, or a teacher decision |

## Example Review

Prompt:

> Make a quiz about sensors.

Status: Revise

Reason: It does not name the learning target, grade, question types, answer-key expectations, or misconceptions.

Revision:

> Create a five-question check for grade 8 students on ultrasonic sensor distance readings. Use the learning target supplied below. Include an answer key, the misconception each question checks, and a teacher review note. Do not invent standards or facts.

- one planning or directions prompt,

- one revision or verification prompt,
- one privacy rule,
- one teacher verification rule,
- one tested and revised prompt.
