## The Teacher Problem

AI drafts are easier to review when the teacher has a map.

Without a map, every output looks possible. A worksheet, a slide deck, a discussion prompt, and a quiz can all seem useful even if none of them match the real target.

The learning target and lesson map gives AI a job and gives the teacher a way to judge whether the output belongs in the course.

## The Artifact

Build one map that connects the source goal to the student-facing course content.

Fill in one row per field:

| Map field | What you write here |
| --- | --- |
| Source goal or problem | The standard, goal, or classroom need you start from. |
| Student-friendly target | The plain-language action students should be able to take. |
| Practice path | The rehearsal students need before the evidence task. |
| Evidence | The product or explanation the teacher can inspect. |
| AI support | The one narrow draft task AI may help with. |
| Teacher verification | The check the teacher owns before students see it. |
| Revision trigger | The signal that tells you to revise after teaching. |

## Example Output

| Map field | Example |
| --- | --- |
| Source goal or problem | Students rush robot startup and skip checklist decisions. |
| Student-friendly target | I can choose the next safe startup step using the approved checklist. |
| Practice path | Two fictional scenarios, then powered-off rehearsal. |
| Evidence | Scenario choices plus checklist explanation. |
| AI support | Draft three fictional startup scenarios. |
| Teacher verification | Check scenarios against approved class checklist and equipment rules. |
| Revision trigger | Revise after first class if students miss the same checklist step. |

This map tells AI what to draft and tells the teacher what to inspect.

## Build Step

1. Choose one source goal, standard, or messy classroom need.
2. Write the student-friendly target.
3. Name the practice path students need before assessment.
4. Name the evidence students produce.
5. Name one narrow AI support task.
6. Name one teacher verification task.
7. Add the revision trigger you will use after teaching.

## Quality Check

Before keeping the map, check:

- [ ] The target is student-friendly and observable.
- [ ] The practice path prepares students for the evidence task.
- [ ] The evidence matches the target.
- [ ] The AI support task is narrow enough to review.
- [ ] The teacher verification task is explicit.
- [ ] The revision trigger is tied to a real teaching moment.

## Safety / Accessibility / Source Note

Do not paste private standards documents, student data, local safety policy, or confidential accommodations into AI unless you have permission.

Use approved public text, teacher-created summaries, or fictional scenarios when possible. Keep the verification note visible in the map.

## Reflection

Which row in your map would another teacher question first? Strengthen that row before moving on.

## Capstone Connection

This map becomes part of the final mini-course packet. It proves the lesson did not start as a random AI draft. It started from a target, evidence, and teacher verification plan.
