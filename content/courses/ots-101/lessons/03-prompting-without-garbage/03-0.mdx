# Prompting AI Without Generic Garbage

## Chapter Purpose

Most weak AI course content does not start weak because the tool is useless.

It starts weak because the prompt asks AI to finish the lesson before the teacher has supplied the source, context, task, format, and review checks that make the output teachable.

Chapter 3 turns the learning target and lesson map from Chapter 2 into a reusable prompt teachers can actually inspect.

## The Teacher Problem

A one-sentence prompt is tempting:

> Make me a lesson about digital citizenship.

AI will answer. It may even answer beautifully.

But if the prompt does not include the source, target, classroom situation, evidence need, and teacher review rules, the output will fill in the blanks with generic school language.

## The Idea

A useful prompt is not long because longer is better.

It is specific because the teacher has already made the instructional decision.

The prompt tells AI:

- what source to use
- what classroom context matters
- what one student-facing artifact to draft
- what format to return
- what examples and non-examples to follow
- what the teacher must verify

<FrameworkBlock title="The prompt is not the lesson" eyebrow="Core distinction">
  <ConceptCard title="Teacher decision">
    Name what students need to learn, what evidence will count, and what constraints matter in the room.
  </ConceptCard>
  <ConceptCard title="AI draft task">
    Ask AI to produce one specific artifact: directions, examples, rubric language, question stems, or a lesson sequence.
  </ConceptCard>
  <ConceptCard title="Teacher verification">
    Check accuracy, fit, accessibility, privacy, safety, and whether the artifact supports the learning target.
  </ConceptCard>
</FrameworkBlock>

## Classroom Scenario

The robotics teacher from Chapter 2 has this target:

> Students can choose the next safe startup step using the approved checklist.

Weak prompt:

> Make a robot safety lesson.

Better prompt:

> Use the approved startup checklist below. Draft three fictional startup scenarios where students choose the next safe action and explain the checklist step that supports it. Do not invent equipment rules. Return a table with scenario, correct next action, teacher check, and one accessibility note.

The second prompt does not hand AI the course. It hands AI one draft job.

## What You Will Build

You will build a reusable course-content prompt with these pieces:

| Prompt piece | What it protects |
| --- | --- |
| Source | accuracy and alignment |
| Context | classroom fit |
| Task | scope control |
| Format | reviewability |
| Example and non-example | course voice and specificity |
| Teacher check | safety, source, accessibility, and privacy review |

## Lessons in This Chapter

1. Diagnose why generic prompts create fake lessons.
2. Build the Source-Context-Task-Format-Check pattern.
3. Add examples and non-examples to steer quality.
4. Build the reusable course-content prompt.
5. Check whether the AI output sounds like your course.

## Exit Criteria

Before moving to Chapter 4, you should have:

- one reusable prompt tied to your Chapter 2 map
- one acceptable example
- one non-example to avoid
- one teacher-check section
- one accepted / revised / rejected AI output note

## Safety / Accessibility / Source Note

Do not paste private student information, confidential policy, student work, or protected documents into AI just to get a better draft. Use fictional examples, approved source text, or teacher-created summaries when needed.

## Reflection

Where does your current AI prompt ask the model to guess: source, context, task, format, or review check?

## Capstone Connection

The reusable prompt becomes part of the final mini-course packet. It shows how AI was used, what boundaries were set, and what the teacher still approved.
