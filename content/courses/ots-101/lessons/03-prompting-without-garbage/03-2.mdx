## Why This Framework Exists

A strong classroom prompt does not need to sound technical.

It needs to give AI enough information to draft one usable piece and enough limits that the teacher can inspect the result.

Use this pattern:

**Source. Context. Task. Format. Check.**

## The Framework

  </ConceptCard>

## Parts of the Model

| Part | Teacher question | Example line |
| --- | --- | --- |
| Source | What should AI use? | Use only the learning target and startup checklist below. |
| Context | Who is this for? | This is for a 35-minute middle school robotics class. |
| Task | What one thing should AI draft? | Draft three fictional startup scenarios. |
| Format | How should I receive it? | Return a table with scenario, student task, and teacher check. |
| Check | What must be verified? | End with assumptions and safety items I must review. |

## Classroom Scenario

The teacher has this Chapter 2 target:

> Students can choose the next safe startup step using the approved checklist.

Before AI drafts student-facing course content, the teacher uses the prompt pattern to keep the source, context, task, format, and check visible.

A useful prompt does not ask for a whole lesson. It asks for the piece the teacher needs next.

```text
Source:
Use the approved startup checklist below. Do not invent safety rules.

Context:
Middle school robotics, 35 minutes. Students have seen the checklist but skip steps when excited.

Task:
Draft three fictional startup scenarios where students choose the next safe action.

Format:
Return a table with scenario, correct next action, student explanation prompt, and teacher-check note.

Check:
List any assumptions or safety details I must verify before using the scenarios.
```

## Weak Version

Weak prompt:

> Make a startup safety lesson.

That asks AI to decide source, context, task, format, and check.

## Better Version

Better prompt:

> Use the approved startup checklist below. Draft three fictional scenarios where students choose the next safe action and cite the checklist. Return a table. Do not invent rules. End with teacher-check notes.

That gives AI a narrow job and keeps the teacher in charge.

## How to Use It

1. Start with the learning target and lesson map from Chapter 2.
2. Fill in each part of the prompt pattern.
3. Remove anything AI should not decide.
4. Ask for one artifact, not the whole course.
5. Review the teacher-check section before keeping the output.

## Failure Mode

| Missing part | What usually happens |
| --- | --- |
| Source | AI invents the center. |
| Context | AI writes for imaginary students. |
| Task | AI overproduces. |
| Format | Review takes longer. |
| Check | The draft sounds more certain than it should. |

## Quality Check

Your prompt is ready when you can point to each part and explain why it belongs there.

## Safety / Accessibility / Source Note

The source section is a boundary. Do not put private student data, confidential policy, or protected material there unless your school has approved that use.
