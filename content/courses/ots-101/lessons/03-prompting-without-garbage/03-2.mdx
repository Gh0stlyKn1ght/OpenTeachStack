# The Source-Context-Task-Format-Check Prompt Pattern

A strong classroom prompt does not need to be fancy.

It needs to be complete enough that the AI knows what role it is playing and limited enough that the teacher can inspect the result.

Use this pattern:

**Source. Context. Task. Format. Check.**

## 1. Source

The source tells AI what to anchor to.

This could be:

- a learning target
- a standard
- teacher notes
- a reading passage
- a rubric
- a prior lesson
- a district policy
- a sample student-facing task

Without a source, AI tends to invent the center of the lesson.

A source line can be simple:

> Use only the learning target and notes below as your source. Do not invent standards, tools, or facts.

## 2. Context

Context tells AI what kind of classroom this is for.

Useful context includes:

- grade band or course level
- time available
- student readiness
- tools students can access
- accommodations or accessibility needs
- what students already learned
- what the teacher wants to avoid

A context line can sound like this:

> This is for a 7th grade class with 35 minutes. Students have already discussed examples, but they need help turning examples into evidence-based claims.

## 3. Task

The task tells AI exactly what to produce.

Do not ask for everything at once if you only need one piece.

Weak task:

> Make a lesson.

Better task:

> Draft three student-facing practice prompts that help students support a claim with evidence from the source text.

The better task is narrower, which makes the output easier to inspect.

## 4. Format

Format tells AI how to return the work.

A format can be a table, a list, a script, a rubric row, a slide outline, or a student handout draft.

For course content, format is not decoration. It is part of usability.

A format line can sound like this:

> Return the draft as a two-column table: Student-facing directions and Teacher review notes.

## 5. Check

The check tells AI what to flag instead of pretending everything is ready.

Ask AI to identify:

- assumptions it made
- missing source information
- claims that need verification
- privacy or accessibility concerns
- places where teacher judgment is required

A check line can sound like this:

> End with a short teacher-check section listing anything I must verify before students see this.

## Put it together

Here is the skeleton:

```text
Use the source below as the ground truth.

Context:
[Who this is for, timing, tools, prior learning, constraints]

Task:
[The one artifact you want drafted]

Format:
[The structure you want back]

Check:
[What the AI must flag for teacher review]

Source:
[Paste or summarize the teacher-approved source material]
```

<TakeawayStrip tone="teacher">
  This pattern does not make AI perfect. It makes the draft inspectable, which is the point.
</TakeawayStrip>

## Why This Framework Exists

This framework keeps the source-context-task-format-check prompt pattern from becoming a pile of disconnected AI outputs. It gives the teacher a small set of parts to inspect before the material reaches students.

## Parts of the Model

- The teaching decision names what students should learn or do.
- The AI support drafts, organizes, or revises a bounded piece.
- The teacher verification checks accuracy, safety, accessibility, and classroom fit.
- The saved artifact preserves what changed for future reuse.

## How to Use It

Use the model on Reusable course-content prompt by labeling each part before revising. If a part cannot be labeled, slow down and clarify the teaching decision first.
