# Organizing a Course Content System

> OTS-101 frame: AI supports teacher-approved course content that becomes student-facing only after review.

A course is not organized because files exist.

A course is organized when the teacher can answer three questions quickly:

- Where is the source of truth?
- What is ready for students?
- What still needs review?

If those answers are hidden across folders, exports, drafts, platform pages, and old AI outputs, the course becomes fragile.

This chapter is about building a small system that keeps course content understandable.

<FrameworkBlock title="The course content system" eyebrow="Organization layer">
  <ConceptCard title="Source of truth">
    The teacher-approved place where the real course content lives.
  </ConceptCard>
  <ConceptCard title="Working drafts">
    AI-assisted or teacher-created material that is not ready yet.
  </ConceptCard>
  <ConceptCard title="Student-ready content">
    Material approved for students to use.
  </ConceptCard>
  <ConceptCard title="Platform exports">
    Copies adapted for an LMS, website, document, slide deck, or handout.
  </ConceptCard>
  <ConceptCard title="Archive">
    Old versions kept for reference, not accidentally reused as current content.
  </ConceptCard>
</FrameworkBlock>

## Why this chapter matters

AI makes file sprawl easier.

A teacher can generate five versions of a lesson, three rubrics, two handouts, a slide outline, and an LMS post in a few minutes. That can be useful, but only if the teacher knows which version is real.

Without a system, AI-assisted work creates a pile of plausible fragments.

With a system, those fragments become drafts, reviewed artifacts, exports, and archives.

## What you will build

By the end of this chapter, you will have a course content system map.

It will define:

- where teacher-approved source content lives
- where AI drafts go
- where student-ready artifacts go
- how platform exports are labeled
- how old versions are archived
- how review status is tracked

## The chapter path

1. First, we separate source content from platform exports.
2. Then we design a folder structure that makes status visible.
3. Then we add naming and version habits.
4. Then you build a course content system map.
5. Finally, you check whether another teacher could understand the system without you explaining it.

<TakeawayStrip tone="teacher">
  Organization is not clerical cleanup. It is how teachers keep control of what students actually receive.
</TakeawayStrip>

## Chapter Purpose

This chapter gives teachers one usable way to handle organizing course content system without asking AI to invent the whole course. The point is to name the teaching decision first, then use AI only where it can support that decision.

## Lessons in This Chapter

- Start by naming what the teacher is actually building.
- Separate the learning plan from the student-facing material.
- Use AI for draft support, examples, and cleanup only after the teacher has named the target.
- Save one visible artifact so the chapter ends with evidence, not just reading.

## Exit Criteria

By the end of this chapter, the teacher should be able to point to Course content system map and explain what changed because of the chapter, what still needs review, and what is safe to reuse with students.
