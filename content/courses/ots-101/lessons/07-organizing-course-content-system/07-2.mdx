> OTS-101 frame: AI supports teacher-approved course content that becomes student-facing only after review.

A folder structure should answer a question before anyone opens a file.

The question is:

> What is this file's job and status?

If draft files, reviewed lessons, exports, and old versions all live together, the folder cannot help the teacher make decisions.

## Use status folders

Start with a simple structure.

```text
course-content/
  source/
  drafts/
  student-ready/
  exports/
  archive/
  teacher-notes/
```

This is not the only possible structure, but it shows the important separations.

  </ConceptCard>

## Keep drafts away from student-ready content

Drafts should not sit beside approved content with similar names.

This is risky:

```text
lesson.md
lesson-final.md
lesson-new.md
lesson-ai-revised.md
lesson-use-this-one.md
```

That folder tells a future teacher almost nothing.

A clearer structure is:

```text
source/
  lesson-03-evidence-comparison.md

drafts/
  lesson-03-ai-draft-2026-06-26.md

student-ready/
  lesson-03-student-page.md

exports/
  canvas-lesson-03-2026-06-26.md

archive/
  lesson-03-old-version-2026-05-10.md

teacher-notes/
  lesson-03-facilitation-notes.md
```

Now the folder location helps explain the file.

## Make review visible

A file can be well-written and still not reviewed.

Use simple status labels:

- draft
- needs-review
- approved
- exported
- archived

Put the status somewhere visible:

- filename
- folder location
- frontmatter
- checklist row
- tracker sheet

The point is not the tool. The point is that another teacher can see the status before using the file.

## Keep teacher notes separate

Teacher notes may include:

- pacing reminders
- answer keys
- facilitation moves
- private context

They should not be mixed into student-ready materials by accident. Use a separate folder or clear label for teacher-only material.

```text
teacher-notes/
  lesson-03-facilitation-notes.md
  lesson-03-answer-key.md
```

## Use ASCII maps for clarity

A simple folder map is often clearer than a paragraph.

Example:

```text
ots-101/
  source/
    chapter-01/
    chapter-02/
  drafts/
    ai-drafts/
    teacher-drafts/
  student-ready/
    lessons/
    assignments/
  exports/
    lms/
    print/
    slides/
  archive/
```

That map gives collaborators a mental model of the system.

## Classroom Scenario

A teacher has three useful files: an AI draft, an approved student page, and a Canvas export. All three mention the same lesson.

If they sit in one folder with similar names, a future teacher has to guess which one students should see.

## Weak Version

Weak folder pattern:

```text
lesson-03.md
lesson-03-new.md
lesson-03-ai.md
lesson-03-final.md
```

Those names preserve confusion.

## Better Version

Better folder pattern:

```text
source/
  lesson-03-evidence-comparison.md

drafts/
  lesson-03-ai-draft-2026-06-26.md

student-ready/
  lesson-03-student-page-approved.md

exports/
  canvas-lesson-03-export-2026-06-26.md
```

Now the folder structure shows status before anyone opens a file.

## Build Step

Sketch the smallest folder map that would separate source, drafts, student-ready files, exports, archives, and teacher notes in your course.

Keep this as a map first. You are not reorganizing every drive, repository, or LMS space in this lesson. You are deciding what status labels your Mini Course Content Packet needs so drafts, source files, student-ready artifacts, exports, and teacher-only notes do not blur together.

## Quality Check

The folder design works when a colleague can tell where a new AI draft belongs and where an approved student handout belongs.

## Safety / Accessibility / Source Note

Keep teacher-only notes, answer keys, private student context, and draft AI output out of student-ready folders. Folder structure is part of privacy protection.

## When This Helps

Use this workflow when course files are multiplying faster than review status can keep up: AI drafts, approved lessons, exports, old versions, and teacher notes all start blending together.

## Steps

1. List the file types your course already has.
2. Sort each type into source, draft, student-ready, export, archive, or teacher-notes.
3. Draw only the folders or labels needed to show those statuses.
4. Identify one confusing file you would move or relabel later as a test.
5. Write the folder rule that would prevent the same confusion next time.
