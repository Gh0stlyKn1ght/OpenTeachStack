> OTS-101 frame: AI supports teacher-approved course content that becomes student-facing only after review.

A folder structure should answer a question before anyone opens a file.

The question is:

> What is this file's job and status?

If draft files, reviewed lessons, exports, and old versions all live together, the folder cannot help the teacher make decisions.

## Use status folders

Start with a simple structure.

```text
course-content/
  source/
  drafts/
  student-ready/
  exports/
  archive/
  teacher-notes/
```

This is not the only possible structure, but it shows the important separations.

  </ConceptCard>

## Keep drafts away from student-ready content

Drafts should not sit beside approved content with similar names.

This is risky:

```text
lesson.md
lesson-final.md
lesson-new.md
lesson-ai-revised.md
lesson-use-this-one.md
```

That folder tells a future teacher almost nothing.

A clearer structure is:

```text
source/
  lesson-03-evidence-comparison.md

drafts/
  lesson-03-ai-draft-2026-06-26.md

student-ready/
  lesson-03-student-page.md

## Make the Handoff Visible

The reader should see where teacher judgment stops and AI support begins.

exports/
  canvas-lesson-03-2026-06-26.md
```

## Keep the Trail

A reusable workflow saves the decision path, not just the final wording.

Now the folder location helps explain the file.

## Make review visible

A file can be well-written and still not reviewed.

Use simple status labels:

- draft
- needs-review
- approved
- exported
- archived

Those labels can live in filenames, frontmatter, a checklist, or a tracking sheet.

The point is visibility.

## Keep teacher notes separate

Teacher notes may include pacing reminders, answer keys, facilitation moves, and private context.

They should not be mixed into student-ready materials by accident.

## Check the Artifact

The output is ready only when the artifact can stand outside the chat.

Use a separate folder or clear label.

```text
teacher-notes/
  lesson-03-facilitation-notes.md
  lesson-03-answer-key.md
```

## Use ASCII maps for clarity

A simple folder map is often clearer than a paragraph.

Example:

```text
ots-101/
  source/
    chapter-01/
    chapter-02/
  drafts/
    ai-drafts/
    teacher-drafts/
  student-ready/
    lessons/
    assignments/
  exports/
    lms/
    print/
    slides/
  archive/
```

That map gives collaborators a mental model of the system.

<TakeawayStrip tone="teacher">
  A good folder structure does not just store files. It tells the teacher what can be trusted, what needs review, and what is only an export.
</TakeawayStrip>

## When to Use This Workflow

Use this workflow when design folders that show status needs to become repeatable enough that another teacher could follow the same decision path without guessing what happened.

## Workflow Steps

1. Name the teaching problem before opening the AI tool.
2. Gather the existing source material, examples, or constraints.
3. Ask for one narrow output tied to Course content system map.
4. Check the output against the lesson goal, privacy limits, and classroom reality.
5. Save the accepted version with a short revision note.
