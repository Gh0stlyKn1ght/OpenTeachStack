# Build Artifact: Course Content System Map

In this lesson, you will build a course content system map.

This map should be simple enough that another teacher could understand where things belong without a meeting.

## What the map needs

Your map should identify:

- source of truth
- draft location
- student-ready location
- export location
- archive location
- teacher-only notes location
- review status method
- naming pattern

## Step 1: Name the source of truth

Write one sentence.

```text
The source of truth for this course is:
```

Examples:

```text
The source of truth for this course is the source folder in the shared course drive.
```

```text
The source of truth for this course is the Markdown lesson folder in the course repository.
```

```text
The source of truth for this course is the teacher-approved curriculum document, not the LMS export.
```

## Step 2: Draw the folder map

Use a simple ASCII map.

```text
course-content/
  source/
  drafts/
  student-ready/
  exports/
  archive/
  teacher-notes/
```

Customize it to match your context.

## Step 3: Define each folder's rule

Add one rule per folder.

```text
source/ = teacher-approved source files edited first
drafts/ = AI or teacher drafts not ready for students
student-ready/ = approved student-facing artifacts
exports/ = platform-specific copies made from source or student-ready files
archive/ = retired versions that should not be used as current content
teacher-notes/ = private planning, answer keys, pacing, and facilitation notes
```

## Step 4: Add a naming pattern

Choose one pattern and use it consistently.

```text
[course]-[lesson]-[artifact]-[topic]-[status]-[date-if-needed]
```

Example:

```text
ots101-lesson-05-student-page-evidence-comparison-approved.md
```

## Step 5: Add review status

Choose how review status will be visible.

Options:

- folder location
- filename status
- frontmatter field
- checklist row
- spreadsheet tracker
- review note at the top of the file

Use more than one if the course is shared by a team.

## Step 6: Add the update rule

Write the rule that prevents source/export confusion.

```text
Update rule:
Make content changes in the source first. Then update student-ready files and exports. Do not edit exports as the primary source unless the source is updated immediately after.
```

## Build Step

1. Open the current version of Course content system map.
2. Mark the part that is unclear, missing, or unsafe for students.
3. Ask AI for a narrow revision of that one part, not a full replacement.
4. Accept, revise, or reject the output in teacher language.

## Quality Check

Before keeping the revision, confirm that Course content system map has a visible learning target, a student-facing direction, and one review point the teacher can actually inspect.
