> OTS-101 frame: AI supports teacher-approved course content that becomes student-facing only after review.

A course content system is only useful if someone can use it when you are not there to explain it.

That someone might be a teammate.

It might be a substitute.

It might be future you, tired in October, trying to remember which file is safe to publish.

## Run the right-version test

Imagine another teacher opens your course folder.

Can they answer these questions?

- Which file is the source of truth?
- Which files are drafts?
- Which files are student-ready?
- Which files are platform exports?
- Which files are old versions?
- Which files still need review?
- Where should a new AI draft go?
- Where should a revised student handout go?

## Fix confusing signals

If the answer is no, fix the system before making more content.

Common fixes:

- rename unclear files
- move drafts out of student-ready folders
- add an archive folder
- label exports with dates
- add a README with folder rules
- add review status to filenames or metadata
- remove duplicate final versions

Small cleanup now prevents big confusion later.

For this checkpoint, "fix" can mean writing the rule or marking the problem for later cleanup. Do not turn the capstone into a full drive migration. The goal is to make the right version findable enough for review.

## Add a README

A short README can explain the system in plain language.

Example:

```text
This folder contains the course source, drafts, student-ready content, exports, archives, and teacher notes.

Edit source files first.
Do not publish drafts.
Exports are copies for platforms and may be stale.
Archived files are not current.
```

A README is not fancy, but it gives the folder a voice.

## Final system review note

Use this note before moving on:

```text
Course content system review:
- Source of truth is clear: yes/no
- Drafts are separated: yes/no
- Student-ready content is labeled: yes/no
- Exports are labeled as exports: yes/no
- Archive exists for old versions: yes/no
- Naming pattern is understandable: yes/no
- Another teacher could find the right version: yes/no
- Status: ready / revise / not ready
```

## What this protects

This system protects against the exact mess AI can create quickly:

- fake-complete lessons
- duplicate versions
- stale exports
- old drafts appearing in the course
- platform pages that drift away from source content
- teachers losing trust in the course folder

<TakeawayStrip tone="teacher">
  If the right version is hard to find, the course is not organized yet. The folder may be full, but the system is not doing its job.
</TakeawayStrip>

## Evidence to keep

Save your system review note as evidence that the source of truth is findable without you. The trigger for the next review is the next time you add a course file and have to decide where it belongs.
