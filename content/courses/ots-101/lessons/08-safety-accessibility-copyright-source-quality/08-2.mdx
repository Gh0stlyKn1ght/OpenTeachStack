# Accessibility Is Part of the Lesson Design

> OTS-101 frame: AI supports teacher-approved course content that becomes student-facing only after review.

Accessibility is not something to sprinkle on after the lesson is finished.

It is part of how the lesson is built.

A student-facing page should be readable, navigable, and usable. Students should not have to fight the format before they can do the thinking.

## Start with ordinary access

Before getting complicated, check the basics:

- Can students open the materials?
- Are links visible and named clearly?
- Are directions broken into steps?
- Are headings used to organize the page?
- Are lists actually formatted as lists?
- Is the expected product clear?
- Are images or media explained?
- Is the language direct enough for the intended students?

These are accessibility moves and good teaching moves.

## Use headings as signposts

Headings tell students where they are.

A lesson page should not be one long scroll of paragraphs.

Useful headings include:

- Learning target
- Why this matters
- Materials
- Directions
- Example
- Before you submit
- Need help?

Those headings help students scan and return to the right section.

## Use real lists

If something is a list, format it as a list.

Bad:

```text
learning target
short explanation
example
student task
submission instructions
quality check
```

Better:

- learning target
- short explanation
- example
- student task
- submission instructions
- quality check

The second version gives the eye structure.

## Explain media and visuals

If a lesson includes an image, diagram, video, chart, or slide, students need enough context to use it.

Ask:

- What should students notice?
- Is there text alternative or explanation?
- Does the media require sound?
- Is a transcript or caption needed?
- Does the image carry essential information?

If the visual is decorative, keep it from becoming noise. If it is instructional, explain how students should use it.

## Do not make access depend on guessing

Students should not have to guess where the file is, which link matters, or what to submit.

Use specific wording:

```text
Open the file named source-comparison-chart.
```

instead of:

```text
Open the chart.
```

Specific directions reduce unnecessary barriers.
