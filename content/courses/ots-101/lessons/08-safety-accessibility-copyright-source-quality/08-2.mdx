> OTS-101 frame: AI supports teacher-approved course content that becomes student-facing only after review.

Accessibility is not something to sprinkle on after the lesson is finished.

It is part of how the lesson is built.

A student-facing page should be readable, navigable, and usable. Students should not have to fight the format before they can do the thinking.

## Start with ordinary access

Before getting complicated, check the basics:

- Can students open the materials?
- Are links visible and named clearly?
- Are directions broken into steps?
- Are headings used to organize the page?
- Are lists actually formatted as lists?
- Is the expected product clear?
- Are images or media explained?
- Is the language direct enough for the intended students?

These are accessibility moves and good teaching moves.

## Use headings as signposts

Headings tell students where they are.

A lesson page should not be one long scroll of paragraphs.

Useful headings include:

- Learning target
- Why this matters
- Materials
- Directions
- Example
- Before you submit
- Need help?

Those headings help students scan and return to the right section.

## Use real lists

If something is a list, format it as a list.

Bad:

```text
learning target
short explanation
example
student task
submission instructions
quality check
```

Better:

- learning target
- short explanation
- example
- student task
- submission instructions
- quality check

The second version gives the eye structure.

## Explain media and visuals

If a lesson includes an image, diagram, video, chart, or slide, students need enough context to use it.

Ask:

- What should students notice?
- Is there text alternative or explanation?
- Does the media require sound?
- Is a transcript or caption needed?
- Does the image carry essential information?

If the visual is decorative, keep it from becoming noise. If it is instructional, explain how students should use it.

## Do not make access depend on guessing

Students should not have to guess where the file is, which link matters, or what to submit.

Use specific wording:

```text
Open the file named source-comparison-chart.
```

instead of:

```text
Open the chart.
```

Specific directions reduce unnecessary barriers.

## Classroom scenario

A teacher has a good student task, but the platform page is one long block of text. The video is embedded without context, the directions say "complete the chart," and the chart is linked under a vague file name.

Some students will still manage. Others will spend their energy figuring out the page instead of learning the content.

## Weak version

```text
Watch the video and complete the chart. Submit when finished.
```

This may be clear to the teacher who built the page, but it hides too much from students.

## Better version

```text
1. Watch the video titled "How Algorithms Sort Information."
2. Use captions or the transcript if sound is not available.
3. Open the file named algorithm-sort-chart.
4. In the chart, write one claim, one example, and one question.
5. Submit the completed chart in the assignment box.
```

The better version names the media, access option, file, thinking task, and submission path.

## How to use the framework

Use the accessibility checks on one lesson page before polishing the language.

Mark each part:

- Structure: headings, bullets, steps, and page order
- Language: direct wording and explained vocabulary
- Media: context, captions, transcript, alt text, or alternative path
- Navigation: visible links, clear file names, submission path, and next step

If one part is missing, fix that part before asking AI to make the lesson sound better.

## Build step

Choose one student-facing lesson page and add an accessibility note to the checklist.

Record:

- one formatting change that will make the page easier to scan
- one direction that needs to be more specific
- one media or visual item that needs context or an alternative
- one link, file, or submission path that needs a clearer label

## Quality check

Ask a simple question: could a student who missed your verbal explanation still understand what to open, what to do, and what to submit?

If the answer is no, the page is not accessible enough yet.

## Reflection

Which part of your lesson currently depends on students already knowing your classroom routine?

Revise that part so the page carries the needed information.

## Capstone connection

Your mini-course packet should show that accessibility was designed into the lesson, not added as a final apology after the content was already built.
