> OTS-101 frame: AI supports teacher-approved course content that becomes student-facing only after review.

The teacher view can lie.

Not because the platform is trying to trick you, but because teacher accounts often have access students do not have.

A link that works for the teacher may fail for students. A file may be visible to the owner but not the class. A rubric may be attached but hidden. A due date may be missing. A module may be unpublished.

That is why student view matters.

## Start from the student path

Do not only preview the page directly.

Start where students start.

For example:

- course home page
- module page
- classwork stream
- weekly agenda
- assignment list
- linked calendar item

Ask whether the lesson is findable from the path students actually use.

## Open every required item

Click like a student.

Check:

- links
- documents
- videos
- forms
- slides
- images
- downloads
- embedded content
- rubric or success criteria

If students need it, test it.

## Compare directions to platform settings

Student directions and platform settings must match.

Common mismatches:

| Direction says | Platform actually does |
|---|---|
| Upload a file | Text entry only |
| Complete the form | Form link missing |
| Use the attached rubric | Rubric hidden |
| Submit by Friday | No due date shown |
| Work with a partner | Individual submission settings only |
| Watch the video | Video blocked or private |

Fix mismatches before release.

## Check formatting after publish

Look for the basics:

- headings are visible
- bullets are bullets
- numbered steps stay numbered
- tables are readable
- examples are separated from directions
- success check is easy to find
- long paragraphs are broken up

If the platform flattens the content, rebuild the structure.

## Add a release note

A short release note helps you remember what was checked.

```text
Student view checked:
- module path visible
- links open for students
- assignment accepts text entry
- rubric visible
- source note recorded
```

<TakeawayStrip tone="teacher">
  If the student view does not work, the lesson is not published yet. It is only placed somewhere.
</TakeawayStrip>

## When to Use This Workflow

Use this workflow when check the student view needs to become repeatable enough that another teacher could follow the same decision path without guessing what happened.

## Workflow Steps

1. Name the teaching problem before opening the AI tool.
2. Gather the existing source material, examples, or constraints.
3. Ask for one narrow output tied to Platform publishing checklist.
4. Check the output against the lesson goal, privacy limits, and classroom reality.
5. Save the accepted version with a short revision note.
