# Build Artifact: Platform Publishing Checklist

> OTS-101 frame: AI supports teacher-approved course content that becomes student-facing only after review.

In this lesson, you will build a checklist for publishing approved content to a platform.

This checklist helps you catch problems that happen after the lesson leaves the source file.

## What the checklist covers

Your checklist should cover:

- approved source
- platform destination
- formatting
- files and links
- student access
- submission settings
- rubric or success criteria
- safety and accessibility
- source/export note

## Step 2: Confirm source and destination

```text
Source and destination:
[ ] I used the approved source, not an AI draft or stale export.
[ ] The platform destination is correct.
[ ] The platform copy is labeled as an export or includes a source note.
[ ] The source file remains the place to edit future content.
```

## Step 3: Check formatting and structure

```text
Formatting and structure:
[ ] Learning target is visible.
[ ] Headings organize the page.
[ ] Bullets and numbered steps render correctly.
[ ] Examples and non-examples are visually separated.
[ ] Success check or rubric is easy to find.
[ ] Long text is broken into readable sections.
```

## Step 4: Check links, files, and tools

```text
Links, files, and tools:
[ ] Required links open.
[ ] Files have student access permissions.
[ ] Media plays or has an alternative.
[ ] Tool instructions match the actual tool.
[ ] No unexpected account, payment, download, or personal-data barrier appears.
```

## Step 5: Check student submission

```text
Submission settings:
[ ] The platform accepts the kind of work directions ask students to submit.
[ ] Due date or availability settings are correct if used.
[ ] Rubric or success criteria are visible where students need them.
[ ] Students know what to do when finished.
```

## Step 6: Add final decision

```text
Publishing decision:
[ ] Ready to release
[ ] Revise platform copy and check again
[ ] Do not release yet

Release notes:

Known risks or follow-up:
```

<TakeawayStrip tone="teacher">
  The publishing checklist is the last mile between a strong source lesson and a course page students can actually use.
</TakeawayStrip>

## Build Step

Publish one lesson to the actual platform you use, then open it as a student would, logged out or in a student test account. Add a checklist line for every difference you find between the source file and the student view.

## Quality Check

The checklist is ready when running it would catch the gap between "looks right in my source file" and "works in the student's hands": broken bullets, a private link, a missing submission box.

## Capstone Connection

If the capstone packet includes a platform copy, attach this checklist as proof the export was verified from the student side, not just pasted in and assumed correct.
