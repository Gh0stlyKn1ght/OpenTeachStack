The order of the packet matters.

If the checklist comes before the lesson, the reader has no context. If the assignment appears before the target, the evidence feels unanchored. If exports appear before the source, the packet invites confusion.

Use an order that tells the story of the course content.

## Section 1: Overview

Start with a short overview.

Include:

- mini-course title
- audience or class
- time span
- source of truth
- status
- reviewer
- date

Example:

```text
Mini-course title: Evaluating Online Sources
Audience: Grade 8 media literacy
Time span: 2 class periods
Source of truth: course-content/source/evaluating-online-sources/
Status: ready for teacher review
Reviewer: JC
Date: 2026-06-26
```

## Section 2: Learning path

Include the learning target and map.

Then include the student-facing lesson draft.

The reader should be able to see how the target becomes student action.

## Section 3: Evidence and assessment

Add the assignment or lab packet.

Include:

- student task
- evidence students submit
- rubric or success criteria
- feedback prompts

This section answers: how will we know students learned it?

## Section 4: Review and verification

Add checklists and notes.

Include:

- AI draft verification checklist
- safety and access checklist
- source quality notes
- copyright or reuse notes
- platform publishing checklist if a platform export exists

This section answers: what has the teacher checked?

## Section 5: Revision plan

End with a revision plan.

Use three categories:

```text
Ready now:

Needs revision:

Do not publish yet:
```

This keeps the packet honest.

## Use a table of contents

A small table of contents helps the packet feel navigable.

```text
1. Overview
2. Learning path
3. Student-facing lesson
4. Assignment and rubric
5. Feedback prompts
6. Verification checklists
7. Platform publishing notes
8. Revision plan
```

## When to use this workflow

Use this workflow when the capstone pieces exist but the packet still feels like a folder dump.

The goal is not to impress someone with volume. The goal is to help another teacher follow the course logic, inspect the decisions, and know what is ready.

## Classroom scenario

A teacher shares a folder with a strong lesson, a useful rubric, several checklists, and a platform export. The reviewer opens it and cannot tell where to start.

The problem is not missing content. The problem is missing order.

## Weak version

```text
Folder order:
- rubric-final-final
- AI prompt notes
- lesson export
- old lesson
- safety checklist
- student task
- screenshots
```

This order makes the reviewer reconstruct the course from fragments.

## Better version

```text
Packet order:
1. Overview
2. Learning path
3. Student-facing lesson
4. Assignment or lab
5. Rubric and feedback prompts
6. Verification checklists
7. Source, safety, and access notes
8. Platform publishing note
9. Revision plan
```

This order tells the story of the course content.

## Workflow steps

1. Start with the overview so the reviewer knows the audience, focus, status, and source of truth.
2. Put the learning path before the student materials.
3. Put student-facing materials before assessment and feedback artifacts.
4. Put verification evidence after the materials it checks.
5. Put publishing notes only after the source material is clear.
6. End with an honest revision plan.

## Quality check

Ask:

- Can a reviewer understand the course focus in the first minute?
- Can they see the learning target before the activity?
- Can they tell which materials students see?
- Can they find the evidence and review notes without hunting?
- Can they tell what is ready and what is not?

## AI support

AI can help propose a table of contents or rename sections, but the teacher decides what is current, safe, and review-ready.

Do not let AI flatten the distinction between source files, platform exports, student pages, and teacher-only notes.

## Next action

Assemble your packet in this order and add a short table of contents at the top. Then mark any section that is still missing, stale, or not ready for student use.
