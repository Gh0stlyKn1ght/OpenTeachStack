# Build Artifact: Mini-Course Content Packet

In this lesson, you will assemble the final capstone artifact: a mini-course content packet.

This packet should be small, real, and reviewable.

Do not pad it with fake completeness.

## Packet template

Use this structure.

```text
Mini-Course Content Packet

1. Overview
2. Source of truth and system map
3. Course content inventory
4. Learning target and lesson map
5. Student-facing lesson
6. Assignment or lab packet
7. Rubric or success criteria
8. Feedback prompts
9. AI draft verification notes
10. Safety, accessibility, copyright, and source checks
11. Platform publishing notes
12. Final review and revision plan
```

## Step 1: Create the overview

```text
Mini-course title:
Audience:
Time span:
Learning focus:
Source of truth:
Packet status:
Reviewer:
Date:
```

## Step 2: Add the source and system map

Include your course content system map from Chapter 7.

Make sure it answers:

- where the source lives
- where drafts live
- where student-ready content lives
- where exports live
- where archived material lives

## Step 3: Add the planning artifacts

Include:

- course content inventory
- learning target and lesson map
- reusable course-content prompt if it was used

These artifacts explain the instructional decisions behind the packet.

## Step 4: Add the student-facing lesson

Include the student-facing lesson from Chapter 5.

Check that it has:

- learning target
- why this matters
- materials
- directions
- example
- non-example or common mistake
- success check
- submission instructions

## Step 5: Add assignment, rubric, and feedback

Include the assignment and feedback packet from Chapter 6.

Check that it names:

- task
- evidence
- rubric or criteria
- feedback prompts
- teacher verification notes

## Step 6: Add review checklists

Include:

- AI draft verification checklist
- safety and access checklist
- source quality and reuse notes
- platform publishing checklist if a platform copy exists

These prove the packet is not just AI output with nicer formatting.

## Step 7: Add the final review and revision plan

End with:

```text
Ready now:
- 

Needs revision:
- 

Do not publish yet:
- 

Next review date:

Teacher decision:
[ ] Ready for teacher review
[ ] Ready for limited classroom pilot
[ ] Revise before classroom use
[ ] Do not publish
```

## Finished packet checklist

Before you call the packet complete, check:

- [ ] Source of truth is named.
- [ ] Drafts and exports are separated.
- [ ] Learning target is visible.
- [ ] Student-facing lesson is readable.
- [ ] Assignment produces evidence.
- [ ] Rubric measures observable qualities.
- [ ] Feedback supports revision.
- [ ] AI use is documented.
- [ ] Safety and access checks are included.
- [ ] Publishing notes are included if relevant.
- [ ] Revision plan is honest.

<TakeawayStrip tone="teacher">
  The best mini-course packet is not the biggest one. It is the one another teacher can inspect, trust, and improve.
</TakeawayStrip>

## Required Pieces

- A clear course content inventory.
- Student-facing lesson material.
- An assignment, rubric, or practice task.
- A source and verification note.
- A revision note showing what AI helped with and what the teacher approved.

## Assembly Steps

1. Put each finished artifact in the packet.
2. Remove duplicate or outdated draft text.
3. Check that every student-facing item has directions and evidence.
4. Add one final teacher note about what still needs classroom testing.

## Release Check

The packet is ready to share only if a teacher can explain the learning target, the student action, the evidence, and the safety check without opening the AI chat history.

## Final Revision Note

End with a plain-language note: what changed, why it changed, and what you will review after teaching it once.
