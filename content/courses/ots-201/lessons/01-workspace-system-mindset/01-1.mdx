# Tool Collection vs Workspace System

## Why This Matters

A teacher can use Google Drive, Docs, Slides, Sheets, and Forms every day and still not have a workspace system. A tool collection stores things. A workspace system helps teaching work happen reliably.

This distinction matters because educators usually feel the pain at the worst moment: students cannot open the right file, a teammate edits the wrong document, or an old slide deck gets reused because it looked current.

A workspace system works when the current material, student evidence, access rules, and maintenance routine are visible to someone besides the teacher who built them.

## What You'll Learn

- Identify when a workflow is only a collection of tools.
- Name the evidence that a workflow is becoming a system.
- Spot hidden decisions that make a folder hard for another teacher to use.
- Add a system rating to your Workspace workflow audit.

## Core Idea

A tool collection answers "Where did I put the file?"

A workspace system answers "How does this teaching workflow run, who can use it, what evidence does it produce, and how does it stay current?"

The difference is not about how pretty the Drive folder looks. It is about whether the workflow survives real school conditions: absences, co-teaching, late submissions, old resources, new students, and rushed planning time.

## Classroom Scenario

Jordan has a Drive folder called `Advisory`. It contains check-in Forms, SEL slide decks, parent-contact templates, student goal sheets, and several files named `Copy of Advisory Plan`. Jordan can usually find what he needs, but his co-teacher cannot tell which check-in Form is current.

That is a tool collection. It becomes a workspace system when the current Form is named and linked, the response Sheet is known, the co-teacher has the right access, and the old Forms are archived.

## Model Artifact or Example

Add a system rating to one row of your audit.

| Audit field | Tool collection version | Workspace system version |
| --- | --- |
| Workflow | Advisory weekly check-in | Advisory weekly check-in |
| Current file | Several Forms in one folder | `Advisory Check-In Form - Current` |
| Student access | Teacher posts whichever link seems right | Current Form link lives in the advisory launch Doc |
| Evidence | Responses go to unknown Sheets | Response Sheet is linked in the tracker |
| Ownership | Jordan remembers what to update | Jordan owns Form; co-teacher can view responses |
| Rating | Tool collection | Emerging system |
| Next fix | Not clear | Archive old Forms and rename response Sheet |

## Common Mistake

The common mistake is calling a folder "organized" because it has subfolders. A folder named `Unit 2` with ten old slide decks and three current directions Docs may look organized, but it is not a system if no one can identify the current teaching path.

Another mistake is assuming that personal memory counts as documentation. If the workflow only works because you remember the link, the workspace is still fragile.

## Verification Checklist

- The rating is based on evidence, not confidence.
- The current file or link is named.
- Old versions are identified or marked as a risk.
- A second teacher could tell where student evidence goes.
- Access and ownership are not left to memory.
- The first improvement is small enough to complete this week.
