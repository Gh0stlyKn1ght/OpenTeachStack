# Workspace Workflow Audit

## Why This Matters

The Workspace workflow audit is the first real artifact in OTS-201. It prevents you from reorganizing Drive by guesswork. Before you build a folder map, template set, tracker, operations workflow, or command center, you need to know what is actually happening in one teaching workflow.

The audit turns memory into evidence.

A workspace system works when the audit shows what is current, who owns it, where evidence goes, and what needs maintenance.

## What You'll Learn

- Build a complete workflow audit row.
- Trace current files, links, evidence, ownership, access, and maintenance.
- Flag risks that would make the workflow hard to hand off.
- Choose the first workspace improvement based on evidence.

## Core Idea

An audit is not a cleanup checklist. It is a map of the current workflow.

You are looking for the points where a teacher, student, substitute, or teammate would have to guess. The most useful audit row usually exposes one of these problems:

- the current file is unclear,
- the student-facing link is not the same as the teacher's file,
- student evidence lands somewhere no one reviews,
- too many people can edit,
- old versions remain in the active folder,
- no one knows when the workflow is maintained.

## Classroom Scenario

Priya teaches an eighth-grade design challenge. Students open a directions Doc, submit a planning Form, and upload photos of prototypes. Priya knows the routine, but the folder contains last semester's planning Form and two versions of the photo upload instructions.

Instead of fixing everything immediately, Priya audits the workflow first.

## Model Artifact or Example

| Audit field | Model entry |
| --- | --- |
| Workflow name | Prototype planning and photo submission |
| Teaching purpose | Students plan a prototype, submit the plan, then upload two progress photos for feedback. |
| Current student-facing files | `Design Challenge - Planning Directions - Current`; `Prototype Photo Upload Instructions - Current` |
| Evidence location | Planning Form responses in `Prototype Plans - Responses`; photos in assignment folder. |
| Owner | Priya owns directions and Form; teaching assistant can view responses. |
| Access | Students can view directions, respond to Form, and upload through assignment. Students cannot edit teacher files. |
| Status rating | Emerging system |
| Risk found | Last semester's planning Form is still in the active folder and has a similar name. |
| First improvement | Move old Form to archive and add `Archive 2025` to the file name. |
| Maintenance trigger | Review after the first prototype checkpoint. |

## Common Mistake

The common mistake is writing the ideal workflow instead of the current workflow. An audit is useful because it tells the truth. If a link is confusing, write that. If ownership is unclear, write that. If old files are mixed with current files, write that.

Another mistake is auditing too much at once. One strong workflow audit is better than a vague map of the entire school year.

## Verification Checklist

- At least three audit rows are complete.
- Each row names a current file, link, folder, Form, Sheet, or workflow step.
- Student-facing access is separated from teacher-only access.
- Evidence location is specific enough to find.
- Each row includes one risk or uncertainty.
- The first improvement is small, visible, and connected to the risk.
- No private student information is copied into the audit.

## Reflection / Handoff

Choose one audit row and ask:

> Could another teacher use this workflow for one day without asking me where the current material lives?

If the answer is no, revise the row until the missing information is visible.
