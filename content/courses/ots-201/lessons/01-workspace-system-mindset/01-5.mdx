# Checkpoint Review

## Why This Matters

A checkpoint is where you decide whether the artifact is useful enough to guide the next chapter. For OTS-201, that means your Workspace workflow audit should help you make a real workspace decision, not merely prove that you filled in a table.

A workspace system works when another educator can inspect the workflow and understand what happens before, during, and after students use the materials.

## What You'll Learn

- Review your audit for clarity, safety, and usefulness.
- Decide which rows are ready for Drive architecture work.
- Identify rows that still depend on personal memory.
- Choose the first workspace change for Chapter 02.

## Core Idea

The audit is ready when it can answer a handoff question:

> If another teacher had to use this workflow tomorrow, could they find the current material, give students the right access, collect the evidence, and know what to update afterward?

If the answer is no, the audit has not failed. It has found the next improvement.

## Classroom Scenario

Sam audits a workflow for weekly reading responses. The audit names the student Form, the response Sheet, and the teacher who reviews responses. During checkpoint review, Sam notices that the Form is current but the response Sheet has no clear owner. Two teachers assume the other one reviews the responses.

That row is not ready for the next chapter yet. The fix is not a new folder. The fix is an ownership decision.

## Model Artifact or Example

| Audit row | Checkpoint result | Evidence | Next action |
| --- | --- |
| Weekly reading response Form | Revise | Form is current, but response Sheet owner is missing. | Assign reviewer and add review day. |
| Lab directions Doc | Ready | Current Doc, view-only student link, and archive rule are named. | Use this row for Drive folder planning. |
| Substitute lesson folder | Clarify | Folder exists, but student-facing and teacher-only files are mixed. | Separate access levels before folder redesign. |

## Review The System

Review every row in your **Workspace workflow audit**. Mark each row:

- **Ready**: clear enough to guide Chapter 02 folder planning.
- **Revise**: useful, but missing a file, owner, access rule, evidence location, or maintenance trigger.
- **Clarify**: too broad or dependent on personal memory.
- **Archive**: not worth systematizing right now.

## Common Mistake

The common mistake is marking every row ready because the audit is complete. Completion is not readiness. A row is ready only when the workflow can be followed without private explanation.

Another mistake is treating every problem as a Drive problem. Some rows need a naming rule, access decision, Form cleanup, response review routine, or archive rule before folder architecture matters.

## Verification Checklist

- Every audit row has a checkpoint status.
- Ready rows name current files or links, evidence location, owner, access, and maintenance trigger.
- Revise rows include a specific missing decision.
- Access concerns are named before the workflow is shared more broadly.
- No row requires student-identifiable data to explain the workflow.
- At least one row is selected for Chapter 02 Drive architecture work.
