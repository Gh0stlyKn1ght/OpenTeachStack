# Chapter Plan

## Why This Matters

Drive feels simple until a teacher has three copies of the same unit, a student-facing link pointing to the wrong file, and last year's answer key sitting beside the current directions.

A workspace system works when current materials, private planning files, shared team resources, student-facing copies, evidence, and archives each have a clear place.

This chapter turns Drive from a storage pile into a map of teaching responsibilities.

## What You'll Learn

- Name folder roles before moving files.
- Separate current teaching materials from planning, evidence, student-facing exports, shared team resources, and archives.
- Use course, unit, and year folders so old and current work do not collide.
- Match permissions to real teaching responsibilities.
- Build a Drive architecture map another educator could follow.

## Core Idea

Do not start by cleaning everything. Start by assigning jobs to folders.

For OTS-201, a folder architecture should answer:

| Question | Folder decision |
| --- | --- |
| What is active now? | Current unit folder |
| What can students open? | Student-facing folder or published links |
| What is teacher-only? | Planning and assessment folders |
| What can the team reuse? | Shared team templates and common resources |
| What is no longer active? | Archive folder by year or unit |

## Classroom Scenario

Maya teaches robotics and has files in `Robotics`, `Robotics 2024`, `Copy of Robotics`, and a shared department folder. The current sensor unit works only because Maya remembers which files are safe to use.

Her Drive architecture map starts with folder roles:

| Folder role | Model folder |
| --- | --- |
| Course root | `Robotics - 2026` |
| Active unit | `Unit 03 - Sensors - ACTIVE` |
| Student-facing materials | `Student Links - Unit 03` |
| Teacher planning | `Teacher Planning - Unit 03` |
| Evidence | `Evidence - Unit 03` |
| Team shared resources | `Robotics Team Shared Templates` |
| Archive | `Archive - 2025 and Earlier` |

## Verification Checklist

- The current unit is visually obvious.
- Student-facing files are separated from teacher-only planning.
- Evidence has a named destination.
- Shared team resources are not mixed with private planning notes.
- Archive folders are labeled by year, unit, or term.
- A colleague could find the current material without asking which copy is real.

## Reflection

Which folder currently does too many jobs? That folder is the first one to split or relabel.
