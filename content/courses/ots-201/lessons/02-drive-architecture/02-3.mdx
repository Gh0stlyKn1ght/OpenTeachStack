# Archive and Version Patterns

## Why This Matters

Teachers need old material. They also need old material to stop pretending it is current. Archive and version rules protect the active workspace from old decks, outdated links, and answer keys from previous years.

A workspace system works when completed or outdated materials stay findable without competing with today's teaching path.

## Core Idea

Archive by decision, not by panic.

Before moving files, decide what counts as active, what gets revised, what closes, and what goes to archive.

## Model Archive Rules

| Situation | Rule |
| --- | --- |
| Unit is still being taught | Keep current files in the active unit folder. |
| Student link is no longer valid | Mark file or link `CLOSED` and remove it from active student links. |
| File needs improvement before reuse | Move or label as `REVISE` with a revision note. |
| Prior-year material is useful only for reference | Move to year archive. |
| Answer key or assessment evidence is complete | Keep in teacher-only evidence or assessment folder, not student-facing area. |

## Classroom Example

At the end of Unit 03, Maya makes these decisions:

| Item | Decision | Destination |
| --- | --- | --- |
| `Sensor Lab Directions - Student` | Keep for reuse, revise intro step | `Unit 03 - Sensors - CLOSED`, label `REVISE` |
| `Sensor Launch Slides` | Archive old deck after copying useful diagram | `Archive - 2026 Unit 03` |
| `Sensor Lab Exit Ticket Form` | Close to new responses | `Evidence - Unit 03` with status `Closed` |
| `Exit Ticket Responses` | Keep teacher-only evidence | `Evidence - Unit 03` |
| Old `Robotics 2025 Sensors` folder | Reference only | `Archive - 2025 and Earlier` |

## Verification Checklist

- Active materials and archives are visually distinct.
- Closed student links are removed from active student-facing locations.
- Revision-needed files say what must change.
- Prior-year folders do not sit beside current folders without labels.
- Evidence and answer keys remain teacher-only.
- Archive rules can be repeated at the end of the next unit.

## Common Mistake

A common mistake in **Archive and Version Patterns** is treating the visible file or activity as proof that the teaching system is ready. The reviewer still needs to see the classroom purpose, the artifact evidence, the privacy or safety boundary, and the next revision trigger.

For this lesson, do not mark **Drive architecture map** ready until another educator can explain what changed, why it matters for **Drive Architecture**, and what would cause the work to be revised.
