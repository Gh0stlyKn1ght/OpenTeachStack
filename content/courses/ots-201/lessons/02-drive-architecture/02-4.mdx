# Drive Architecture Map

## Why This Matters

The Drive architecture map is the artifact that turns folder choices into a system another educator can inspect. It shows where materials live, who can access them, and what happens when the unit ends.

A workspace system works when the map makes current, private, shared, student-facing, evidence, and archive locations visible.

## Core Idea

Build a map that answers real teaching questions.

The goal is not a perfect folder tree. The goal is a Drive structure that reduces searching, permission mistakes, and version confusion.

## Completed Model

```text
Robotics - 2026
  00 Course Admin
    Course calendar
    Syllabus copy
  01 Team Shared Templates
    Lab directions template
    Reflection Form template
  Unit 03 - Sensors - ACTIVE
    Student Links - Unit 03
      Sensor Lab Directions - Student
      Sensor Launch Slides
      Sensor Lab Exit Ticket - Unit 03
    Teacher Planning - Unit 03
      Sensor lab teacher notes
      Threshold misconception warm-up
    Evidence - Unit 03
      Sensor Lab Exit Ticket Responses - Spring 2026
      Unit 03 tracker
  Archive - 2025 and Earlier
```

Map notes:

| Folder | Role | Access | Closeout |
| --- | --- | --- | --- |
| `Student Links - Unit 03` | Current student-facing materials | Students view or submit only | Remove closed links after unit |
| `Teacher Planning - Unit 03` | Teacher notes and answer keys | Teacher edit; teammate comment | Move useful revisions into next unit plan |
| `Evidence - Unit 03` | Response Sheets and tracker | Teacher-only or team-comment | Keep after unit; never share broadly |
| `Archive - 2025 and Earlier` | Reference material | Teacher/team access | Not active teaching |

Then add a small folder tree:

```text
Course root
  Course admin
  Shared templates
  Current unit
    Student-facing materials
    Teacher planning
    Evidence
  Archive
```

## Peer Review

Ask a colleague to answer these questions from the map:

- Which folder is active right now?
- Which files can students open?
- Where are teacher-only notes?
- Where does evidence land?
- Where do old versions go?

Revise the first answer they cannot find quickly.

## Verification Checklist

- The active unit is obvious.
- Student-facing files are separated from teacher-only files.
- Evidence has a protected location.
- Shared templates are separated from active copies.
- Archives are labeled by year, term, or unit.
- Permissions and closeout rules are part of the map, not hidden in memory.
