# Checkpoint Review

## Why This Matters

A Drive map is ready only if it helps during a realistic teaching moment. If another teacher still has to ask which file is current, which folder students can see, or where old versions go, the map needs revision.

A workspace system works when the Drive architecture reduces searching and permission risk during a busy week.

## Core Idea

Run a findability and access test.

Do not judge the map by whether it looks tidy. Judge it by whether it answers classroom questions quickly.

## Walkthrough Test

Use your **Drive architecture map** to answer:

| Test | Evidence from your map | Status |
| --- | --- | --- |
| Find the current unit folder. |  |  |
| Find the student-facing directions. |  |  |
| Find the teacher-only planning notes. |  |  |
| Find where response evidence belongs. |  |  |
| Find where an old slide deck should go. |  |  |
| Identify who owns permission review. |  |  |

Use statuses:

| Status | Meaning |
| --- | --- |
| Ready | Another educator could follow the map now. |
| Revise | The map exists but needs a clearer label, access note, or closeout rule. |
| Missing | The folder, role, or rule is not mapped yet. |
| Risky | The map could expose private material or confuse current and old files. |

## Permission Scenario

Test one risk:

> A student says they cannot open the lab directions. You find the file in Drive. What do you check before changing sharing?

Strong answer:

- Confirm the file is the current student-facing copy.
- Confirm it is not the teacher planning copy.
- Confirm students need view access, not edit access.
- Confirm the classroom post or command center points to the right link.
- Record the fix in the revision or maintenance note if the link was wrong.

## Completion Standard

Your Drive map is complete when it includes:

- course root,
- active unit folder,
- student-facing location,
- teacher planning location,
- evidence location,
- shared templates location,
- archive location,
- permission notes,
- closeout rules.
