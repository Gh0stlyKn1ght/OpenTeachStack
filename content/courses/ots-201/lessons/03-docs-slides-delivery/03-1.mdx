# Docs as Directions and Lesson Templates

## Why This Matters

In this chapter segment, the Doc is the “source of truth” for a lesson. If a substitute opens the lesson, they should not have to guess a sequence.

For the student-facing robotics lesson, the Doc must answer:

- What is required for each student group before start?
- Where do they find each link they need this week?
- What is the exact evidence format expected?
- What do students do if they get stuck?

## Core Idea

Your template should split content into:

- **Student-facing instructions** (kept visible)
- **Teacher-only execution notes** (kept in a collapsible section)
- **Version and access metadata** (kept at the top)

## Concrete Template Draft

Use this structure for your reusable direction Doc:

1. Header: lesson title, version ID (example `v1.1`), owner, backup owner, reviewed date
2. Learning target and materials
3. Step-by-step procedure with minute targets
4. Evidence requirements and exact submission path
5. Exit reflection prompt
6. Troubleshooting and support options
7. Teacher reset checklist

Robotics example:

`Distance Calibration Mini-lesson`

- Target: “I can collect five sensor readings and defend a stop-distance threshold.”
- Student artifact path: `Unit_03/Responses/Distance_Checks`
- Exit prompt: “What changed from my predicted threshold and why?”

## Workspace Move

Update your `Reusable Docs/Slides template set` with one concrete Doc template row.

| Template field | Robotics example |
| --- | --- |
| Lesson focus | Docs as reusable directions |
| File name | `Robot_Distance_Directions_v1.1` |
| Current students use | 3 readings, 1 team log row, 1 reflection statement |
| Link chain | Linked from unit tracker, command center, and Slide title slide |
| Permissions | Edit: teacher only; view: students + substitute; comment: lead teacher |
| Visibility note | Teacher-only troubleshooting block is folded by default |

## Common Mistake

Don’t let the Doc become a “beautiful note dump.” If any teacher or student needs verbal clarification for key step one, the Doc is not yet reusable.

Examples of failure:

- Two different versions of the exit ticket are still active.
- Materials list omits required setup gear.
- Evidence instructions mention “record your work” without naming folder path.

## Build Task

Create one full section inside the template:

### Student-facing “Evidence Submission” block

- Required file path
- Expected columns (`Team`, `Timestamp`, `Threshold`, `Justification`, `Anomaly`)
- Required number of rows
- Teacher sign-off field

## Quality Check

Ask a colleague to run this with a timer:

1. Can a substitute identify what students should do in the first 45 seconds?
2. Are submit paths and file names identical in Doc and Slide?
3. Can the colleague file one fake evidence sample without asking for clarification?
