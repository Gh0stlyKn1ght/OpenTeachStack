# Slides as Delivery Structure

## Why This Matters

Slides are most useful when they reduce cognitive load, not when they “look complete.” A deck should show movement:
launch → model → practice → evidence capture → closure.

## Core Idea

Use the deck as a **time and attention map** for the lesson. The Doc carries detail; the Slides carry sequence.

For the robotics lesson, each deck should have explicit timer cues and a one-sentence teacher action note on every key slide.

## Delivery Map

Build a reusable six-slide skeleton for each recurring class routine:

| Slide | Purpose | Teacher action |
| --- | --- | --- |
| 1 | Launch question + goal | Read target and open safety note |
| 2 | Materials and norms | Confirm all teams are matched to same gear |
| 3 | Procedure checkpoint | Start timer, circulate for misconceptions |
| 4 | Mid-class check | Prompt one sample sentence for threshold defense |
| 5 | Submission and support | Direct students to `Robot Directions` Doc and evidence sheet |
| 6 | Reflection + close | Record next-step revision note in 30 seconds |

## Concrete Example

Slide 5 is the most practical. In the robotics unit, it includes three hard links:

- student directions Doc
- response Sheet
- team support path (`/safety/quick-reset`)

The speaker note on slide 5 states:  
“Before students exit, verify every team has one row and one justification sentence.”

## Common Mistake

Common failure: slide decks that include all procedure detail and still break when questions change.

If the deck is not tied to one Doc section map, the classroom loses continuity and you lose evidence continuity.

## Quality Check

- Can a substitute launch from slide 1 and complete the full flow without opening a second document?
- Are all student-facing links on slides active and current to version `v1.1`?
- Is each slide's student action legible in a single glance?
