# Accessibility and Student-Facing Language

## Why This Matters

If students cannot interpret instructions in real time, a robust workflow still fails.  

In the robotics lesson, one unclear phrase can cause every team to stop at the same point, which turns a reusable design into a noisy routine.

## Core Idea

Use the same task text in both formats, but with role-appropriate detail:

- Doc: precision and policy for the teacher
- Slide: simple action commands for students

Keep vocabulary stable: if the Doc says “threshold,” the slide and worksheet must say “threshold” too.

## Accessibility Pass

For your template row, make these updates in the student-facing sections:

| Check | Concrete change |
| --- | --- |
| Instruction length | Replace long blocks with one action line plus one support line |
| Support visibility | Add one link labeled “Need help now?” for students |
| Reading load | Shorten key terms to one concept per sentence |
| Visual clarity | Add alt-text requirement for every image and icon |
| Audio support | Add note for oral script or captioning fallback |

Example:  
`Take one set of five distance measurements and record each in your sheet.`  
instead of  
`Take multiple observations from the ultrasonic distance sensor and determine the most robust threshold.`  

## Concrete Artifact Update

Create one language revision row for your set:

| Artifact field | Robotics lesson entry |
| --- | --- |
| Lesson focus | Accessibility and language consistency |
| Student-ready wording | 1) “Read target” 2) “Collect 5 readings” 3) “Choose stop distance” |
| Alt-text rule | Every image includes a 1-sentence purpose line |
| Help flow | `Need help now?` link on Doc and Slide 5 |
| Fallback path | `Printer backup` for students needing text-only steps |

## Common Mistake

Avoid separate vocab worlds for Doc and Slides. If students need translation by memory each time, the template is not reusable.

## Practice Task

Choose one section of your current Student Directions and apply a “first-readability” test:

- Can a student describe the success condition in one sentence?
- Can a substitute execute the same text without editing?
- Is there a visible fallback path for students who need support?

## Quality Check

Have another team member evaluate:

1. Can they complete the first action in under 60 seconds?
2. Can they locate the help link before a question is asked?
3. Do any terms shift between Doc, slide, and submission sheet?

## Reflection

What one phrase still causes confusion for students? Replace it now and update your template row.
