# Reusable Delivery Template

## Why This Matters

This section is the artifact build: a real template that can be duplicated into future classes without rewriting structure.

Use the robotics example to prove:

- doc and deck are versioned together
- evidence path is explicit
- access and reset tasks are assigned

## Core Idea

Create a template bundle with both teacher and student obligations visible from one source.

### Required bundle files

- `Robot_Sensor_Directions_Template_v1.x` (Doc)
- `Robot_Sensor_Launch_Deck_v1.x` (Slides)
- `Sensor_Response_Template` (Sheet)
- `Template_Check_Log` (tracker notes)

## Complete Template Build

Build your final artifact as a paired map:

| Doc section | Slide section | Classroom outcome |
| --- | --- | --- |
| Opening target | Launch slide | Students know what to produce in 10 seconds |
| Materials + setup | Safety and timing slide | No setup drift and no missing components |
| Step sequence | Work block slides | Students can move from task to task without verbal reset |
| Evidence instructions | Exit prompt slide | Student evidence lands in one shared Sheet location |
| Reflection prompt | Closing slide | Student responses are review-ready |
| Troubleshooting notes | N/A (Teacher notes) | Room stalls reduced and clearer support |

## Review Rule

To pass this checkpoint, demonstrate that a reviewer can answer:

1. Which doc is active for the week?
2. Which slide deck should launch with that doc?
3. Where is the artifact evidence saved?
4. Who archives old versions and when?

If any answer is missing, your pack stays at revise.
