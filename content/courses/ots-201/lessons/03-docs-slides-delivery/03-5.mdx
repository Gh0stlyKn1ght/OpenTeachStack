# Checkpoint Review

## Why This Matters

Checkpoint review confirms your template set is not just complete, but dependable under short planning windows.

This chapter should now prove that another educator can run the same teaching scenario with consistent student evidence and handoff clarity.

## Core Idea

Use a three-column review:

- **Function**: does the artifact let teaching continue?
- **Visibility**: can a colleague inspect the current state quickly?
- **Maintenance**: is revision routing clear and assigned?

## Run the Check

Run this 10-minute review against your current active bundle:

### Function

- Is there one active Doc link and one active deck link?
- Do both reference the same lesson target?
- Are evidence and support paths connected to both assets?

### Visibility

- Can a reviewer find the current version in under 30 seconds?
- Are student-facing instructions legible and complete?
- Is there a clear privacy boundary for who can edit?

### Maintenance

- Is there a named person and scheduled time to archive?
- Is there a status entry (`ready`, `revise`, `blocked`)?
- Is at least one known failure example recorded with fix action?
