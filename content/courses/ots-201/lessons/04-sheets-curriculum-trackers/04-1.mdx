# Rows, Columns, and Teacher Decisions

## Why This Matters

Inconsistent columns create inconsistent teaching decisions.

For the robotics tracker, one row should always be one lesson run, and each column should answer one decision.

## Core Idea

Design rows for reusable interpretation:

- Row = one lesson occurrence
- Column = one recurring teacher decision

For this chapter, avoid free-form narrative in the grid. Keep decisions in constrained fields.

## Recommended Tracker Columns

Use this exact starter header:

| Column | Purpose |
| --- | --- |
| Unit | Course period grouping |
| Lesson | Short title + sequence |
| Readiness status | Draft, Review, Active, Ready, Archived |
| Standard | Reference code and target statement |
| Evidence path | Exact Sheet/Doc/Form destination |
| Owner | Primary teacher + backup |
| Next review date | Date and time |
| Blocker | What prevents launch |
| Status note | One-line reason for current status |

### Robotics Example Row

| Unit | Lesson | Readiness status | Standard | Evidence path | Owner | Next review date | Blocker | Status note |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| Unit 03 | Distance calibration | Active | Data interpretation | Unit-03/Responses | Maya + Jordan | Friday 2:00 PM | None | Launch in 2 days; test threshold sample |

## Workspace Move

Build a row template with three locked rules:

1. No row is `Active` without a valid evidence path.
2. No row is `Ready` without owner + next review date.
3. No row is `Review needed` without a written blocker and owner.

## Common Mistake

Rows become too vague when “ready” and “review needed” are the only cues.

Use explicit blockers and next actions so another educator can execute without asking the original author.

## Quality Check

Ask one colleague to open your sheet and answer:

- Which lesson is active?
- What is its blocker?
- What must happen before teaching?

If they ask more than one clarifying question, fix your columns.

## Reflection

Which row in your tracker still hides a hard decision in comments or chat notes? Bring that decision into the grid.
