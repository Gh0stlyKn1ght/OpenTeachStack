# Tracking Standards and Lessons

## Why This Matters

Alignment is only useful if teachers can act on it.

For the robotics mini-unit, standards tracking must tell the teacher what evidence proves readiness, not just list labels.

## Core Idea

Use three linked fields:

1. Standard or outcome phrase
2. Lesson evidence target
3. Evidence location

If evidence is missing, status must reflect that immediately.

## Concrete Standard Mapping

Add three rows to your tracker with clear alignment language:

| Unit | Lesson | Standard | Evidence requirement | Evidence location | Status |
| --- | --- | --- | --- | --- | --- |
| Unit 03 | Distance setup | Measurements and uncertainty | 5 readings, 1 threshold decision sentence | `Unit03/Responses` | Active |
| Unit 03 | Threshold decision | Justification from data | Student justification entry + team reflection | `Unit03/Reflection` | Active |
| Unit 04 | Path planning | Data-informed design choices | Design sketch + chosen stopping rule | `Unit04/Artifacts` | Draft |

## Artifact Work

Create a “alignment note” field in your template with this formula:

- If evidence location is blank, status cannot be `Active` or `Ready`.
- If blocker is present, status cannot be `Ready`.
- If review date is missing, status must be `Review needed`.

## Common Mistake

Do not treat standards list as a compliance exercise.

A lesson is aligned only when evidence plus review condition appears in the row.

## Practice Task

For one active lesson, write:

- the learner-facing evidence command
- the exact scoring signal
- one short rationale for `Active` status

## Quality Check

Can a teammate identify:

- which standard this lesson supports
- where proof for it is stored
- what changes status from Review needed to Ready?

If yes, this section is ready.

## Reflection

Which alignment field is still "best guess"? Tighten it to one verifiable evidence phrase.
