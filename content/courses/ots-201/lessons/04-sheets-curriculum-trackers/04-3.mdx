# Tracking Resources and Assessment Evidence

## Why This Matters

In robotics, resources change quickly: a broken link or inaccessible sheet can block a lesson before it starts.

The tracker should make those dependencies visible before class.

## Core Idea

Track three things per lesson run:

- required materials
- assessment evidence destination
- license/privacy/access condition

## Practical Tracking Pattern

For each lesson row, add these columns:

| Column | Why it matters |
| --- | --- |
| Resource links | Must be versioned and tested |
| Evidence link | Shows exactly where student work is stored |
| Access rule | Who can view/edit/submit |
| Reliability check | `good`, `needs test`, `blocked` |

### Robotics Example

| Lesson | Resource links | Evidence link | Access rule | Reliability check |
| --- | --- | --- | --- | --- |
| Distance setup | Directions Doc v1.1, Launch deck v1.1 | `Unit03/Responses` | Students view, teacher edit | good |
| Threshold decision | Threshold rubric v1.0, sample sheet v1.0 | `Unit03/Reflection` | Students edit own team row | good |
| Path planning | Path sketch guide v0.9 | Not yet linked | pending backup | needs test |

## Workspace Move

Add one row per lesson where at least one link is stale or unverified and set status to `Review needed` with owner.

Example artifact note:

`Unit 04 Path planning is blocked because path guide URL is pending; fallback to sample route until updated.`

## Common Mistake

Treating links as stable just because they “worked last week.”

Your tracker should capture verification date for links that students or substitute teachers depend on.

## Quality Check

Run a 60-second dependency sweep:

1. Pick one active lesson.
2. Open each link in the tracker.
3. Verify that resource and evidence links are accessible by student role and teacher role.

## Reflection

Which lesson currently has the riskiest evidence path? Move it to `Review needed` with a named owner.
