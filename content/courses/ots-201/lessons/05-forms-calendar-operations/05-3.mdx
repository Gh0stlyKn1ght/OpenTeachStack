# Calendar as Pacing and Maintenance

## Why This Matters

Calendar is often treated as a place to put due dates. In a workspace system, Calendar also protects the teacher actions that make the workflow reliable: reviewing responses, returning feedback, checking links, and archiving old materials.

If review and maintenance do not have time on the calendar, they become invisible labor. Invisible labor is the first thing to disappear during a busy week.

A workspace system works when the calendar shows when teaching action happens after information is collected.

## Core Idea

Schedule the operational rhythm, not only the class deadline.

For Forms and Calendar, the minimum rhythm is:

| Calendar event | Purpose |
| --- | --- |
| Launch | When students receive the Form or task. |
| Review | When the teacher checks responses. |
| Action | When feedback, grouping, reteaching, or logistics happen. |
| Closeout | When links, Forms, response Sheets, and folders are updated or archived. |

## Classroom Example

Maya's sensor lab has four calendar events, not one.

| Event | Calendar title | Notes |
| --- | --- |
| Launch | `Teach: Sensor Lab 1` | Link to student directions Doc and exit-ticket Form. |
| Review | `Review: Sensor Lab Exit Ticket` | Link to response Sheet. Sort for misconception and access issues. |
| Action | `Update: Sensor Warm-Up` | Revise Wednesday warm-up using response pattern. |
| Closeout | `Closeout: Unit 03 Sensor Files` | Close Form, archive old deck, update tracker status. |

This gives another teacher a visible path: teach, review, act, close out.

## Verification Checklist

- Calendar includes review time, not only student deadlines.
- Event titles name the operation clearly.
- Event notes link to the current file or response Sheet.
- The action event says what changes after review.
- The closeout event prevents stale links and old Forms from staying active.
- Backup visibility is appropriate for your team and school policy.

## Common Mistake

A common mistake in **Calendar as Pacing and Maintenance** is treating the visible file or activity as proof that the teaching system is ready. The reviewer still needs to see the classroom purpose, the artifact evidence, the privacy or safety boundary, and the next revision trigger.

For this lesson, do not mark **Operations workflow map** ready until another educator can explain what changed, why it matters for **Forms and Calendar for Classroom Operations**, and what would cause the work to be revised.
