# Operations Workflow Map

## Why This Matters

This chapter's artifact is the place where Forms, response Sheets, Calendar, and teacher action become one visible routine.

Without a map, operations hide in memory: "I know which Form to use," "I usually check responses later," or "I clean up at the end." Those phrases are not enough for a dependable workspace.

A workspace system works when the collection tool, evidence location, review time, teacher action, and closeout rule are visible in one place.

## Core Idea

The map should show the whole chain:

> trigger -> Form -> response Sheet -> review time -> teacher action -> closeout

If any part of that chain is missing, the workflow can collect information without improving teaching.

## Completed Model

Use this model as the standard for your artifact.

| Map field | Completed entry |
| --- | --- |
| Workflow name | Sensor lab exit ticket |

| Review event | Tuesday 3:10-3:25 PM, linked to response Sheet |
| Teacher action | Revise Wednesday warm-up; identify teams needing retest support |
| Owner | Maya reviews responses; teammate can comment |
| Access boundary | Students can submit Form; students cannot view response Sheet |
| Closeout rule | Close Form Friday, archive response Sheet after unit reflection, update Unit 03 tracker |

## Common Mistakes

| Mistake | Why it fails | Fix |
| --- | --- | --- |
| `Check responses later` | Later is not a review routine. | Add a Calendar event with a link to the response Sheet. |
| `Students submit the Form` | Submission is not the end of the workflow. | Name what the teacher does because of responses. |
| `Response Sheet` | Default names become hard to find. | Rename the Sheet with unit, purpose, and term. |
| `Everyone can edit` | Too much access can damage evidence. | Separate student submission access from teacher response access. |
| `Archive at some point` | Old links stay active. | Add a closeout event or tracker status. |

## Peer Review

Ask a colleague to inspect your map for five minutes. They should answer:

- What starts the workflow?
- Where does the evidence land?
- When does the teacher review it?
- What action follows review?
- How does the workflow close?

Revise the first field they cannot answer without your explanation.

## Verification Checklist

- The map includes trigger, collection, evidence, review, action, and closeout.
- File and Sheet names are specific enough to find.
- Student access and teacher access are separated.
- Calendar events include links or notes.
- The teacher action is visible and realistic.
- The closeout rule prevents stale Forms, links, and response Sheets.
