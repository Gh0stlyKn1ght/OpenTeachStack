# Checkpoint Review

## Why This Matters

A workflow can look complete on paper and still fail when responses arrive. This checkpoint tests whether your operations map creates action instead of another unread response Sheet.

A workspace system works when another educator can open the map, follow the links, review the evidence, and know the next teacher move.

## Core Idea

Test the workflow with sample responses before relying on it with students.

The question is not "Did I make a Form?" The question is:

> If responses arrive today, what happens next and when?

## Sample Response Test

Use this sample set to test Maya's sensor exit ticket.

| Team | Threshold choice | Evidence sentence | Next test |
| --- | --- | --- | --- |
| A | 20 cm | Most readings were near 20 cm. | Test with the robot moving faster. |
| B | 50 cm | Bigger number is safer. | Try the code again. |
| C | 19 cm | 18, 19, 20, 21, and 22 are close, so 19 is a good first test. | Retest near the wall. |
| D | 5 cm | The robot should stop close to the wall. | Not sure. |

The map should tell the teacher what to do:

| Pattern found | Action |
| --- | --- |
| Teams A and C use the readings. | They can move to the next lab challenge. |
| Teams B and D choose thresholds without using the data range. | Wednesday warm-up should compare threshold choices against data. |
| Team D is unsure about the next test. | Teacher checks in during the first build block. |

If the map does not reveal those actions, the Form questions or review plan need revision.

## Access and Safety Check

Before marking the map ready, answer these checks:

| Check | Ready? |
| --- | --- |
| Students can submit the Form without editing the response Sheet. |  |
| The response Sheet is stored in the correct teacher folder. |  |
| The Form does not ask for unnecessary private information. |  |
| A Calendar review event exists or is ready to create. |  |
| The event notes point to the current response Sheet. |  |
| The closeout rule says when the Form closes or archives. |  |

## Revision Decision

Choose one status:

| Status | Meaning |
| --- | --- |
| Ready to use | Another educator could run the workflow from the map. |
| Needs one revision | One field is unclear, missing, or too dependent on memory. |
| Not ready | The workflow collects responses but does not yet create action. |

Write the revision if needed:

> Before I use this workflow, I need to...
