# Command Center Requirements

## Why This Matters

A command center fails when it becomes a dumping ground. If every folder, file, reminder, and idea gets pasted into one page, the teacher has simply moved the clutter into a new location.

A workspace system works when the command center shows the few links and status signals a teacher needs to act this week.

## Core Idea

Use requirements before links.

A requirement says what the command center must help the teacher do. A link is added only when it serves that requirement.

## Model Requirements

| Requirement | What it means | Model command-center element |
| --- | --- | --- |
| Find the current unit fast | The teacher should not search Drive during class. | `Current Unit 03 - Robotics Sensors` folder link |
| Share only active student links | Students should not receive old or teacher-only materials. | Student links table with status `Active` or `Closed` |
| Review evidence on time | Responses and trackers should lead to action. | Operations queue with review date and owner |
| Maintain the system | Links, permissions, and archives need routine checks. | Maintenance checklist with next check date |
| Record changes | Future teachers need to know why something changed. | Revision log with date, decision, owner, and reason |

## Classroom Example

Maya first writes this requirement:

> During Unit 03, I need one place to launch the current lesson, find the response Sheet, see the next follow-up action, and close old materials at the end of the week.

That requirement prevents the command center from becoming a file encyclopedia.

Her first layout:

| Command-center section | Link or status |
| --- | --- |
| Current unit | `Unit 03 - Robotics Sensors - Active Folder` |
| Today students use | `Sensor Lab Directions - Student`, `Sensor Lab Exit Ticket - Unit 03` |
| Evidence to review | `Sensor Lab Exit Ticket Responses - Spring 2026`; status `Review Tuesday` |
| Maintenance | Close Form Friday; archive old launch deck; update tracker |
| Revision log | `2026-03-12 - Changed stop-threshold prompt after exit-ticket review - Maya` |

## Verification Checklist

- Each requirement starts with a teacher action.
- Student-facing links are clearly separated from teacher-only links.
- The command center does not duplicate full content from linked files.
- Status labels show what is active, ready, blocked, closed, or needs review.
- Maintenance and revision requirements are included.
- Private or sensitive materials are not made easier to expose.

## Common Mistake

A common mistake in **Command Center Requirements** is treating the visible file or activity as proof that the teaching system is ready. The reviewer still needs to see the classroom purpose, the artifact evidence, the privacy or safety boundary, and the next revision trigger.

For this lesson, do not mark **Workspace command center** ready until another educator can explain what changed, why it matters for **Workspace Command Center**, and what would cause the work to be revised.
