# Maintenance Checklist

## Why This Matters

Even a strong workspace decays. Links go stale, Forms stay open, old versions look current, permissions drift, and response Sheets pile up after the lesson is over.

A workspace system works when maintenance tasks are visible before they become emergencies.

## Core Idea

Put maintenance where teaching happens.

The command center should show a small checklist by time horizon: daily, weekly, unit-end, and term-end. That keeps maintenance realistic instead of turning it into a giant cleanup project.

## Model Checklist

| Rhythm | Task | Owner | Evidence |
| --- | --- | --- | --- |
| Daily | Check that today's student links open correctly. | Maya | Active links table shows `Checked today`. |
| Daily | Confirm response Sheet has new submissions after the exit ticket. | Maya | Operations queue status changes to `Review needed`. |
| Weekly | Review permissions on active unit folder. | Maya | Folder note says `Team can comment; students view only`. |
| Weekly | Archive closed Forms and old launch decks. | Maya | Archive folder has dated copies. |
| Unit-end | Update unit tracker with evidence and revision notes. | Maya | Tracker status changes to `Taught - revise warm-up`. |
| Term-end | Copy useful templates into next-term planning folder. | Maya and teammate | Revision log records template decision. |

## Classroom Example

After teaching the sensor lab, Maya sees this maintenance queue:

| Item | Status | Next action |
| --- | --- | --- |
| Sensor Lab Exit Ticket Form | Active | Close Friday after final reflection. |
| Response Sheet | Review needed | Sort for threshold misconception before Wednesday. |
| Launch Slides | Revision needed | Archive old deck and keep current deck linked. |
| Unit Tracker | Update needed | Add evidence note after exit-ticket review. |

This is useful because it says what to do next. It is not a vague reminder to "clean up Drive."

## Verification Checklist

- The checklist is short enough to use.
- Each task has an owner.
- The next date or trigger is visible.
- Closed Forms, old decks, and outdated links have archive rules.
- Permission review is included.
- The live queue names the next action, not just the problem.

## Reflection

Which maintenance task has caused confusion before? Put that task into the live queue with a date or trigger.
