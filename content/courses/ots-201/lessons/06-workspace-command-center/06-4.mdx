# Workspace Command Center

## Why This Matters

This is the final build for OTS-201. The command center brings the course artifacts together so the teacher can run and maintain a real Workspace system.

A workspace system works when the command center shows the current teaching path, the evidence path, the operations path, and the maintenance path in one inspectable surface.

## Core Idea

Build the smallest command center that can survive a busy week.

It should not try to be a course website, gradebook, file cabinet, and archive all at once. It should link to those things when needed and show their current status.

## Completed Model

Use this model as the benchmark for your artifact.

| Section | Model content |
| --- | --- |
| Current unit | `Unit 03 - Robotics Sensors`; dates March 10-21; active folder linked |
| Today students use | `Sensor Lab Directions - Student`; `Sensor Launch Slides`; `Sensor Lab Exit Ticket - Unit 03` |
| Evidence and tracker | `Unit 03 Tracker`; `Sensor Lab Exit Ticket Responses - Spring 2026`; status `Review Tuesday` |
| Operations queue | Exit ticket review Tuesday; warm-up revision Wednesday; Form closeout Friday |
| Maintenance checklist | Check student links daily; review permissions weekly; archive old decks Friday |
| Revision log | `2026-03-12 - Updated threshold prompt after exit-ticket responses showed range confusion - Maya` |
| Access notes | Students view directions/slides and submit Form; response Sheet is teacher-only; teammate can comment |

## Status Labels

Use simple labels so the command center tells you what needs action.

| Label | Meaning |
| --- | --- |
| Active | Use this with students now. |
| Ready | Prepared but not yet launched. |
| Review needed | Evidence exists and needs teacher action. |
| Revise | Keep, but update before reuse. |
| Closed | Do not use with students now. |
| Archived | Stored for reference, not active teaching. |
| Blocked | Cannot proceed until a named issue is resolved. |

## Build Check

Before calling the command center ready, run this five-minute test:

1. Find tomorrow's student-facing material.
2. Find the evidence you need to review.
3. Find the next maintenance task.
4. Find the last revision note.
5. Identify one link students should not receive.

## Verification Checklist

- Every active student link has a status.
- Teacher-only evidence is separated from student-facing materials.
- The operations queue includes review and follow-up actions.
- The maintenance checklist includes link, permission, archive, and tracker checks.
- The revision log explains why changes happened.
- The command center links to artifacts instead of duplicating everything.

## Reflection

What did the five-minute test reveal first: missing link, unclear status, risky access, forgotten evidence, or missing maintenance? Fix that issue before the checkpoint.
