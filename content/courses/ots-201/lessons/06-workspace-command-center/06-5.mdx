# Checkpoint Review

## Why This Matters

The command center is ready only if it can support a real teaching week. A polished page that cannot help you launch, review, revise, and close out instruction is not a workspace system yet.

A workspace system works when a teacher can move from today's lesson to evidence review to maintenance without hunting through Drive.

## Core Idea

Run a teaching-week walkthrough.

Pretend it is Wednesday morning. You have ten minutes before class. Can the command center tell you what to teach, what students need, what evidence needs review, and what must be maintained by Friday?

## Walkthrough Test

Use this checklist with your command center.

| Task | Evidence in command center | Status |
| --- | --- | --- |
| Start today's lesson | Current student directions and launch deck are linked. |  |
| Share the right student link | Active links are separated from closed or teacher-only links. |  |
| Review learning evidence | Tracker or response Sheet is linked with review date. |  |
| Take teacher action | Operations queue names the next follow-up. |  |
| Maintain the workspace | Checklist names link, permission, archive, and tracker tasks. |  |
| Explain a revision | Revision log says what changed, when, who changed it, and why. |  |

Use status labels:

| Status | Meaning |
| --- | --- |
| Ready | Another educator could use this section now. |
| Revise | The section exists but needs clearer labels, links, ownership, or status. |
| Missing | The section or evidence does not exist yet. |
| Risky | The section could expose private material, stale links, or wrong access. |

## Final Review Questions

Answer these before completing OTS-201:

| Question | Your answer |
| --- | --- |
| Which section is strongest? |  |
| Which section still depends on your memory? |  |
| Which student-facing link needs the clearest status label? |  |
| Which teacher-only evidence needs better access protection? |  |
| What is the next maintenance date? |  |
| What revision will you make before reuse? |  |

## Artifact Completion Standard

Your **Workspace command center** is complete when it includes:

- current unit and active folder,
- active student links with status,
- tracker and evidence links,
- operations queue,
- maintenance checklist,
- revision log,
- access notes,
- at least one peer-review revision.

## OTS-201 Handoff

This checkpoint completes OTS-201. Your handoff statement should name the system you built and the next maintenance action:

> My Workspace command center is ready for [unit/course] because another educator can find [current materials], review [evidence], and complete [maintenance action] without asking me to explain the folder.

If that sentence is not true yet, revise the section that blocks it.
