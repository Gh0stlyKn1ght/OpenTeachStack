# OTS-220 Examples

Use these examples as the technical artifact baseline for teacher automation.

- completed-artifact.md: fake-data Apps Script function with Logger output.
- weak-version.md: brittle script pattern missing error handling.
- strong-version.md: improved script with checks, error path, and rollback.
- reviewer-notes.md: safety and test evidence checklist.

Assumptions:
- All rows, sheet ids, and accounts are fake.
- No destructive operations are run from classroom examples.
