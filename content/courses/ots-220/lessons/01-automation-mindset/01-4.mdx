# Checkpoint Review

## Why This Matters

Automation is ready when a teacher can point to the decision, the classroom use, the evidence, and the next review step.

In **Automation Mindset**, the work sits inside teacher automation with Google Apps Script and Workspace files. The risk is automating a process before the trigger, input, output, owner, permission boundary, and failure path are clear. This lesson narrows that risk into one visible update to **Automation candidate list**.

## Core Idea

Run a checkpoint on the **Automation candidate list**. Mark what is ready, what needs revision, what is blocked, and what evidence supports that decision.

The goal is not to make a perfect document in one pass. The goal is to make the teacher decision clear enough that another educator can inspect it, reuse it, or challenge it without needing a private explanation.

## Lesson Focus

Because this is the checkpoint for automation mindset, the teacher reviews the artifact as a working system. The checkpoint looks for missing decisions, unclear ownership, weak evidence, and maintenance steps that would be forgotten during a busy week.

In 01.4, focus on checkpoint review decisions. Choose one concrete classroom material or teacher decision from this list: a spreadsheet row, folder ID, template Doc, script function, or email draft. Connect it to automation candidate list, then decide whether it needs to be created, revised, tested, credited, restricted, published, or archived.

Connect checkpoint review decisions back to automation mindset. That connection matters because a polished artifact can still fail if the lesson purpose, boundary, evidence, and maintenance routine are not visible.

## Classroom Example

Use this working situation: a teacher wants a script that creates unit folders, generates lesson Docs from Sheet rows, and sends reminder emails only after the teacher has reviewed the recipients.

## Quality Check

Before moving on, test the work against this standard: the script can be run on sample data, stopped safely, explained by another teacher, and repaired when an error appears.

A reviewer should be able to find the checkpoint review decisions decision, inspect the automation candidate list evidence, and understand how it supports automation mindset.

If the work only names a tool or a file, revise it. The artifact needs to show the teaching purpose, evidence, boundary, and maintenance routine.
