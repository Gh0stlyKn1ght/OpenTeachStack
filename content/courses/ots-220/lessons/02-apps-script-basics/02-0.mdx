# Chapter Plan

## Why This Matters

Automation is useful only when the teacher can explain exactly what it changes and why. In this chapter, each script should use safe sample data, produce inspectable output, and include a recovery path before touching live class materials.

## Core Idea

Start with one reusable automation case and design it before coding:

- Input format
- Function contract
- Expected output
- Failure path
- Owner and review cadence

## Working Scenario

You will build a safe sandbox automation for robotics planning:

- read one fake Sheet row
- create one sandbox folder
- create one setup doc
- write one compact log row to `First script notes`

Use this fake dataset:

| Field | Fake value |
| --- | --- |
| unitName | `Unit 03 - Sensor Lab` |
| sectionName | `Sample Robotics 8A` |
| ownerEmail | `sample-teacher@example.edu` |
| dueDate | `2026-09-15` |

## Artifact Row

Add one row in **First script notes** before you write code:

| Field | Required entry |
| --- | --- |
| Lesson focus | Chapter 02 initial planning |
| Input source | Sandbox row above |
| Expected result | One folder ID, one doc ID, status `sandbox-created` |
| Failure action | If input is invalid, stop before creating anything |
| Permission note | Script account only has sandbox Drive scope in this phase |
| Rollback note | Remove sandbox folder and doc before re-run |

## Setup and First Test

Use this minimal sample function as your first scaffold:

```javascript
function getSampleUnitRow() {
  return {
    unitName: "Unit 03 - Sensor Lab",
    sectionName: "Sample Robotics 8A",
    ownerEmail: "sample-teacher@example.edu",
    dueDate: "2026-09-15",
  };
}

function logSampleSetupRow() {
  const row = getSampleUnitRow();
  Logger.log(JSON.stringify({ level: "info", ...row, status: "sandbox-ready" }));
  return row;
}
```

## Expected Output

- Execution log shows `level: "info"` and `status: "sandbox-ready"` with all four fields.
- No live course folder IDs are used at this step.

## Failure Handling

If the log does not show all expected fields:

- stop the flow
- correct the object keys in one place
- rerun only this section
- keep the artifact note updated as `revise`

If permission messages appear:

- stop and confirm you are using a sandbox folder id
- keep the test row unchanged until the folder permission is fixed

## Quality Check

Before moving on, verify:

- you can describe the function contract from memory
- you have written the rollback step in `First script notes`
- you have a defined failure condition and owner
