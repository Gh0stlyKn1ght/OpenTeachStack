# Script Editor and Bound Scripts

## Why This Matters

Bound scripts should be attached only after the automation goal is clearly bounded. If the decision logic is outside the spreadsheet, you should not ask a beginner course to run it.

## Core Idea

Use one bound script per active lesson workflow, and confirm it can be run from the spreadsheet without guessing where context comes from.

In this section, your script stays sandbox-safe:

- no real student identifiers
- no real class folders
- no production email sends

## Setup

In Google Sheets, open **Extensions → Apps Script** from a dedicated sandbox spreadsheet and create:

1. `onOpen()` that adds a custom menu
2. `runSandboxUnitSetup()` that reads one row from a clearly named range
3. `buildUnitSummaryFromRow()` that validates required fields

## Concrete Code

```javascript
function onOpen() {
  SpreadsheetApp.getUi().createMenu("Teacher Toolkit").addItem("Run sandbox unit setup", "runSandboxUnitSetup").addToUi();
}

function buildUnitSummaryFromRow(row) {
  const required = ["unitName", "sectionName", "ownerEmail", "dueDate"];
  const missing = required.filter((key) => !row[key]);
  if (missing.length > 0) {
    throw new Error(`Missing required fields: ${missing.join(", ")}`);
  }

  return {
    unitName: row.unitName.trim(),
    sectionName: row.sectionName.trim(),
    ownerEmail: row.ownerEmail.trim().toLowerCase(),
    dueDate: row.dueDate,
    status: "sandbox-validated",
  };
}

function runSandboxUnitSetup() {
  const sample = {
    unitName: "Unit 03 - Sensor Lab",
    sectionName: "Sample Robotics 8A",
    ownerEmail: "sample-teacher@example.edu",
    dueDate: "2026-09-15",
  };
  const result = buildUnitSummaryFromRow(sample);
  Logger.log(JSON.stringify(result));
  return result;
}
```

## Permissions/Access Note

For this step, students only need read access to the file. Teacher role runs the script with Edit rights.

## Expected Output

`Logger.log` should print:

- `unitName: "Unit 03 - Sensor Lab"`
- `sectionName: "Sample Robotics 8A"`
- `status: "sandbox-validated"`

## Failure Path

- if a required field is blank, the script throws a clear “Missing required fields” error
- fix the source object, then rerun only `runSandboxUnitSetup`
- log remains bounded to sandbox data

## Quality Check

Verify that a substitute can:

1. open the spreadsheet
2. choose `Teacher Toolkit > Run sandbox unit setup`
3. read output from logs and confirm required fields are present
