# Functions, Logs, and Errors

## Why This Matters

Functions hide complexity only if they are explicit about inputs and outputs. Logs make outcomes visible. Error handling protects student-facing systems.

For this section, the function should create one sandbox folder and one setup doc from one row, and fail clearly when any contract check fails.

## Core Idea

Build three layers in one routine:

- `buildUnitRequest` for input normalization and validation
- `createUnitArtifacts` for object output
- `createUnitArtifactsSafe` for guardrails and clear error logging

## Concrete Code

```javascript
const SANDBOX_ROOT_ID = "FOLDER_ROOT_SANDBOX_ID";

function buildUnitRequest(values) {
  const required = ["unitName", "sectionName", "ownerEmail", "dueDate"];
  const missing = required.filter((field) => !values[field]);
  if (missing.length) {
    throw new Error(`Missing required fields: ${missing.join(", ")}`);
  }
  return {
    unitName: values.unitName.toString().trim(),
    sectionName: values.sectionName.toString().trim(),
    ownerEmail: values.ownerEmail.toString().trim().toLowerCase(),
    dueDate: values.dueDate.toString().trim(),
  };
}

function createUnitArtifacts(request) {
  const root = DriveApp.getFolderById(SANDBOX_ROOT_ID);
  const unitFolder = root.createFolder(`${request.unitName} - ${request.sectionName}`);
  const doc = DocumentApp.create(`Setup notes - ${request.unitName}`);

  const result = {
    level: "info",
    unitName: request.unitName,
    sectionName: request.sectionName,
    folderId: unitFolder.getId(),
    docId: doc.getId(),
    message: "Sandbox setup created",
  };

  Logger.log(JSON.stringify(result));
  return result;
}

function createUnitArtifactsSafe(values) {
  try {
    const request = buildUnitRequest(values);
    return createUnitArtifacts(request);
  } catch (error) {
    const failure = {
      level: "error",
      message: error.message,
      unitName: values.unitName || "",
      sectionName: values.sectionName || "",
    };
    Logger.log(JSON.stringify(failure));
    throw error;
  }
}
```

## Expected Output

```json
{
  "level": "info",
  "unitName": "Unit 03 - Sensor Lab",
  "sectionName": "Sample Robotics 8A",
  "folderId": "sandbox-folder-id",
  "docId": "sandbox-doc-id",
  "message": "Sandbox setup created"
}
```

## Failure Example

Input without `sectionName`:

```
Error: Missing required fields: sectionName
```

Recovery:

1. Pause script
2. update row field
3. rerun with sandbox data only
4. confirm one new folder and one new doc

## Permissions / Rollback

- Required: Drive editor rights for `FOLDER_ROOT_SANDBOX_ID`; Docs create permission
- Rollback: delete sandbox folder and setup doc after each failed attempt
- Privacy: no student emails, names, or grades in Logger payload
