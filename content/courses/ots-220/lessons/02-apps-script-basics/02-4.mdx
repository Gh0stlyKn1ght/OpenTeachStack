# Workshop: First Custom Menu

## Why This Matters

Custom menus reduce accidental execution by making the action explicit and reviewable. In a class context, this is a practical safety win.

## Core Idea

Build one menu item that triggers the already safe sandbox routine from an explicit button and logs the action.

## Workshop Build

```javascript
function onOpen() {
  SpreadsheetApp.getUi()
    .createMenu("Teacher Automation")
    .addItem("Create sandbox unit workspace", "runSandboxMenuFlow")
    .addToUi();
}

function getSampleSandboxRow() {
  return {
    unitName: "Unit 03 - Sensor Lab",
    sectionName: "Sample Robotics 8A",
    ownerEmail: "sample-teacher@example.edu",
    dueDate: "2026-09-15",
  };
}

function runSandboxMenuFlow() {
  runAuthorizationCheck(); // from 02.3
  const row = getSampleSandboxRow();
  const result = createUnitArtifactsSafe(row);
  Logger.log(JSON.stringify({ level: "info", action: "menu-run", ...result }));
  return result;
}
```

## Setup Notes

- Open spreadsheet as sandbox editor.
- Reload the sheet to see the new menu.
- Click **Teacher Automation → Create sandbox unit workspace**.

## Expected Result

- `Teacher Automation` menu appears.
- Menu action writes one folder and one document in sandbox.
- Logger includes `action: "menu-run"` and pass-through IDs.
- No email sending and no real students impacted.

## Failure and Recovery

If menu is missing:

- confirm `onOpen` exists in this bound script project
- reload spreadsheet and try once more

If folder action fails:

- validate `runAuthorizationCheck` status
- fix environment settings
- rerun after clearing failed outputs

## Permissions Reminder

This custom menu step is safe only when:

- only sandbox folder IDs are configured
- no live mailing or grade data is used in the script
- the acting teacher account owns the script test environment

## Quality Check

Run three checks:

1. Menu appears without errors.
2. Logger shows a complete success payload.
3. A failed run does not create a partial production artifact.
