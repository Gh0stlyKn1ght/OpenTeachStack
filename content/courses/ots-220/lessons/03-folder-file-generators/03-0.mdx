# Chapter Plan

## Why This Matters

Folder generation is one of the highest-risk automations for teachers because it touches file systems and can create hidden duplicates. The classroom risk is reduced when folder naming, owner, and rollback logic are explicit before any student work is touched.

## Core Idea

Treat this chapter as a contract between one lesson and one folder structure:

- define a predictable folder path
- define what inputs create it
- define the expected output ids
- define the stop condition when output is invalid

## Classroom Situation

A robotics teacher wants a repeatable setup for each mini-unit:

- one unit folder
- one subfolder for student evidence
- one planning doc linked in a tracker row

At all times, a single lesson run must produce only the expected set of sandbox artifacts.
