# Naming and Idempotency

## Why This Matters

Automation is ready when a teacher can point to the decision, the classroom use, the evidence, and the next review step.

In **Folder and File Generators**, the work sits inside teacher automation with Google Apps Script and Workspace files. The risk is automating a process before the trigger, input, output, owner, permission boundary, and failure path are clear. This lesson narrows that risk into one visible update to **Folder generator**.

## Core Idea

Turn the lesson idea into an inspectable decision. Name the classroom use, the evidence, the review step, and the next revision trigger.

The goal is not to make a perfect document in one pass. The goal is to make the teacher decision clear enough that another educator can inspect it, reuse it, or challenge it without needing a private explanation.

## Lesson Focus

Because this is a focused section, the teacher isolates the naming idempotency decision from the rest of the course. The lesson asks what this idea changes, where it appears, and how the teacher will know it worked.

In 03.2, focus on naming idempotency. Choose one concrete classroom material or teacher decision from this list: a spreadsheet row, folder ID, template Doc, script function, or email draft. Connect it to folder generator, then decide whether it needs to be created, revised, tested, credited, restricted, published, or archived.

Connect naming idempotency back to folder file generators. That connection matters because a polished artifact can still fail if the lesson purpose, boundary, evidence, and maintenance routine are not visible.
