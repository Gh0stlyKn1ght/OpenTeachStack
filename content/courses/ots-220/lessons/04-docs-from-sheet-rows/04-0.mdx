# Chapter Plan

## Why This Matters

When each lesson is generated from one clean row, the teacher reduces duplicate work and keeps evidence consistent. The risk is using mixed column names, wrong placeholders, or a wrong template file.

## Core Idea

Set up one reliable mapping:

- spreadsheet columns = placeholders
- template markers = expected variables
- generated docs = reviewed output

## Classroom Situation

For a robotics mini-unit, the team wants one setup doc per student group automatically generated from a Sheet row with:

- `unitName`
- `learningTarget`
- `evidencePrompt`
- `sectionName`

The script should produce predictable doc titles and content without changing live student folders.

## Required Artifact Entry

Before coding, add one row in **Doc generator** notes:

| Field | Entry |
| --- | --- |
| Lesson focus | Sheet-driven doc generation |
| Input source | sandbox Sheet row + sandbox template |
| Expected output | one doc copy with mapped fields and status row |
| Safety boundary | no live folders, no real student names |
| Failure signal | missing field or placeholder mismatch throws explicit error |
| Rollback | delete generated doc and rerun with fixed row |
