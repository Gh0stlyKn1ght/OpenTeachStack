# Sheet Data as Source

## Why This Matters

If sheet columns and script field names drift, docs become wrong but still look "working." This section makes source-of-truth explicit.

## Core Idea

One row = one generated doc.

Use explicit required headers and validate before any write occurs.

## Concrete Row and Field Map

| Source header | Script field | Example value |
| --- | --- | --- |
| `unitName` | `unitName` | `Unit 03 - Sensor Lab` |
| `learningTarget` | `learningTarget` | `Interpret sensor data and defend a threshold` |
| `evidencePrompt` | `evidencePrompt` | `Record 5 values and highlight one outlier` |

## Implementation

```javascript
function getSanitizedSourceRow() {
  return {
    unitName: "Unit 03 - Sensor Lab",
    learningTarget: "Interpret sensor data and defend a threshold",
    evidencePrompt: "Record 5 values and highlight one outlier",
    sectionName: "Sample Robotics 8A",
  };
}

function validateSourceRow(row) {
  const required = ["unitName", "learningTarget", "evidencePrompt", "sectionName"];
  const missing = required.filter((field) => !row[field] || !row[field].toString().trim());
  if (missing.length) {
    throw new Error(`Missing required field(s): ${missing.join(", ")}`);
  }
  return row;
}
```

## Expected Failure and Repair

Failure:

`Error: Missing required field(s): sectionName`

Repair:

1. Fix the Sheet header or row value.
2. rerun validation only.
3. confirm no doc created before rerun.

## Quality Check

- The field list is fixed in one place.
- Missing field path fails before writes.
- A teammate can run validation on fake row and predict what is required.
