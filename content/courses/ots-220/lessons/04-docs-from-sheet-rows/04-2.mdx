# Document Templates and Placeholders

## Why This Matters

Template drift causes "silent bad docs." The class should see exactly what changed and where.

## Core Idea

Use one placeholder map and log unmapped markers.

## Mapping Table

| Template marker | Source value |
| --- | --- |
| `{{unitName}}` | row.unitName |
| `{{learningTarget}}` | row.learningTarget |
| `{{evidencePrompt}}` | row.evidencePrompt |
| `{{sectionName}}` | row.sectionName |

## Implementation

```javascript
function applyPlaceholdersToDoc(templateDocId, row) {
  const doc = DocumentApp.openById(templateDocId);
  const body = doc.getBody();

  body.replaceText("{{unitName}}", row.unitName);
  body.replaceText("{{learningTarget}}", row.learningTarget);
  body.replaceText("{{evidencePrompt}}", row.evidencePrompt);
  body.replaceText("{{sectionName}}", row.sectionName);

  const remaining = ["{{unitName}}", "{{learningTarget}}", "{{evidencePrompt}}", "{{sectionName}}"].filter((token) => doc.getBody().findText(token));
  if (remaining.length > 0) {
    throw new Error(`Unresolved placeholders: ${remaining.join(", ")}`);
  }

  doc.saveAndClose();
  return doc.getUrl();
}
```

## Quality Check

Run on fake row and verify:

- all markers replaced
- no unresolved placeholders remain
- template url and doc title are correct
- no student data is present

## Failure Drill

- unresolved placeholder should throw a readable error
- failure requires template marker fix before rerun
