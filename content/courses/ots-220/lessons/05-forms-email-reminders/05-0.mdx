# Chapter Plan

## Why This Matters

Teachers often discover reminders too late because follow-up depends on memory, not a system.  
In this chapter you build a **small, auditable reminder pipeline** so communication is predictable, not magical.

In **Forms, Email, and Reminders**, automation quality is visible when you can explain:

- what triggers the email,
- what input is allowed,
- who approves each message,
- how failures are handled,
- how you roll back safely.

This section defines the workflow before writing code.

## Chapter Scenario

Your team runs a robotics unit and uses a Form called **Checkpoint Submission Form**.

Example process:
- Student submits form when assignment is missing documentation.
- A row appears in the response sheet.
- Teacher later approves a reminder draft.
- Script sends only approved, non-sensitive reminders.

You will plan around three artifacts:
- `Checkpoint_Form` (Google Form)
- `ReminderQueue` (Sheet with approval state)
- `Reminder_Template` (Doc or hardcoded template text in script comments)

## Core Idea

You are designing a workflow that writes reminders to a staging sheet, not directly to email.  
That single decision creates:
- safer testing,
- easier auditing,
- and an explicit human-review gate.

## Plan the Workflow

Complete this design table before moving to 05.1.

| Question | Your choice |
| --- | --- |
| Trigger source | Form submit to `Checkpoint_Form` |
| Queue destination | `ReminderQueue` with status `needs_review` |
| Approval owner | You (teacher) only |
| Email limit gate | Send only rows with status `approved` |
| Personal data handling | Store `studentAlias`, **never** store raw student email |
| Backup/reset plan | Keep a copy of the trigger output sheet for one week |
| Evidence to show for grading | One approved row, one withheld row, and execution log |
| Review cadence | After each unit change or schedule change |

## Delivery Definition

Your chapter is ready to continue only if:
- You can explain why draft stage exists and where it lives,
- At least one consent-safe sample row is prepared,
- You know exactly who changes status from `needs_review` to `approved`.

## Quick Script Snippet

```javascript
function openReminderQueue() {
  const reminderSheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("ReminderQueue");
  if (!reminderSheet) {
    throw new Error("ReminderQueue sheet is missing.");
  }
  Logger.log(`ReminderQueue rows: ${reminderSheet.getLastRow() - 1}`);
}
```

If `ReminderQueue` is missing, run a manual check and pause the chapter before any reminder code is added.
