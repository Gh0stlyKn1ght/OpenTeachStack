# Forms and Responses

## Why This Matters

Form responses are only safe to automate if each response is normalized into a predictable reminder request.
This section builds the first transform step: **form response → queue row**.

## Core Idea

Treat every response as untrusted until it passes:
- required fields check,
- consent check,
- sanitization check,
- duplicate check.

You are not sending email here. You are creating a stable `ReminderQueue` row that can be reviewed by a teacher.

## Workshop: Map Response to Queue

Use this function on form submit:

```javascript
function onCheckpointFormSubmit(e) {
  const row = e.namedValues;

  const payload = {
    requestId: `RQ-${Date.now()}`,
    studentAlias: (row.StudentAlias?.[0] || "").trim(),
    guardianEmail: normalizeEmail(row.GuardianEmail?.[0] || ""),
    missingItem: (row.MissingItem?.[0] || "").trim(),
    dueDate: row.DueDate?.[0] || "",
    consent: String(row.Consent?.[0] || "no").toLowerCase() === "yes",
    status: "needs_review",
    notes: row.Notes?.[0] || ""
  };

  validateReminderPayload(payload);
  appendReminder(payload);
  Logger.log(`Queued request ${payload.requestId}`);
}

function normalizeEmail(value) {
  return value.trim().toLowerCase();
}

function validateReminderPayload(payload) {
  if (!payload.studentAlias) throw new Error("Missing StudentAlias.");
  if (!payload.consent) throw new Error("Consent not granted.");
  if (!payload.guardianEmail.includes("@")) throw new Error("Invalid guardian email.");
  if (!payload.missingItem) throw new Error("Missing reminder reason.");
  if (!payload.dueDate) throw new Error("Missing due date.");
}

function appendReminder(payload) {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("ReminderQueue");
  const row = [
    payload.requestId,
    payload.studentAlias,
    payload.guardianEmail,
    payload.missingItem,
    payload.dueDate,
    payload.status,
    "",
    "",
    payload.notes
  ];
  sheet.appendRow(row);
}
```

## Required Queue Columns

`ReminderQueue` should have this header order:
`requestId | studentAlias | guardianEmail | missingItem | dueDate | status | reviewedAt | reviewer | notes`

Rows with missing consent stay out of queue by throwing an error in `validateReminderPayload`.

## Error Cases to Expect

- Form schema mismatch (renamed header): log and pause trigger.
- Invalid email text: script throws, row not written.
- Duplicate submissions: handle manually (or add a dedupe check in revision).

## Quality Check

- Trigger is installable and fires on form submit.
- `ALEX-Q` row writes with `needs_review`.
- `BLAKE-R` with consent `no` raises a consent error instead of queuing.
- `ReminderQueue` row order matches headers exactly.
