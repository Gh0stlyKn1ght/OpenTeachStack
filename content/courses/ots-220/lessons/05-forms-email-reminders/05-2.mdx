# Email Safety and Rate Limits

## Why This Matters

Even a correct queue can hurt if it sends too many emails, at the wrong time, or to wrong addresses.
This section adds a **safety rail layer** before actual sending.

## Core Idea

Your send phase needs:
- address hygiene,
- approval gate (`status === approved`),
- simple throttling,
- recoverable logging,
- and a hard stop rule for suspected misuse.

## Classroom Scenario

Robotics teachers in your team review reminders before sending, but they still need to avoid:
- spamming a guardian by duplicate sends,
- hitting daily quota limits,
- accidentally sending `needs_review` rows.

You will enforce these as code rules.

## Safety Utilities

```javascript
const QUEUE_STATUS_APPROVED = "approved";
const RATE_LIMIT = 30; // max sends per execution for this lesson

function canSendReminderRow(row) {
  if (row[5] !== QUEUE_STATUS_APPROVED) {
    Logger.log(`Skip ${row[0]}: status=${row[5]}`);
    return false;
  }

  if (!/^[^@\s]+@[^@\s]+\.[^@\s]+$/.test(row[2])) {
    Logger.log(`Skip ${row[0]}: invalid email=${row[2]}`);
    return false;
  }

  return true;
}

function withBackoffRetry(fn, tries = 2) {
  for (let attempt = 0; attempt <= tries; attempt++) {
    try {
      return fn();
    } catch (err) {
      if (attempt === tries) throw err;
      const sleepMs = 500 * Math.pow(2, attempt);
      Logger.log(`Retry ${attempt + 1} after ${sleepMs}ms`);
      Utilities.sleep(sleepMs);
    }
  }
}
```

## Rate-Limit Pattern

Read queue rows and only send up to `RATE_LIMIT` per run.

```javascript
function sendWithSafety(sheet, startRow) {
  const data = sheet.getRange(startRow, 1, sheet.getLastRow() - startRow + 1, 9).getValues();
  let sent = 0;

  for (const row of data) {
    if (sent >= RATE_LIMIT) {
      Logger.log("Rate cap reached, leaving remaining rows for next run.");
      break;
    }
    if (!canSendReminderRow(row)) continue;

    withBackoffRetry(() => {
      const email = row[2];
      const subject = `Reminder: ${row[3]}`;
      const body = `Hi ${row[1]}, a reminder for ${row[3]} is due ${row[4]}.`;
      MailApp.sendEmail({
        to: email,
        subject,
        body,
      });
    });

    sent++;
    Logger.log(`Sent reminder ${row[0]}`);
  }
}
```

| RQ-1 | ALEX-Q | alex.guardian@example.edu | Safety checklist | 2026-09-20 | approved |
| RQ-2 | BLAKE-R | blake.guardian@example.edu | Lab rubric | 2026-09-22 | needs_review |

- `RQ-2` is skipped because status is not `approved`.
- Invalid email or missing approval stays in sheet; no send, no crash.

## Privacy and Permission Notes

- Never send reminders if rows contain direct student emails.
- Keep `approved` transitions manual in your workflow.
- Include a comment-only template in script docs: which fields are redacted from logs.

## Quality Check

- A queue with 40 approved rows sends only first `RATE_LIMIT` for this run.
- Duplicate send avoided by processing pointer or timestamp marking in revision.
- At least one skipped row is written with reason in logs.
