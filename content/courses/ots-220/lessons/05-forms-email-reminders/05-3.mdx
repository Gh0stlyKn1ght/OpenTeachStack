# Workshop: Reminder Emails from Sheet Data

## Why This Matters

This workshop connects everything:
- form responses,
- safe staging in queue,
- sending guardrails.

You now build and run a low-risk routine: send a small batch to a test label and move rows through statuses.

## Core Idea

A production-safe workshop script does three things in one pass:
1) selects approved rows,
2) sends emails with logging,
3) marks row status with audit data.

```javascript
function runReminderBatch() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("ReminderQueue");
  const start = 2; // skip header
  const rows = sheet.getDataRange().getValues();
  const maxSends = 5;
  let sent = 0;

  for (let i = 1; i < rows.length && sent < maxSends; i++) {
    const row = rows[i];
    const rowNum = i + 1;

    if (!canSendReminderRow(row)) continue;

    try {
      const subject = `Reminder: ${row[3]}`;
      const body =
        `Hi ${row[1]},\n\n` +
        `This is a reminder for ${row[3]} due ${row[4]}.\n\n` +
        `Please review and reply to your student directly.`;

      withBackoffRetry(() =>
        MailApp.sendEmail({
          to: row[2],
          subject,
          body,
        })
      );

      sheet.getRange(rowNum, 6).setValue("sent");
      sheet.getRange(rowNum, 7).setValue(new Date());
      Logger.log(`Sent ${row[0]}`);
      sent++;
    } catch (err) {
      Logger.log(`Failed ${row[0]}: ${err.message}`);
      sheet.getRange(rowNum, 6).setValue("error");
      sheet.getRange(rowNum, 9).setValue(`send_failed:${err.message}`);
    }
  }

  Logger.log(`Run complete: ${sent} sent`);
}
```

| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| RQ-1001 | ALEX-Q | alex.guardian@example.edu | safety checklist | 2026-09-20 | approved | 2026-09-16 07:30 | Teacher Alex | confirmed |
| RQ-1002 | KIM-P | kim.guardian@example.edu | 3D model update | 2026-09-21 | approved | 2026-09-16 07:31 | Teacher Alex | confirmed |

Expected after running `runReminderBatch()`:
- RQ-1001, RQ-1002 statuses become `sent`.
- `reviewedAt` set to run timestamp.
- any failure writes `send_failed:<reason>` in notes column.

## Test Loop Plan

1) Test with fake addresses first (`sample-guardian@...`).  
2) Verify logs show both successful sends and status transitions.  
3) Mark one row `needs_review`; confirm it is skipped.  
4) Add a wrong email and confirm it becomes `error`.

## Rollback

If this run sends too many messages:
- open sheet,
- revert mistaken rows back to `approved`,
- do a second controlled run with smaller batch size.

## Quality Check

- At least one successful send in log.
- One skipped row (non-approved) remains unchanged.
- One failed row writes a reason.

## Reflection

If your teacher asks, "Who approved each message and when?" can your sheet prove that from `reviewedAt` and `reviewer` fields?
