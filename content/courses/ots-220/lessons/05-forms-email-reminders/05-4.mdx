# Checkpoint Review

## Why This Matters

Checkpointing turns a working script into a teacher-readable process.
You should prove both behavior and intention before leaving the chapter.

## Core Review Rubric

- **Ready**: clear approvals, queue flow, test evidence, and safe fallback behavior.
- **Revise**: missing review boundary, inconsistent queue status handling, unclear logs.
- **Blocked**: consent not enforced, non-reversible sends, secrets in script.

## Evidence Checklist

Create a one-page evidence snapshot for the chapter:

1. `ReminderQueue` headers and sample rows
2. One logged successful send
3. One failed/errored send with reason
4. At least one skipped non-approved row
5. Rollback steps documented in lesson notes

## Required Outputs

Capture and attach (or screenshot) for this checkpoint:

| Output | Expected |
| --- | --- |
| `getLastRow()` output | Shows at least 2 queue rows |
| Form submit test | Row moves to `needs_review` or `approved` |
| Manual review | Owner sets approved rows |
| Email run log | Shows sent count and errors |
| Permission audit | Apps Script auth dialog confirms minimal scopes |
