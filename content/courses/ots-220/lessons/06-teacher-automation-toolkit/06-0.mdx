# Chapter Plan

## Why This Matters

Earlier chapters built useful automations: folder creation, document generation, form capture, and controlled reminders.
In this chapter, you package those pieces into something a colleague can understand, review, and maintain.

## Core Idea

An automation toolkit is not one script file.
It is a **system with ownership, permissions, checks, rollback, and handoff notes**.
If this is missing, automations stop being sustainable after one unit.

## Final Artifact You Will Build

By the end of 06, complete this 5-part toolkit:

- `AutomationManifest` sheet (scope, owners, scripts, trigger map)
- `Review Checklist` section
- `Runbook` section (how to test and deploy)
- `Release Snapshot` section (what changed this week)
- `Rollback` section (how to pause all automation immediately)

## Classroom Context

Use the same robotics workflow from OTS-220:
- students submit checkpoint forms,
- reminders are drafted and reviewed,
- approved reminders are sent in controlled batches.

Your toolkit keeps that flow safe across team changes.

## Setup Checklist Before 06.1

1. Confirm you have `ReminderQueue` and `AutomationManifest` in your working spreadsheet.
2. Confirm previous chapter’s functions exist (or notes for deferred implementation).
3. Confirm script authorization scope includes only what each automation needs.
4. Confirm an emergency toggle exists (`STOP_AUTOMATION`) in a clearly visible sheet cell.

function initializeToolkitManifest() {

  const sheet = ss.getSheetByName("AutomationManifest") ??
    ss.insertSheet("AutomationManifest");
  if (!sheet.getLastRow()) {
    sheet.appendRow(["component", "owner", "trigger", "status", "lastReviewed"]);
    sheet.appendRow(["onCheckpointFormSubmit", "Teacher", "Form submit", "active", new Date()]);
