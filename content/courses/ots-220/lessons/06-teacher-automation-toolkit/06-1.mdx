# Script Review Checklist

## Why This Matters

Before a script is trusted in class operations, another adult should be able to review it in under ten minutes and answer:
- What does it do?
- What can it change?
- What can go wrong?

## Core Idea

Your review checklist becomes a repeatable contract, not a personal preference.
Every script function should answer:
- trigger source,
- input sources,
- output destinations,
- permission needs,
- and explicit failure response.

## Review Template

Add this section to your `AutomationManifest`:

| Function | Trigger | Inputs | Outputs | Risk | Safe stop |
| --- | --- | --- | --- | --- | --- |
| onCheckpointFormSubmit | Form submit | StudentAlias, GuardianEmail | ReminderQueue row | medium | stop form trigger |
| runReminderBatch | time-based | ReminderQueue approved rows | sent email + status update | medium-high | set STOP_AUTOMATION=true |

## Example Checklist Function

```javascript
const CHECKLIST = [
  { fn: "onCheckpointFormSubmit", owner: "Teacher", status: "approved" },
  { fn: "runReminderBatch", owner: "Teacher", status: "approved" },
  { fn: "sendWithSafety", owner: "Teacher", status: "reviewed" }
];

function writeChecklistReport() {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const sheet = ss.getSheetByName("AutomationChecklist") ||
    ss.insertSheet("AutomationChecklist");
  sheet.clear();
  sheet.appendRow(["timestamp", "function", "status", "reviewer", "notes"]);
  const reviewer = Session.getActiveUser().getEmail() || "manual";
  CHECKLIST.forEach((r) => sheet.appendRow([new Date(), r.fn, r.status, reviewer, "checked-in-" + Session.getActiveUser().getEmail()]));
  Logger.log("Checklist recorded");
}
```

## Risk Review Questions

- Does any function call `MailApp.sendEmail` outside the approved run?
- Are there hard-coded emails or IDs in source that should be in properties?
- Is there an opt-out/consent condition before any send logic?
- Can you stop it immediately with one condition flag?

## Common Failure Modes

- New script added without reviewer note.
- Status transitions not logged (`needs_review`, `approved`, `sent`, `error`).
- Trigger runs with stale or unexpected parameters.
- Logs expose sensitive data.

## Quality Check

- Add/update one row per reviewed function in `AutomationChecklist`.
- Confirm `status` values are only `approved`, `reviewed`, or `blocked`.
- Confirm one script has a stop condition test documented in notes.
