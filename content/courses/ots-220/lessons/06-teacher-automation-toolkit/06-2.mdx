# Documentation and Handoff

## Why This Matters

Most automations fail not from code bugs but from missing handoff.
Someone else inherits a script, sees it works, and then can’t explain why it exists.

## Core Idea

Treat documentation as an operating contract:
- what it does,
- when to run it,
- what output to expect,
- how to troubleshoot,
- when not to run it.

## Team Handoff Template

Create a sheet named `AutomationRunbook` with:

| Field | Value |
| --- | --- |
| System purpose | Reminders from form responses with manual approval |
| Primary owner | Your email alias |
| Secondary owner | Backup teacher email alias |
| Emergency stop | Set `STOP_AUTOMATION = TRUE` in `Config` sheet |
| Last release | date + short commit note |
| Known risks | no consent check means immediate pause |

## Shareable Runbook Generator (Script)

```javascript
function writeHandoffNotes() {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const runbook = ss.getSheetByName("AutomationRunbook") || ss.insertSheet("AutomationRunbook");
  runbook.clear();
  runbook.getRange("A1:B1").setValues([["Field", "Value"]]);
  runbook.getRange("A2:B2").setValues([
    ["System purpose", "Reminder workflow with form queue + manual approval"]
  ]);
  runbook.getRange("A3:B3").setValues([
    ["Emergency stop", "Set STOP_AUTOMATION=true"]
  ]);
  runbook.getRange("A4:B4").setValues([
    ["Owner", Session.getEffectiveUser().getEmail() || "manual-owner"]
  ]);
  runbook.getRange("A5:B5").setValues([["Last updated", new Date()]]);
  Logger.log("Runbook updated");
}
```

## Handoff Checklist for a New Steward

- open `AutomationManifest`,
- confirm `STOP_AUTOMATION` switch behavior,
- run `writeChecklistReport()`,
- verify last two runs in `Execution Log`,
- confirm at least one approved row can still become sent.
