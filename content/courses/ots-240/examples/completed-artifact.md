# Completed Artifact

## Completed Open Curriculum Repository Sample

This sample shows a public-safe repository packet for a fictional robotics mini-unit.

## README Sections

| Section | Completed example |
| --- | --- |
| Purpose | Share a Grade 8 robotics sensor mini-unit for collecting data and justifying threshold decisions. |
| Audience | Middle school robotics teachers and instructional coaches. |
| Included | Unit map, student directions, sample data, rubric, source notes, license notes, contribution checklist. |
| Not included | Real student work, answer keys, school-only links, private notes, unlicensed media. |
| Maintenance | Review source links, accessibility notes, and license notes before each semester. |

## Source And License Rows

| Resource | Use | Decision |
| --- | --- | --- |
| Teacher-created sensor data table | Student sample | Keep under repository license. |
| Official sensor setup documentation | Teacher reference | Cite and follow source terms. |
| OER robotics activity | Adapted structure | Include attribution and change note. |
| Blog image | Decoration | Exclude because license is unclear. |

## Contribution Review Rows

| Contribution | Decision | Reason |
| --- | --- | --- |
| Adds fictional sample data row | Accept | Improves evidence without exposing students. |
| Adds blog image | Revise | Needs license note and text alternative. |
| Uploads real student work | Reject | Violates privacy boundary. |

Use this as a minimum baseline before public sharing. A stronger repository also includes issue templates, changelog entries, and release notes for each semester.
