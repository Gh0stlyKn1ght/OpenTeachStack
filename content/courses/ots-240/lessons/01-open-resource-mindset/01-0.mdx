# Chapter Plan

## Why This Matters

Open resources can still harm learners and schools if usage rights, privacy expectations, and maintenance rules are unclear.
In this chapter you build one decision system before sharing anything publicly.

## Core Idea

Every shared file answers the same questions:

- What is this?
- Who can use it?
- Under what terms?
- What conditions require revision?
- Who owns updates?

If you can answer these in one documented place, your material stays reusable and safe.

## Scenario

You are preparing a robotics lesson kit for other educators:
- lesson PDFs,
- code snippets,
- a worksheet image,
- and a gradebook rubric.

You want others to remix them for class pacing but protect student examples.

Create a file named **`Open resource decision log`** with one row per artifact.

## Log Template

| Artifact | Goal | Reuse level | Allowed sources | Privacy rule | Owner | Next review |
| --- | --- | --- | --- | --- | --- |
| Robot challenge guide | Classroom activity | remixed | public domain + CC-BY | remove student names | You + backup lead | after each unit |
| Safety rubric image | Demonstration | only template use | teacher created | no student photos | You | every August |
| Sample gradebook template | internal + optional external use | attribution required | no student data | sanitized IDs only | monthly |

## Minimum Definitions

- **Reuse level**: allowed, attribution required, adapted only, or private.
- **Allowed sources**: where content came from and what rights were granted.
- **Privacy rule**: exact rule for removing or masking student data.

## Quality Check Before Chapter Start

- Create the table in your repository notes.
- Give every artifact a row before discussing licensing.
- Add a date and one owner for each row.

If a row is empty, you pause before moving to 01.1.
