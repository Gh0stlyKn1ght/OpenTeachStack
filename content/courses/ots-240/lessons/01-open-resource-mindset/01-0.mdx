# Chapter Plan

## Why This Matters

Open work needs when a teacher can point to the decision, the classroom use, the evidence, and the next review step.

In **Open Resource Mindset**, the work sits inside open curriculum publishing with GitHub, README files, licenses, attribution, and review routines. The risk is treating free-to-view material as automatically open, reusable, remixable, or safe to publish. This lesson narrows that risk into one visible update to **Open resource decision log**.

## Core Idea

Map the chapter before building. Choose one classroom material or teacher decision from this list: a repository file, README section, issue, pull request, source note, or license entry. Then name the classroom purpose it serves and the evidence that will show whether the work is ready.

The goal is not to make a perfect document in one pass. The goal is to make the teacher decision clear enough that another educator can inspect it, reuse it, or challenge it without needing a private explanation.

## Lesson Focus

Because this is the opening plan for open resource mindset, the teacher names the sequence before creating files or artifacts. The plan separates what will be learned, what will be built, and what evidence will prove that the build is ready.

In 01.0, focus on chapter planning decisions. Choose one concrete classroom material or teacher decision from this list: a repository file, README section, issue, pull request, source note, or license entry. Connect it to open resource decision log, then decide whether it needs to be created, revised, tested, credited, restricted, published, or archived.

Connect chapter planning decisions back to open resource mindset. That connection matters because a polished artifact can still fail if the lesson purpose, boundary, evidence, and maintenance routine are not visible.

## Quality Check

Before moving on, test the work against this standard: a reviewer can see what may be reused, what must be credited, what changed, and how the repository will be maintained.

A reviewer should be able to find the chapter planning decisions decision, inspect the open resource decision log evidence, and understand how it supports open resource mindset.

If the work only names a tool or a file, revise it. The artifact needs to show the teaching purpose, evidence, boundary, and maintenance routine.
