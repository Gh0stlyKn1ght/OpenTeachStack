# Free, Open, and Allowed

## Why This Matters

If a resource appears in your repository, students assume it is safe to use.
You need a repeatable filter:

- Is it actually available for classroom use?
- Does it match the grade level and subject?
- Does it conflict with policy?

## Three-Question Triage

Use this checklist on each source before storing it in your repo:

1. **License visibility**
2. **Data sensitivity**
3. **Attribution requirement**

## Classroom Example

A physics teacher shares a wiring diagram found in a forum and a sample student file.

Decision log action:

| Artifact | Is it allowed? | Why |
| --- | --- | --- |
| Forum diagram | no | no visible license and mixed author comments |
| Vendor template | yes | explicit educational use with attribution |
| Student file | no | includes student photos and PII |

## Action Plan

Create a review note for each artifact in `Open resource decision log`:

- mark `yes`, `yes-with-credit`, or `no`.
- include exact link or source.
- include one sentence explaining the decision.

## Failure Handling

- If license is missing: do not add to shared materials.
- If attribution is unclear: request owner permission or replace with created content.

Before 01.2, every artifact has a status column value and a short reason.
