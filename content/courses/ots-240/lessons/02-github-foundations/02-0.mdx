# Chapter Plan

## Why This Matters

A teacher repository without version history is hard to restore after a bad edit.
GitHub gives you a shared recovery path, but only if you use branches, commits, and reviews consistently.

## What You Will Build

- A simple repository map.
- A clean commit message pattern.
- A branch strategy for class updates.

## Chapter Setup

Create an orientation note before editing:

| Area | Owner | Purpose |
| --- | --- | --- |
| `/content` | You | lesson files |
| `/README.md` | You | usage summary |
| `/LICENSE` | You + admin | rights rules |
| `/docs` | You | source notes |

## Commit Rules

Use this format:
`course: <scope> | action | reason`

Example:
`ots-240 | add | initial repository orientation notes`
