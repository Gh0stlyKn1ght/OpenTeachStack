# Repos, Files, Commits, and Branches

## Why This Matters

Version control is your safety net when curriculum changes are rushed.
This lesson teaches the difference between `main` (stable) and `draft` (work-in-progress).

## Scenario

You maintain a robotics module with:
- weekly lesson updates,
- recurring typo fixes,
- teacher feedback on examples.

Use a dedicated branch for every edit batch.

## Example Commands

```bash
# open an existing repo
cd /tmp/robotics-open-course
# create a branch for your weekly edits
git checkout -b week-09-rev

git status

git add .
git commit -m "course: ots-240 | add | weekly lesson metadata updates"

git checkout main
git merge week-09-rev
```

## Commit Review

Before commit, verify:
- one purposeful change per commit,
- no student identifiable data in tracked files,
- branch name reflects intent.
