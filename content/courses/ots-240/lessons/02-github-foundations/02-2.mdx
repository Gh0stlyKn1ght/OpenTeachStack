# README as Teacher Documentation

## Why This Matters

A repository without a clear README fails fast for substitute teachers and future-you.

## Minimum Sections in README

- What this repo is for.
- Who should use it.
- How to install or run setup.
- What changed recently.
- Who reviews updates.

## Starter Notes for This Course

```markdown
## Quick start
1. Open lesson files in `content/`.
2. Read `docs/` for deployment notes.
3. Review `LICENSE` before reusing materials.

## Review cycle
- Weekly branch review.
- Monthly privacy check.
```

Add a `Quick start`, `Review cycle`, and `Contact` section to your repository README draft.
