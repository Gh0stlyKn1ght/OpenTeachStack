# Purpose, Audience, and Scope

## Why This Matters

A good README starts by narrowing who should read it.

## Audience Table

| Audience | Pain point | What README must answer |
| --- | --- | --- |
| New teacher substitute | I need quick setup | first lesson and repo check steps |
| Team lead | Need quality checks | review and release process |
| Partner school teacher | Need reuse rules | license and attribution |

## Writing Rule

Use one sentence per heading:
- what this repo is,
- who it is for,
- what they can and cannot do.
