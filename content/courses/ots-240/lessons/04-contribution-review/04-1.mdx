# Issues, Pull Requests, and Review

## Why This Matters

Without review, open changes can silently replace lesson intent.

## Small Workflow

1. Open an issue with expected change and reason.
2. Make changes in a named branch.
3. Open PR with checklist links.
4. Reviewer confirms license/privacy and marks approved.

## Example PR Template

```text
## Change
- what changed and why

## Review checklist
- [ ] No student personal data
- [ ] Files are scoped to lesson scope
- [ ] Attribution file updated
- [ ] Rollback is documented
```
