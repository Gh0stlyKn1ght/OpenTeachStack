# Contribution Guidelines

## Why This Matters

Guidelines reduce conflict and speed up teaching team trust.

## Minimum Content

- branch naming conventions,
- commit message format,
- review expectation,
- what not to change in one pass.

## Template Snippet

```text
- Use branch names like `teacher/review-YYYY-MM`.
- Keep one idea per PR.
- Never remove student examples without approval.
- If terms change, update `Open resource decision log` first.
```

## Quality Check

When peers contribute, they should know exactly where to document changes and where to stop.
