# Contribution Guidelines

## Why This Matters

Open work needs when a teacher can point to the decision, the classroom use, the evidence, and the next review step.

In **Contribution and Review Workflows**, the work sits inside open curriculum publishing with GitHub, README files, licenses, attribution, and review routines. The risk is treating free-to-view material as automatically open, reusable, remixable, or safe to publish. This lesson narrows that risk into one visible update to **Contribution checklist**.

## Core Idea

Turn the lesson idea into an inspectable decision. Name the classroom use, the evidence, the review step, and the next revision trigger.

The goal is not to make a perfect document in one pass. The goal is to make the teacher decision clear enough that another educator can inspect it, reuse it, or challenge it without needing a private explanation.

## Lesson Focus

Because this is a focused section, the teacher isolates the contribution guidelines decision from the rest of the course. The lesson asks what this idea changes, where it appears, and how the teacher will know it worked.

In 04.2, focus on contribution guidelines. Choose one concrete classroom material or teacher decision from this list: a repository file, README section, issue, pull request, source note, or license entry. Connect it to contribution checklist, then decide whether it needs to be created, revised, tested, credited, restricted, published, or archived.

Connect contribution guidelines back to contribution workflows. That connection matters because a polished artifact can still fail if the lesson purpose, boundary, evidence, and maintenance routine are not visible.

## Classroom Example

Use this working situation: a teacher is preparing a curriculum repository with a README, source notes, a license file, contribution guidance, and a release checklist.

## Quality Check

Before moving on, test the work against this standard: a reviewer can see what may be reused, what must be credited, what changed, and how the repository will be maintained.

A reviewer should be able to find the contribution guidelines decision, inspect the contribution checklist evidence, and understand how it supports contribution workflows.

If the work only names a tool or a file, revise it. The artifact needs to show the teaching purpose, evidence, boundary, and maintenance routine.
