# Workshop: Review a Small Change

## Goal

Review a one-file edit as if you were the project reviewer.

## Simulation

1. Open a pull request that edits one lesson section.
2. Check diff for:
   - scope,
   - accuracy,
   - privacy,
   - attribution updates.
3. Record result in **Review log**.

## Reviewer Decision Form

| Field | Expected |
| --- | --- |
| Scope | one intended file |
| Risk | none / low / medium |
| Privacy | no student identifiers |
| Decision | approve or request changes |

## Recovery Action

If issue found: request changes, not direct merge.
