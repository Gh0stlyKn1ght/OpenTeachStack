# Chapter Plan

## Why This Matters

By now you have content, READMEs, and attribution decisions.
This final chapter turns those pieces into one reusable repository deliverable.

You finish when another educator can:
- understand what is included,
- know what can be reused,
- and run a predictable release check.

## Core Idea

An open curriculum repository is a **system**, not a folder.
It needs:
- a release surface,
- a maintenance rhythm,
- a safe handoff boundary,
- evidence that checks passed.

## Chapter Outputs

You will produce:
- `Repository Release Checklist`
- `Maintenance Plan`
- final assembled `Open curriculum repository` artifact (README + policy + process files)

## Scenario

Course: Open Resources course, teacher team, one shared repo.

You are preparing to publish to a district shared area with:
- public link,
- clear reuse terms,
- and a weekly maintenance owner.

## Required Initial Setup

| Item | Required value |
| --- | --- |
| Release owner | Primary teacher email |
| Backup owner | Co-owner email |
| Review cadence | Monthly review + release date stamp |
| Safety boundary | No student identifiers in public files |
| Final artifact destination | `/OpenResourcesRepository` |
