# Open Curriculum Repository

## Why This Matters

This lesson assembles everything into a student-safe, teacher-reusable repository package.

## Build Scope

Create or verify:
- `README.md`
- `LICENSE`
- `SOURCE_NOTES.md`
- `ATTRIBUTION.md`
- `docs/maintenance.md`
- `lessons/` with all chapter files

## Required Final Structure

```text
/README.md
/LICENSE
/SOURCE_NOTES.md
/ATTRIBUTION.md
/docs/maintenance.md
/content/courses/ots-240/
```

## Build Script (Check-only)

```bash
ls README.md LICENSE SOURCE_NOTES.md ATTRIBUTION.md
rg -n "Student|email|phone" README.md SOURCE_NOTES.md ATTRIBUTION.md
```

If any sensitive identifier appears in public files, remove/replace before release.

## Final Assembly Tasks

1. Add release decision at top of `README.md` (`Publish`, `Publish internally`, `Revise`, `Blocked`).
2. Link maintenance table into README.
3. Confirm `LICENSE` summary exists in first section.
4. Confirm `ATTRIBUTION.md` has at least one external and one internal source row.
5. Confirm review owner and date are documented.

## Evidence Package

Keep these 6 evidence items for chapter exit:
- completed `SOURCE_NOTES.md`,
- complete `LICENSE`,
- complete `ATTRIBUTION.md`,
- `maintenance.md` with owner/date,
- release gate table filled,
- reviewer signoff note.

## Sample Release Artifact Text

```text
Release decision: Publish internally
Reason: ready for district-wide teachers; external-facing license review still in progress.
Owner: You
Next review: 2026-10-01
```
