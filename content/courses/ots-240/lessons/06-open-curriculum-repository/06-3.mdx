# Open Curriculum Repository

## Why This Matters

Open work needs when a teacher can point to the decision, the classroom use, the evidence, and the next review step.

In **Open Curriculum Repository**, the work sits inside open curriculum publishing with GitHub, README files, licenses, attribution, and review routines. The risk is treating free-to-view material as automatically open, reusable, remixable, or safe to publish. This lesson narrows that risk into one visible update to **Open curriculum repository**.

## Core Idea

Build the **Open curriculum repository** as a usable artifact, not just a note. The finished version shows the decision, the classroom context, the evidence, and the next maintenance step.

The goal is not to make a perfect document in one pass. The goal is to make the teacher decision clear enough that another educator can inspect it, reuse it, or challenge it without needing a private explanation.

## Lesson Focus

Because this is an artifact lesson, the teacher turns open curriculum repository into something another educator can use. The artifact needs labels, examples, status, review notes, and a next action instead of a vague promise to improve later.

In 06.3, focus on open curriculum repository. Choose one concrete classroom material or teacher decision from this list: a repository file, README section, issue, pull request, source note, or license entry. Connect it to open curriculum repository, then decide whether it needs to be created, revised, tested, credited, restricted, published, or archived.

Connect open curriculum repository back to open curriculum repository. That connection matters because a polished artifact can still fail if the lesson purpose, boundary, evidence, and maintenance routine are not visible.

## Quality Check

Before moving on, test the work against this standard: a reviewer can see what may be reused, what must be credited, what changed, and how the repository will be maintained.

A reviewer should be able to find the open curriculum repository decision, inspect the open curriculum repository evidence, and understand how it supports open curriculum repository.

If the work only names a tool or a file, revise it. The artifact needs to show the teaching purpose, evidence, boundary, and maintenance routine.
