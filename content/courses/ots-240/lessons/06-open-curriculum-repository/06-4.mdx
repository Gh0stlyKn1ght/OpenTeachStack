# Checkpoint Review

## Why This Matters

This final checkpoint decides whether the repo can be published safely.

## Final Rubric

- **Ready**: all release gates pass, maintenance owner/date is current, and no student identifiers are published.
- **Revise**: at least one mandatory gate fails.
- **Blocked**: any privacy or licensing blocker remains unresolved.

## Required Evidence Check

| Requirement | Pass condition |
| --- | --- |
| Licensing | LICENSE exists and matches README |
| Attribution | all external items listed in ATTRIBUTION.md |
| Source transparency | SOURCE_NOTES.md contains source + license + decision |
| Reviewability | maintenance.md has owner and next review date |
| Privacy safety | scan of public files shows no student identifiers |

## Exit Decisions

- `Publish`: all requirements pass and reviewer confirms.
- `Publish internally`: passes but external visibility is restricted by policy.
- `Revise`: fix missing or inconsistent fields.
- `Blocked`: remove or redact non-compliant material before any release.
