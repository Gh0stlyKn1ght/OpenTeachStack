# Media Purpose Map

## Why This Matters

Media belongs in a lesson when a teacher can point to the decision, the classroom use, the evidence, and the next review step.

In **Media With Purpose**, the work sits inside instructional media, AI image review, diagrams, slides, captions, transcripts, and accessibility checks. The risk is adding attractive media that distracts from the learning target, invents details, hides important information, or excludes students. This lesson narrows that risk into one visible update to **Media purpose map**.

## Core Idea

Build the **Media purpose map** as a usable artifact, not just a note. The finished version shows the decision, the classroom context, the evidence, and the next maintenance step.

The goal is not to make a perfect document in one pass. The goal is to make the teacher decision clear enough that another educator can inspect it, reuse it, or challenge it without needing a private explanation.

## Lesson Focus

Because this is an artifact lesson, the teacher turns media purpose map into something another educator can use. The artifact needs labels, examples, status, review notes, and a next action instead of a vague promise to improve later.

In 01.3, focus on media purpose map. Choose one concrete classroom material or teacher decision from this list: an image, diagram, slide, short video, transcript, caption file, or media review row. Connect it to media purpose map, then decide whether it needs to be created, revised, tested, credited, restricted, published, or archived.

Connect media purpose map back to media purpose. That connection matters because a polished artifact can still fail if the lesson purpose, boundary, evidence, and maintenance routine are not visible.

## Classroom Example

Use this working situation: a teacher is building a media packet with one explanatory diagram, one delivery deck, one short video plan, and accessibility notes for each asset.

For **Media Purpose Map**, choose one concrete classroom material or teacher decision from this list: an image, diagram, slide, short video, transcript, caption file, or media review row. Keep the example small enough to review during the lesson and focused on media purpose map.

Example entry:

| Decision | Classroom version |
| --- | --- |
| Teaching purpose | This work supports media purpose map inside media purpose. |
| Evidence | The teacher can point to purpose notes, prompt logs, alt text, captions, transcripts, diagram labels, slide roles, and peer review notes. |
| Boundary | The teacher names what media purpose map may change and what must stay protected or reviewed. |
| Maintenance | The next update is triggered by a date, failed check, student confusion, changed source, or media purpose map review. |

## Build The Work

Update the **Media purpose map** with one row or section for **Media Purpose Map**.

| Field | Your note |
| --- | --- |
| Lesson focus | media purpose map |
| Classroom context | Where media purpose appears in your course, unit, tool, site, media packet, safety routine, or capstone |
| Working material | Classroom material or teacher decision: an image, diagram, slide, short video, transcript, caption file, or media review row |
| Teacher decision | What media purpose map causes you to keep, revise, test, publish, restrict, or remove |
| Artifact status | What proves media purpose map is ready or what blocks it |
| Maintenance trigger | When the media purpose map decision should be checked again |

## Quality Check

Before moving on, test the work against this standard: the media makes the lesson easier to understand and can be used by students who need text, audio, visual, or pacing support.

A reviewer should be able to find the media purpose map decision, inspect the media purpose map evidence, and understand how it supports media purpose.

If the work only names a tool or a file, revise it. The artifact needs to show the teaching purpose, evidence, boundary, and maintenance routine.
