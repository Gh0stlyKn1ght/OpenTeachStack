# Prompting for Curriculum Images

## Why This Matters

Image prompts become reliable when they state teaching intent first and style constraints second. The easiest failure is asking for “a clean picture” and getting an asset that is irrelevant, misleading, or inaccessible.

## Core Idea

Write prompts from this formula:

- **Who** is the learner and what can they do?
- **What** concept is shown in one sentence?
- **What** must be visible in the image?
- **What** must never appear (safety/bias/accuracy constraints)?
- **How** should the output be constrained (format, color, labeling, file type)?

## Prompt Scaffold

Use this scaffold for every image request in this chapter.

```text
Role: You are a teacher creating a classroom-facing educational image.
Audience: [grade level], [subject], [known prior knowledge]
Objective: [learning goal in one sentence]
Required elements: [list 3-5 required labels/objects]
Visual style: [diagram / illustration / realistic / cartoon], [palette], [resolution], [aspect ratio]
Accessibility: high contrast, large text, clear labels, no decorative text-only meaning
Safety & ethics: no PII, no stereotypes, no copyrighted brand logos
Output format: [PNG/PDF], [size], [filename convention]
```

### Example

**Input**  
“Create a textbook-style diagram showing a database query flow for grade 9 students. Include source, filter, sort, and output boxes with arrows. Use a high-contrast palette and readable labels. Do not include people. PNG, 16:9, 2048px width. Add one line of alt text.”

| Candidate prompt | Final draft prompt |
| Alt-text draft | One short line + context sentence |
| Review owner | Name |
| Decision | READY/REVISE/BLOCKED |

## Immediate Quality Check

Before moving to 02.2, run a quick self-audit:
- Did the prompt encode learning intent explicitly?
- Does it prevent common model drift (wrong details, copyright, unsafe style)?
- Can another person run the same prompt and get a usable candidate?

## Reflection

What rule in your prompt list will prevent the next wrong image from slipping in? Add it explicitly.
