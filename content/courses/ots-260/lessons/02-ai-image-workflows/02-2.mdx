# Bias, Accuracy, and Source Notes

## Why This Matters

Beautiful outputs are not automatically correct. In teaching, accuracy failures and cultural bias can spread misconceptions faster than good explanation can correct them.

## Core Idea

Treat every image as a claim. If a learner sees the image and trusts it, it must be accurate, neutral, and attributable to a known source or clear reasoning process.

## Review Checklist

For each candidate image, complete the four checks before moving to `READY`:

1. **Accuracy check**: Can you label each non-text element correctly?  
2. **Bias check**: Does the scene exclude unnecessary stereotypes, demographics, or harmful framing?  
3. **Context check**: Is the image consistent with your class’s curriculum and prior lessons?  
4. **Source note**: Is the prompt version and validation decision clearly recorded?

## Source Notes Template

Use this log row format:

| Check | Notes |
| --- | --- |
| Source of claim | “Generated from curriculum requirements + diagram conventions” |
| Review evidence | “Arrow flow matches algorithm steps in lesson notes” |
| Bias correction applied | “Removed implied identity markers, replaced with neutral icons” |
| Accessibility fix | “Increased contrast for labels; added explicit alt text” |
| Citation | If inspired by textbook or standards, note section/page or URL |
| Decision | READY/REVISE/BLOCKED |

## Classroom Example

Scenario: a generated chemistry image labels containers as “A/B.”  
Review outcome:
- Accuracy: PASS (labels match expected symbols)
- Bias: PASS (no culture-coded people imagery)
- Source notes: PASS (prompt version + correction history logged)
- Decision: READY after one contrast pass

## Practice

Pick one candidate image from 02.1 and run it through the four checks.
Write the result directly into your artifact with:
- two corrective edits if needed,
- and the exact decision date.

## Common Mistake

Only checking one dimension (usually style) and ignoring content validity. If accuracy is uncertain, status must be `BLOCKED` or `REVISE`, never `READY`.

## Reflection

Which check failed first when you reviewed your image, and what exact change would fix it in one line?
