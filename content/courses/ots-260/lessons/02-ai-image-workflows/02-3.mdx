# Workshop: Generate and Review a Visual

## Why This Matters

This is where the workflow becomes practical. You now produce assets, compare outputs, and pick a version based on review evidence, not visual preference alone.

## Workshop Instructions

1. Choose one prompt from 02.1.
2. Generate **three versions** with the same constraints.
3. For each version, run this 6-point matrix:
   - Concept clarity
   - Label readability
   - Visual hierarchy
   - Accessibility (contrast, text size, alt text)
   - Factual alignment
   - Bias/sensitivity
4. Keep one version and archive the rejected candidates.
5. Record decision rationale in `Image prompt and review log`.

## Review Notes (Must add for winner and one rejected)

For the selected version, add:
- Why it is selected,
- What was improved in prompt or constraints,
- Whether it requires an accessibility revision,
- Status (`READY` or `REVISE`).

For one rejected version, add:
- The single critical failing point,
- One concrete fix if regenerated.

## Practice

Populate the scoring matrix and decision notes using a real generated image.
