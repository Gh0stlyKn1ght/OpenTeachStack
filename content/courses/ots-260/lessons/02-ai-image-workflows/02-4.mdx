# Checkpoint Review

## Why This Matters

Without a hard checkpoint, this chapter can feel complete while hidden flaws remain. This final step forces a concise decision before any classroom reuse.

## Core Check

Answer these three questions in your artifact:

1. Can we defend the image against one reviewer’s question set?
2. Are all required checks documented and dated?
3. Is there a recheck rule for drift and new curriculum needs?

## Ready/Revise/Blocked Rubric

| Status | Meaning | Required evidence |
| --- | --- | --- |
| READY | Safe to use in class | All checks passed, alt text exists, ownership assigned, recheck date set |
| REVISE | Needs one bounded fix | Missing one non-critical issue (contrast, wording, label clarity) |
| BLOCKED | Do not use | Accuracy failure, unresolved bias issue, source mismatch, or unapproved safety risk |

## Final Log Row

Complete this final checkpoint section:

| Field | Required entry |
| --- | --- |
| Final decision | READY / REVISE / BLOCKED |
| Evidence links | Prompt version IDs, review notes, alt text location |
| Open risks | Name remaining risks and owner |
| Next action | Revise, rerun, archive, or publish |
| Check date | Date + reviewer initials |

## Common Mistake

Marking `READY` while `Blocked` risks remain because they are waiting for “later.” If a risk is real, gate it in the log with clear owner and date.
