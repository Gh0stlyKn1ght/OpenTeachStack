# Chapter Plan

## Why This Matters

Diagrams carry teaching burden. If a diagram is unclear, students memorize appearance instead of process. This chapter standardizes how to turn abstract content into visuals that are accurate, testable, and reusable.

## Core Idea

Every visual must answer four questions before students see it:

1. What idea does this diagram explain?
2. What learner action does it support?
3. What could students misunderstand from it?
4. How will we review it before use?

The chapter artifact is a coherent `Diagram set` that captures this audit.

## Chapter Outcomes

| Outcome | Proof in artifact |
| --- | --- |
| Purpose-first diagram selection | Learning objective mapped to diagram type |
| Tool-aware construction | Mermaid source or design choices recorded |
| Accessibility-ready visuals | Labels, contrast, and alt text requirements captured |
| Review-ready versioning | Decision status, owner, and next review date |

## Chapter Plan Template

Complete one planning row before creating the final set.

| Field | Entry |
| --- | --- |
| Lesson focus | One sentence summary of what the diagram teaches |
| Diagram type | Flowchart, timeline, systems map, or architecture map |
| Learner action | Trace, compare, label, predict, or summarize |
| Constraints | No text-only cues, no low-contrast coding, no ambiguity in arrows |
| Review owner | Name + date of next check |
| Status | READY / REVISE / BLOCKED |

## Classroom Example

Topic: sorting process in computer science.

| Field | Example |
| --- | --- |
| Lesson focus | “Compare unsorted input to sorted output across steps.” |
| Diagram type | Flowchart |
| Learner action | Follow arrows and identify state changes |
| Constraints | Max 5 nodes visible, arrows only between named steps, 18px labels |
| Review owner | Lead instructor |

## Plan the Work

Open `Diagram set` and add the planning row with status `REVISE` or `READY` based on your first pass completeness.

## Common Mistake

Choosing diagram style before concept clarity. A perfect shape style cannot fix a wrong teaching structure.

## Reflection

What concept did you choose? What would make this diagram fail for a real student without additional explanation?
