# Flowcharts, Timelines, and Systems Maps

## Why This Matters

Different concepts need different structures. A flowchart may show sequence, a timeline shows change over time, and a systems map shows relationships. Choosing the wrong form makes students solve the wrong task.

## Core Idea

Select structure by learner action:

- Use **flowchart** when learners trace steps.
- Use **timeline** when learners compare sequence and causality over time.
- Use **systems map** when learners reason about dependencies.

## Selection Checklist

| Concept | Best structure | Why |
| --- | --- | --- |
| Process steps | Flowchart | Clear order and branch outcomes |
| Historical progression | Timeline | Time-aware comparison |
| Interdependent factors | Systems map | Relationship visibility |

## Practice

1. Concept name  
2. Chosen structure  
3. Learner action  
4. One reason this structure fits best

## Micro Example

Concept: data processing pipeline  
Structure: flowchart  
Learner action: follow input → validation → output  
Reason: sequence and conditional branches are central.

## Common Mistake

Forcing one template across concepts. If the structure doesn’t match action, students will memorize arrows instead of understanding.
