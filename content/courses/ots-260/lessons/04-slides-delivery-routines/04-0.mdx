# Chapter Plan

## Why This Matters

Slides are a delivery system, not a design showcase. Learners should complete actions faster and with fewer misunderstandings when slides match instruction flow.

## Core Idea

Set a reusable structure before adding content.

- Segment the class into timing blocks.
- Assign each slide one of three roles: orient, teach, do, check, or close.
- Define evidence that shows students can follow the sequence without verbal rescue.

This chapter builds a practical `Delivery deck` artifact.

## Chapter Outcomes

| Outcome | Evidence in artifact |
| --- | --- |
| Predictable slide roles | Every slide is mapped to a role |
| Timed flow | Minute-level pacing added per section |
| Student-facing instructions | Directions and calls-to-action are explicit |
| Accessibility checks | Alt text, contrast, and cognitive load checks are recorded |

## Planning Template

| Field | Entry |
| --- | --- |
| Class length | Total minutes and pacing intent |
| Slide roles | Number of orient/teach/do/check/close slides |
| Delivery risks | Top two places where students commonly stall |
| Accessibility notes | Font, contrast, alt text plan |
| Owner | Who reviews and approves the deck |
| Status | READY / REVISE / BLOCKED |

## Practical Example

Topic: introducing a process in a 25-minute lesson.

- Class length: 25
- Delivery roles: 2 orient, 4 teach, 3 do, 1 check, 1 close
- Risks: unclear do-step directions, skipped keyboard path
- Accessibility note: bullet labels, no color-only status cues

## Plan the Work

Create the first row in `Delivery deck` with your class length, roles, and two risks before writing content.

## Common Mistake

Adding slides without a role map. Unmapped slides become "slide noise" and increase cognitive overload.

## Reflection

What delivery risk can be reduced simply by labeling slide roles more clearly?
