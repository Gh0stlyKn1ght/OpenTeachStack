# Workshop: Build a Delivery Deck

## Why This Matters

The deck should pass an end-to-end dry run: learner can move from slide to slide and complete the task without missing context.

## Build Steps

1. Create 8–10 slide skeletons using role plan.
2. Add explicit directions and checks to action slides.
3. Test sequence with a short run-through timer.
4. Record friction points and fixes.
5. Set `Delivery deck` status.

## Required Artifact Notes

- run-through time,
- two friction points,
- one improvement applied,
- status (`READY`, `REVISE`, or `BLOCKED`).
