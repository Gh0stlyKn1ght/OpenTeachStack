# Checkpoint Review

## Why This Matters

The checkpoint protects a finished-looking deck from becoming unusable in real class timing.

## Final Standards

Review `Delivery deck` against the three outcomes below:

- Role clarity: each slide has a single primary purpose.
- Instructional clarity: each active slide has a clear direction + success criteria.
- Accessibility: all critical visuals and calls-to-action can be read by screen and low-vision students.

## Gate Table

| Status | Meaning | Required proof |
| --- | --- | --- |
| READY | Approved for class | Completed role map, dry-run notes, and accessibility checks |
| REVISE | Improve one bounded issue | Specific failing row + owner |
| BLOCKED | Do not use | Safety/accuracy/accessibility risk unresolved |

## Review Task

Complete one chapter summary row in `Delivery deck`.

- Total slides
- Ready slides
- Revised slides
- Blocked slides
- Reviewer and recheck date

## Reflection

What is the single checkpoint that would most likely be skipped under pressure? Add a guard for it.
