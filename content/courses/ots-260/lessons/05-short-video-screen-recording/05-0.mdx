# Chapter Plan

## Why This Matters

Short videos only scale when they target one learning bottleneck and include reusable accessibility hooks.

## Core Idea

Use this structure:

1. Define learning problem.
2. Decide video length and format.
3. Script one-to-two key points.
4. Record, caption, transcript, and review.

Keep `Short video plan` focused on one lesson outcome.

## Outcomes Checklist

| Outcome | Artifact evidence |
| --- | --- |
| Audience need is explicit | Problem statement + student context |
| Time budget is set | Max length and number of sections |
| Accessibility plan exists | captions/transcript owner + format |
| Review status | READY / REVISE / BLOCKED |

## Chapter Start

Fill your plan with one concrete use case, one format, and one ownership line.

Example: a 90-second review video for students who miss class.

## Practice

Open `Short video plan` and add the opening planning row:

- Target audience
- One learning task
- Length and sections
- Accessibility requirement
- Decision status

## Reflection

What happens if this video is used by a student watching in a noisy environment?
