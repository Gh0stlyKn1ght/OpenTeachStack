# Completed Artifact

## Completed Teacher Safety Review Sample

This sample uses fictional details and avoids storing secrets, passwords, student data, or screenshots with private information.

## Personal Risk Map

| Risk area | Calm action | Review trigger |
| --- | --- | --- |
| Public profile shows personal email | Replace with role-based school contact. | Recheck each semester. |
| Old classroom site links to private folder | Remove or restrict link. | Recheck after website updates. |
| Public examples look like real student work | Label fictional samples clearly. | Recheck before publishing. |

## Account Hygiene Checklist

| Account type | Password manager | MFA | Next action |
| --- | --- | --- | --- |
| District account | Unique password stored | Enabled | Review after policy change. |
| LMS account | Unique password stored | Enabled if available | Remove old shared credentials. |
| Classroom app trial | Unique password stored | Not available | Keep/archive/delete by semester end. |

## Phishing Response

| Step | Safe action |
| --- | --- |
| Pause | Do not click, reply, or forward. |
| Inspect | Compare sender, link, request, and urgency. |
| Verify | Use known district contact channel. |
| Report | Use district-approved phishing process. |
| Recover | Change password and check MFA if clicked. |

## Incident Response

| Situation | First step | Contact |
| --- | --- | --- |
| Suspicious login | Change password from known-good device. | District IT/help desk. |
| Public private-link exposure | Remove or restrict link. | Site owner/web admin. |
| Lost device | Follow district lost-device process. | IT/security office. |

Use this as a minimum baseline for teacher safety planning. A stronger artifact includes local district contact paths and review dates.
