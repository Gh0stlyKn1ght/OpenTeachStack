# Chapter Overview

## Core Idea

Account Hygiene and MFA is the maintenance discipline behind reliable teaching systems.  
In this chapter we move from “I should do better security” to a small, repeatable operating rhythm for accounts you use every day.

## Do This

- Write three account groups you actively use: for example `district account`, `personal account`, `public publishing account`.
- For each group, identify one high-impact account category (example: email, learning platform, grading system, cloud storage).
- Rank each group by impact if compromised: high, medium, low.
- Decide your chapter target: the one account category you will secure first before the next section.
- Record a short baseline note that will feed your chapter artifact.

## Evidence of Completion

- 1 baseline note with 3 account groups and a clear ranking.
- 1 clear “first action” that is realistic for one week.
- A short link between this chapter and one later chapter (for example usernames, phishing, or website safety).
- A local private version of this baseline is identified for sharing.

## Safety Check

- Do not include any live credentials, recovery codes, or security answers in this note.
- Keep account identifiers partially redacted until you have a secure private workspace.
- If you suspect active compromise, stop and escalate via district IT before continuing coursework.
- Save sensitive notes in a private folder or local-only note, and share only a sanitized summary.

## Reflection

What is one account maintenance step that would reduce risk and is easy to repeat every week?
