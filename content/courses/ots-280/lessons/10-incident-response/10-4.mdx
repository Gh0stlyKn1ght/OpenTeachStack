# Incident Response Plan

## Why This Matters

Teacher safety work should reduce exposure when a teacher can point to the decision, the classroom use, the evidence, and the next review step.

In **Incident Response for Teachers**, the work sits inside teacher digital safety, account hygiene, public profile review, connection safety, and incident response. The risk is turning safety into fear instead of reducing specific exposure with practical routines. This lesson narrows that risk into one visible update to **What to do if something happens plan**.

## Core Idea

Build the **What to do if something happens plan** as a usable artifact, not just a note. The finished version shows the decision, the classroom context, the evidence, and the next maintenance step.

The goal is not to make a perfect document in one pass. The goal is to make the teacher decision clear enough that another educator can inspect it, reuse it, or challenge it without needing a private explanation.

## Lesson Focus

Because this is an artifact lesson, the teacher turns incident response into something another educator can use. The artifact needs labels, examples, status, review notes, and a next action instead of a vague promise to improve later.

In 10.4, focus on incident response. Choose one concrete classroom material or teacher decision from this list: an account, profile, device setting, website form, repository, domain record, or incident note. Connect it to incident response, then decide whether it needs to be created, revised, tested, credited, restricted, published, or archived.

Connect incident response back to incident response teachers. That connection matters because a polished artifact can still fail if the lesson purpose, boundary, evidence, and maintenance routine are not visible.

## Classroom Example

Use this working situation: a teacher is auditing accounts, public profiles, device settings, repo exposure, website forms, and response steps without collecting private student information.

For **Incident Response Plan**, choose one concrete classroom material or teacher decision from this list: an account, profile, device setting, website form, repository, domain record, or incident note. Keep the example small enough to review during the lesson and focused on incident response.

Example entry:

| Decision | Classroom version |
| --- | --- |
| Teaching purpose | This work supports incident response inside incident response teachers. |
| Evidence | The teacher can point to risk map rows, account inventory status, MFA notes, public profile findings, website checks, and response contacts. |
| Boundary | The teacher names what incident response may change and what must stay protected or reviewed. |
| Maintenance | The next update is triggered by a date, failed check, student confusion, changed source, or incident response review. |

## Build The Work

Update the **What to do if something happens plan** with one row or section for **Incident Response Plan**.

| Field | Your note |
| --- | --- |
| Lesson focus | incident response |
| Classroom context | Where incident response teachers appears in your course, unit, tool, site, media packet, safety routine, or capstone |
| Working material | Classroom material or teacher decision: an account, profile, device setting, website form, repository, domain record, or incident note |
| Teacher decision | What incident response causes you to keep, revise, test, publish, restrict, or remove |
| Artifact status | What proves incident response is ready or what blocks it |
| Maintenance trigger | When the incident response decision should be checked again |

## Quality Check

Before moving on, test the work against this standard: the action lowers a named risk, avoids unnecessary data collection, and gives the teacher a calm next step.

A reviewer should be able to find the incident response decision, inspect the incident response evidence, and understand how it supports incident response teachers.

If the work only names a tool or a file, revise it. The artifact needs to show the teaching purpose, evidence, boundary, and maintenance routine.

## Reflection

What would another teacher need to know before trusting this part of the work? Add that missing context to the artifact before you continue.
