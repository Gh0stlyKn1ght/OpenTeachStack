# OTS-301 Examples

Use these examples to support course-site design for accessibility and maintainability.

- completed-artifact.md: course sitemap and boundary map.
- weak-version.md: mixed public/private content with unclear permissions.
- strong-version.md: accessible map with explicit publishing boundaries.
- reviewer-notes.md: broken-link and maintenance review checklist.

Assumptions:
- Use sample domains and non-production examples.
- Accessibility checks include keyboard, semantic headings, and color contrast.
