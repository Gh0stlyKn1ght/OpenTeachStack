# Audience and Use Cases

## Why This Matters

A course site is a maintained teaching surface when a teacher can point to the decision, the classroom use, the evidence, and the next review step.

In **Course Site Strategy**, the work sits inside course websites, static hosting, course hub structure, accessibility, privacy, and maintenance. The risk is treating a live website as finished when links, privacy, accessibility, source notes, and update routines are still unclear. This lesson narrows that risk into one visible update to **Course site plan**.

## Core Idea

Turn the lesson idea into an inspectable decision. Name the classroom use, the evidence, the review step, and the next revision trigger.

The goal is not to make a perfect document in one pass. The goal is to make the teacher decision clear enough that another educator can inspect it, reuse it, or challenge it without needing a private explanation.

## Lesson Focus

Because this is a focused section, the teacher isolates the audience use cases decision from the rest of the course. The lesson asks what this idea changes, where it appears, and how the teacher will know it worked.

In 01.1, focus on audience use cases. Choose one concrete classroom material or teacher decision from this list: a page, link, navigation item, resource entry, deployment target, or maintenance task. Connect it to course site, then decide whether it needs to be created, revised, tested, credited, restricted, published, or archived.

Connect audience use cases back to course site strategy. That connection matters because a polished artifact can still fail if the lesson purpose, boundary, evidence, and maintenance routine are not visible.

## Classroom Example

Use this working situation: a teacher is publishing a course hub with a homepage, course pages, resource pages, safety checks, release notes, and a maintenance calendar.

For **Audience and Use Cases**, choose one concrete classroom material or teacher decision from this list: a page, link, navigation item, resource entry, deployment target, or maintenance task. Keep the example small enough to review during the lesson and focused on audience use cases.

## Quality Check

Before moving on, test the work against this standard: students can find the current material, sensitive files are not public, links work, and the teacher knows when to update the site.

A reviewer should be able to find the audience use cases decision, inspect the course site evidence, and understand how it supports course site strategy.

If the work only names a tool or a file, revise it. The artifact needs to show the teaching purpose, evidence, boundary, and maintenance routine.
