# Course Site Plan

## Why This Matters

A course site is a maintained teaching surface when a teacher can point to the decision, the classroom use, the evidence, and the next review step.

In **Course Site Strategy**, the work sits inside course websites, static hosting, course hub structure, accessibility, privacy, and maintenance. The risk is treating a live website as finished when links, privacy, accessibility, source notes, and update routines are still unclear. This lesson narrows that risk into one visible update to **Course site plan**.

## Core Idea

Build the **Course site plan** as a usable artifact, not just a note. The finished version shows the decision, the classroom context, the evidence, and the next maintenance step.

The goal is not to make a perfect document in one pass. The goal is to make the teacher decision clear enough that another educator can inspect it, reuse it, or challenge it without needing a private explanation.

## Lesson Focus

Because this is an artifact lesson, the teacher turns course site into something another educator can use. The artifact needs labels, examples, status, review notes, and a next action instead of a vague promise to improve later.

In 01.3, focus on course site. Choose one concrete classroom material or teacher decision from this list: a page, link, navigation item, resource entry, deployment target, or maintenance task. Connect it to course site, then decide whether it needs to be created, revised, tested, credited, restricted, published, or archived.

Connect course site back to course site strategy. That connection matters because a polished artifact can still fail if the lesson purpose, boundary, evidence, and maintenance routine are not visible.

## Classroom Example

Use this working situation: a teacher is publishing a course hub with a homepage, course pages, resource pages, safety checks, release notes, and a maintenance calendar.

For **Course Site Plan**, choose one concrete classroom material or teacher decision from this list: a page, link, navigation item, resource entry, deployment target, or maintenance task. Keep the example small enough to review during the lesson and focused on course site.

Example entry:

| Decision | Classroom version |
| --- | --- |
| Teaching purpose | This work supports course site inside course site strategy. |
| Evidence | The teacher can point to site plan rows, sitemap entries, link checks, privacy notes, accessibility fixes, release checklist items, and maintenance tasks. |
| Boundary | The teacher names what course site may change and what must stay protected or reviewed. |
| Maintenance | The next update is triggered by a date, failed check, student confusion, changed source, or course site review. |

## Build The Work

Update the **Course site plan** with one row or section for **Course Site Plan**.

| Field | Your note |
| --- | --- |
| Lesson focus | course site |

| Working material | Classroom material or teacher decision: a page, link, navigation item, resource entry, deployment target, or maintenance task |
| Teacher decision | What course site causes you to keep, revise, test, publish, restrict, or remove |
| Artifact status | What proves course site is ready or what blocks it |
| Maintenance trigger | When the course site decision should be checked again |

## Quality Check

Before moving on, test the work against this standard: students can find the current material, sensitive files are not public, links work, and the teacher knows when to update the site.

A reviewer should be able to find the course site decision, inspect the course site evidence, and understand how it supports course site strategy.

If the work only names a tool or a file, revise it. The artifact needs to show the teaching purpose, evidence, boundary, and maintenance routine.
