# Web Concepts Map

## Why This Matters

A course site is a maintained teaching surface when a teacher can point to the decision, the classroom use, the evidence, and the next review step.

In **Web Basics for Teachers**, the work sits inside course websites, static hosting, course hub structure, accessibility, privacy, and maintenance. The risk is treating a live website as finished when links, privacy, accessibility, source notes, and update routines are still unclear. This lesson narrows that risk into one visible update to **Web concepts map**.

## Core Idea

Build the **Web concepts map** as a usable artifact, not just a note. The finished version shows the decision, the classroom context, the evidence, and the next maintenance step.

The goal is not to make a perfect document in one pass. The goal is to make the teacher decision clear enough that another educator can inspect it, reuse it, or challenge it without needing a private explanation.

## Lesson Focus

Because this is an artifact lesson, the teacher turns web concepts map into something another educator can use. The artifact needs labels, examples, status, review notes, and a next action instead of a vague promise to improve later.

In 02.3, focus on web concepts map. Choose one concrete classroom material or teacher decision from this list: a page, link, navigation item, resource entry, deployment target, or maintenance task. Connect it to web concepts map, then decide whether it needs to be created, revised, tested, credited, restricted, published, or archived.

Connect web concepts map back to web basics teachers. That connection matters because a polished artifact can still fail if the lesson purpose, boundary, evidence, and maintenance routine are not visible.

## Quality Check

Before moving on, test the work against this standard: students can find the current material, sensitive files are not public, links work, and the teacher knows when to update the site.

A reviewer should be able to find the web concepts map decision, inspect the web concepts map evidence, and understand how it supports web basics teachers.

If the work only names a tool or a file, revise it. The artifact needs to show the teaching purpose, evidence, boundary, and maintenance routine.
