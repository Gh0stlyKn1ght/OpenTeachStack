# OTS-320 Privacy, Accessibility, and Safety Review

Status: complete for authored course examples.

## Privacy Review

- Course examples use fake or teacher-created data only.
- Prompts explicitly forbid student names, grades, IDs, emails, rosters, account credentials, hidden answer keys, private messages, and private files.
- Agent-built tool examples are scoped to teacher-owned workflows and avoid production student systems.
- Stop conditions are included for authentication, permissions, payments, student data, and district-managed systems.

## Accessibility Review

- Route-visible lessons use headings, lists, and concise task language.
- The diff review checklist includes preservation of labels, alt text, captions, accessibility text, and safety notes.
- Tool-building guidance requires manual UI review for page or tool changes.
- Fake-data testing includes long text and narrow mobile view checks when a tool has a UI.

## Safety Review

- Every chapter includes verification language before classroom use.
- Agent work is framed as a reviewable draft, not an authority.
- Branch, copy, diff, test, rollback, and approval evidence are required before accepting generated changes.
- Official documentation references are attached or queued for Codex, Claude Code, VS Code, GitHub, and Apps Script behavior.
