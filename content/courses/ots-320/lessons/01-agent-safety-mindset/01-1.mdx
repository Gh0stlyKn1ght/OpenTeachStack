# What Agents Can and Cannot Own

## Why This Matters

An agent can draft changes when a teacher can point to the decision, the classroom use, the evidence, and the next review step.

In **Agent Safety Mindset**, the work sits inside safe coding-agent workflows for small teacher tools, prompts, diffs, tests, and review evidence. The risk is letting an agent own the instructional decision, overwrite authored content, or treat a passing build as the only proof. This lesson narrows that risk into one visible update to **Agent safety checklist**.

## Core Idea

Turn the lesson idea into an inspectable decision. Name the classroom use, the evidence, the review step, and the next revision trigger.

The goal is not to make a perfect document in one pass. The goal is to make the teacher decision clear enough that another educator can inspect it, reuse it, or challenge it without needing a private explanation.

## Lesson Focus

Because this is a focused section, the teacher isolates the what agents can cannot own decision from the rest of the course. The lesson asks what this idea changes, where it appears, and how the teacher will know it worked.

In 01.1, focus on what agents can cannot own. Choose one concrete classroom material or teacher decision from this list: a prompt, branch, diff, component, test result, tool screen, or review log. Connect it to agent safety checklist, then decide whether it needs to be created, revised, tested, credited, restricted, published, or archived.

Connect what agents can cannot own back to agent safety mindset. That connection matters because a polished artifact can still fail if the lesson purpose, boundary, evidence, and maintenance routine are not visible.

## Classroom Example

Use this working situation: a teacher asks an agent to build a small classroom tool, reviews the diff, protects authored content, runs tests, and records what changed.

## Quality Check

Before moving on, test the work against this standard: the teacher can explain the change, verify the behavior, reject unsafe edits, and preserve the learning goal.

A reviewer should be able to find the what agents can cannot own decision, inspect the agent safety checklist evidence, and understand how it supports agent safety mindset.

If the work only names a tool or a file, revise it. The artifact needs to show the teaching purpose, evidence, boundary, and maintenance routine.
