# Chapter Plan

## Why This Matters

An agent can draft changes when a teacher can point to the decision, the classroom use, the evidence, and the next review step.

In **Repo and Branch Workflow**, the work sits inside safe coding-agent workflows for small teacher tools, prompts, diffs, tests, and review evidence. The risk is letting an agent own the instructional decision, overwrite authored content, or treat a passing build as the only proof. This lesson narrows that risk into one visible update to **Branch workflow notes**.

## Core Idea

Map the chapter before building. Choose one classroom material or teacher decision from this list: a prompt, branch, diff, component, test result, tool screen, or review log. Then name the classroom purpose it serves and the evidence that will show whether the work is ready.

The goal is not to make a perfect document in one pass. The goal is to make the teacher decision clear enough that another educator can inspect it, reuse it, or challenge it without needing a private explanation.

## Lesson Focus

Because this is the opening plan for repo branch workflow, the teacher names the sequence before creating files or artifacts. The plan separates what will be learned, what will be built, and what evidence will prove that the build is ready.

In 02.0, focus on chapter planning decisions. Choose one concrete classroom material or teacher decision from this list: a prompt, branch, diff, component, test result, tool screen, or review log. Connect it to branch workflow notes, then decide whether it needs to be created, revised, tested, credited, restricted, published, or archived.

Connect chapter planning decisions back to repo branch workflow. That connection matters because a polished artifact can still fail if the lesson purpose, boundary, evidence, and maintenance routine are not visible.

## Quality Check

Before moving on, test the work against this standard: the teacher can explain the change, verify the behavior, reject unsafe edits, and preserve the learning goal.

A reviewer should be able to find the chapter planning decisions decision, inspect the branch workflow notes evidence, and understand how it supports repo branch workflow.

If the work only names a tool or a file, revise it. The artifact needs to show the teaching purpose, evidence, boundary, and maintenance routine.
