# Safe Agent Prompt

## Why This Matters

An agent can draft changes when a teacher can point to the decision, the classroom use, the evidence, and the next review step.

In **Prompting Coding Agents**, the work sits inside safe coding-agent workflows for small teacher tools, prompts, diffs, tests, and review evidence. The risk is letting an agent own the instructional decision, overwrite authored content, or treat a passing build as the only proof. This lesson narrows that risk into one visible update to **Safe agent prompt**.

## Core Idea

Build the **Safe agent prompt** as a usable artifact, not just a note. The finished version shows the decision, the classroom context, the evidence, and the next maintenance step.

The goal is not to make a perfect document in one pass. The goal is to make the teacher decision clear enough that another educator can inspect it, reuse it, or challenge it without needing a private explanation.

## Lesson Focus

Because this is an artifact lesson, the teacher turns safe agent prompt into something another educator can use. The artifact needs labels, examples, status, review notes, and a next action instead of a vague promise to improve later.

In 03.3, focus on safe agent prompt. Choose one concrete classroom material or teacher decision from this list: a prompt, branch, diff, component, test result, tool screen, or review log. Connect it to safe agent prompt, then decide whether it needs to be created, revised, tested, credited, restricted, published, or archived.

Connect safe agent prompt back to prompting coding agents. That connection matters because a polished artifact can still fail if the lesson purpose, boundary, evidence, and maintenance routine are not visible.

Use this working situation: a teacher asks an agent to build a small classroom tool, reviews the diff, protects authored content, runs tests, and records what changed.

## Quality Check

Before moving on, test the work against this standard: the teacher can explain the change, verify the behavior, reject unsafe edits, and preserve the learning goal.

A reviewer should be able to find the safe agent prompt decision, inspect the safe agent prompt evidence, and understand how it supports prompting coding agents.

If the work only names a tool or a file, revise it. The artifact needs to show the teaching purpose, evidence, boundary, and maintenance routine.
