# Testing Evidence

## Why This Matters

An agent can draft changes when a teacher can point to the decision, the classroom use, the evidence, and the next review step.

In **Reviewed Agent Build**, the work sits inside safe coding-agent workflows for small teacher tools, prompts, diffs, tests, and review evidence. The risk is letting an agent own the instructional decision, overwrite authored content, or treat a passing build as the only proof. This lesson narrows that risk into one visible update to **Reviewed agent-built classroom tool**.

## Core Idea

Keep the technical move small enough to verify. Record the input, expected output, test case, observed result, and what the teacher should do if it fails.

The goal is not to make a perfect document in one pass. The goal is to make the teacher decision clear enough that another educator can inspect it, reuse it, or challenge it without needing a private explanation.

## Lesson Focus

Because this is a focused section, the teacher isolates the testing evidence decision from the rest of the course. The lesson asks what this idea changes, where it appears, and how the teacher will know it worked.

In 06.2, focus on testing evidence. Choose one concrete classroom material or teacher decision from this list: a prompt, branch, diff, component, test result, tool screen, or review log. Connect it to reviewed agent built classroom tool, then decide whether it needs to be created, revised, tested, credited, restricted, published, or archived.

Connect testing evidence back to reviewed agent build. That connection matters because a polished artifact can still fail if the lesson purpose, boundary, evidence, and maintenance routine are not visible.

## Quality Check

Before moving on, test the work against this standard: the teacher can explain the change, verify the behavior, reject unsafe edits, and preserve the learning goal.

A reviewer should be able to find the testing evidence decision, inspect the reviewed agent built classroom tool evidence, and understand how it supports reviewed agent build.

If the work only names a tool or a file, revise it. The artifact needs to show the teaching purpose, evidence, boundary, and maintenance routine.
