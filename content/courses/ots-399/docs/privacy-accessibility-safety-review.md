# OTS-399 Privacy, Accessibility, and Safety Review

Status: complete for the authored capstone tranche.

## Privacy Review

- Capstone examples require fake, public, or teacher-created data only.
- Student names, grades, IDs, emails, rosters, faces, schedules, private messages, private files, credentials, and hidden answer keys are excluded from public artifacts.
- Public, private, and mixed-release artifacts are tracked in the artifact index.

## Accessibility Review

- Route-visible authoring uses headings, lists, clear task language, and explicit evidence checks.
- The release checklist requires accessibility review before completion.
- Publishing review includes headings, link labels, mobile readability, keyboard access, contrast, and text alternatives.

## Safety Review

- Public sites, repositories, downloads, and shareable packets are gated by source, license, privacy, accessibility, link, and maintenance checks.
- Technical evidence must document inputs, outputs, tests, data boundaries, and rollback or maintenance notes.
- Peer review is required before final release.
