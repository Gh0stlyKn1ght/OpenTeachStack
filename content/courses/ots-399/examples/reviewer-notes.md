# Reviewer Notes

## Capstone Studio Reviewer Notes

Use these notes when reviewing an OTS-399 capstone system.

## Ready

- Teaching need is specific and tied to classroom evidence.
- Artifact index links to proposal, system map, technical evidence, safety review, peer review, and maintenance plan.
- Public/private boundaries are visible.
- Technical evidence uses fake data and includes rollback.
- Release decision names owner and next review trigger.

## Revise

- Evidence exists but one artifact lacks owner, date, or reviewer note.
- Safety review is mostly complete but missing a license, accessibility, or access note.
- Presentation explains the project but does not show where reviewers can inspect the evidence.
- Maintenance plan has a date but no condition that would start an earlier review.

## Blocked

- Public packet includes real student data, private school links, restricted files, or unlicensed media.
- Technical evidence writes to live data during review.
- Reviewer cannot trace the teaching need to artifact evidence.
- Release decision says ready without naming privacy, source, access, and maintenance checks.

## Sample Reviewer Comment

> Revise: the capstone is coherent and the technical evidence is fake-data-safe, but public release should wait until the resource license note and access date are visible in the release safety review.
