# Teachers Need Digital Self-Defense

This is not paranoia.

This is basic maintenance for being an educator online.

Teachers are public-facing professionals. We post resources, answer emails,
share links, run clubs, manage Google Classroom, publish videos, build course
sites, and sometimes put our work on GitHub. That visibility can be useful. It
can also expose more than we meant to share.

The problem is not that teachers are careless.

The problem is that most teachers were never taught the workflow.

## The Real Problem

A teacher can easily end up with:

- the same username everywhere
- a personal email connected to public work
- old profiles with family or location clues
- class schedules or room numbers posted publicly
- student screenshots saved in the wrong place
- public websites with too much contact information
- GitHub repos with private notes or config files
- AI tools receiving information that should have stayed private

Nobody has to be doing anything dramatic for this to become a problem. It is
usually just years of accounts, files, profiles, and platforms piling up.

## The Better Frame

Cyber safety for teachers should not sound like fearmongering.

It should sound like this:

> You are a teacher in a public-facing job. You deserve basic digital
> self-defense.

That means:

- use a password manager
- turn on MFA
- separate personal, school, and public identities
- check what your public profiles reveal
- review images and PDFs before publishing
- keep student information out of public tools
- audit teacher websites before sharing them
- check public repos for secrets and private details
- have a simple plan if something goes wrong

## Where This Fits in Teaching Teachers

OTS-101 should stay beginner-friendly. Teachers do not need a full cyber safety
course before they can build a mini-unit system.

But once teachers start publishing course sites, public portfolios, GitHub
repos, resource hubs, YouTube videos, and AI-assisted materials, cyber safety
deserves its own track.

That is why Teaching Teachers now includes a planned course:

**OTS-280 — Cyber Safety for Educators**

The goal is calm, practical digital self-defense:

**Identity, Privacy, Accounts, and Website Safety for Teachers**

## The Maintenance Loop

The workflow is simple:

1. Map what is public.
2. Remove what does not need to be public.
3. Separate identities where it matters.
4. Protect important accounts.
5. Verify before publishing.
6. Repeat the audit once or twice a year.

Good security for teachers is not about becoming a cybersecurity expert.

It is about building habits that reduce risk while keeping the work useful.
