# PDF and Export Strategy

Date: 2026-06-22

Scope: OTS-101 syllabus, template library, template detail pages, and future course-book exports.

## Decision

Teaching Teachers will use browser print-to-PDF as the first PDF path and Markdown downloads as the canonical editable template export.

Do not add fake Google Docs, Google Sheets, DOCX, or PDF download links unless the generated files exist in the repo or are produced by a verified route.

## Current Export Paths

| Surface | Primary export | Editable source | Status |
| --- | --- | --- | --- |
| Syllabus | Browser print / Save as PDF | Site page at `/syllabus` | Active |
| Template library | Individual template pages and Markdown downloads | `src/lib/templates.ts` | Active |
| Template detail page | Browser print / Save as PDF plus Markdown download | `/templates/[slug]/download` | Active |
| Course-book pages and chapters | Browser print / Save as PDF | Book route pages | Active |

## Implementation Notes

- `src/components/PrintPageButton.tsx` is the shared client-side print action.
- `src/app/syllabus/page.tsx` exposes a print/PDF action near the syllabus header.
- `src/app/templates/[slug]/page.tsx` exposes both Markdown download and print/PDF actions.
- `src/app/templates/[slug]/download/route.ts` returns static Markdown from `renderTemplateMarkdown`.
- `src/components/book/BookChapterHeader.tsx` exposes the same print/PDF action across course-book entry, chapter, and section pages.
- `src/app/globals.css` contains print styles that remove navigation, footer, buttons, and `[data-print-hide]` elements from printed output.

## Why This Strategy

Browser print-to-PDF keeps the release small, accessible, and dependency-light. It avoids introducing a server-side PDF rendering stack before the content model settles.

Markdown downloads keep templates editable across Google Docs, Google Sheets, local Markdown editors, LMS pages, and district planning systems. They also avoid implying that official Google files exist before they have actually been created.

## Future Upgrade Path

Add generated PDF or DOCX files only after these conditions are met:

1. The source content has a stable owner and update process.
2. The generated file can be reproduced locally or in CI.
3. The file includes accessible headings, readable table structure, link text, and license notes.
4. The download route or static artifact is covered by a route/link check.
5. The roadmap names the generated file type explicitly.

Potential v1.1 additions:

- `npm run export:pdf` using Playwright or another browser renderer.
- Static `public/downloads/` artifacts generated from syllabus and templates.
- A template pack ZIP only after every included file is real and regenerated by script.

## Release Rule

For v1.0, the release promise is:

- Syllabus: printable page, suitable for browser Save as PDF.
- Templates: site pages, printable pages, and real Markdown downloads.
- Google Docs/Sheets: future pack only after real files exist.
