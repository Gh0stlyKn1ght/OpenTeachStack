# OTS-101 Lesson Type Remediation Summary

Date: 2026-06-26

## What changed

The OTS-101 authored draft pass was too uniform. Many lessons used the same artifact-style section pattern even when the lesson purpose was concept, comparison, workflow, case study, checkpoint, or capstone assembly.

A lesson-type audit was created at `docs/reports/OTS101_LESSON_TYPE_AUDIT.md`.

## Remediation performed

The remediation pass preserved lesson body content but changed repeated section headings according to lesson purpose:

- Concept lessons now use headings like `The Problem`, `The Concept`, `Concrete Example`, `Common Misunderstanding`, and `Better Mental Model`.
- Framework lessons now use headings like `Why This Framework Exists`, `The Framework`, `Failure Mode`, and `How to Use It`.
- Comparison lessons now use headings like `The Confusion`, `Side-by-Side Comparison`, `Decision Rule`, and `Practice`.
- Workflow lessons now use headings like `When to Use This Workflow`, `Workflow Steps`, `Example Run`, `Review Gate`, and `Next Action`.
- Case-study lessons now use headings like `What Happened`, `What Looked Fine`, `What Was Actually Broken`, `Root Cause`, and `The Fix`.
- Checkpoint lessons now use headings like `What You Are Checking`, `Review Criteria`, `Self-Audit`, `Review Checklist`, and `Next Step`.
- Capstone assembly lessons now use headings like `What You Are Assembling`, `Required Pieces`, `Assembly Steps`, `Release Check`, and `Final Revision Note`.

## What did not change

- The course remains a draft.
- Human review is still required.
- The remediation did not invent new examples or fake content.
- Artifact build lessons may still keep artifact-style structure where it fits.

## Remaining review need

A human review pass should still check whether the changed headings match the body content naturally. Some lessons may need deeper prose edits, but the course no longer presents every lesson in the same visible scaffold.
