# HTML/CSS lesson pages

## The real problem

Teachers can lose ownership when every lesson lives inside a platform-specific editor.

## Quick answer

A small static HTML page can hold directions, links, source notes, and student tasks without a heavy app.

## When to use this

Use this for one-day lessons, small course-content packets, resource pages, or public exemplars.

## Workflow

1. Start with semantic HTML.
2. Keep CSS readable.
3. Put the core task near the top.
4. Test on mobile.
5. Archive the source file with the lesson.

## What to verify

Check contrast, heading order, link text, and whether the page still works offline or after export.

## Common mistakes

- Building a full app for a one-page lesson.
- Styling before the learning flow is clear.
- Hiding source notes.
