# Build a one-day lesson site

## The real problem

A good lesson can still be hard to teach when directions, links, sources, and materials are scattered.

## Quick answer

Make one small page with the objective, materials, steps, links, checks, and backup plan. Good enough to teach, then improve it.

## When to use this

Use this when a lesson needs a clear public or local home for students, substitutes, or your future self.

## Workflow

1. Write the learning target in teacher-readable language.
2. Add the materials and links students actually need.
3. Put the lesson flow in order.
4. Add a check for understanding.
5. Add source notes and accessibility checks.
6. Test every link before teaching.

```html
<main>
  <h1>One-Day Lesson Title</h1>
  <p>Today you will...</p>
  <ol>
    <li>Open the source.</li>
    <li>Build the artifact.</li>
    <li>Submit the reflection.</li>
  </ol>
</main>
```

## What to verify

Check links, reading level, privacy, alt text, video captions, and whether students can start without extra explanation.

## Common mistakes

- Turning one day into a full website.
- Hiding the task below decorative sections.
- Embedding resources without checking permissions.
